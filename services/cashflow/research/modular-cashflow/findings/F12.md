# F12: Configuration-Driven UI and Reporting Engine for Modular Financial Platforms

## 1. Schema-Driven Form Rendering (JSON Schema Pattern)

Form builders render forms from JSON schema using a **declarative node tree**. Each node specifies type, validation, labels, and conditional logic. The renderer traverses the tree and maps each node to a concrete input component.

### FormKit Schema (Vue/React) — Best-in-class pattern

```typescript
const financialEntrySchema = [
  {
    $formkit: 'select',
    name: 'account_type',
    label: 'Account Type',
    options: ['asset', 'liability', 'equity', 'revenue', 'expense'],
    validation: 'required',
  },
  {
    $formkit: 'text',
    name: 'account_name',
    label: 'Account Name',
    validation: 'required|min:2|max:100',
  },
  {
    $formkit: 'number',
    name: 'opening_balance',
    label: 'Opening Balance',
    validation: 'required|number',
    help: 'Beginning balance for this period',
  },
  // Conditional: show depreciation field only for fixed assets
  {
    $formkit: 'number',
    if: '$get(account_type).value === "asset"',
    name: 'depreciation_method',
    label: 'Depreciation Method',
    options: ['straight-line', 'declining-balance', 'units-of-production'],
  },
]
```

Key capabilities demonstrated:
- **`$formkit` shorthand**: Maps directly to a registered input component.
- **`if` conditionals**: Fields show/hide based on other field values.
- **`$get(id).value`**: Cross-field references for conditional rendering.
- **Validation strings**: Declarative validation rules (`required|email|min:5`).
- **`$el` nodes**: Raw HTML elements for layout (headings, dividers, sections).
- **`$cmp` nodes**: Any React/Vue component rendered from schema.
- **Expressions**: Arithmetic and boolean logic in `$:` prefixed expressions.

### Pattern for Financial Data Entry

```typescript
const journalEntrySchema = [
  { $el: 'h2', children: 'Journal Entry', attrs: { class: 'text-xl font-bold' } },
  {
    $formkit: 'date',
    name: 'entry_date',
    label: 'Transaction Date',
    validation: 'required',
  },
  {
    $formkit: 'text',
    name: 'description',
    label: 'Description',
    validation: 'required|min:3',
  },
  // Dynamic line items via loops
  {
    $el: 'div',
    for: ['line', 'index', '$lines'],
    children: [
      { $formkit: 'select', name: 'account', label: 'Account', options: '$chartOfAccounts' },
      { $formkit: 'number', name: 'debit', label: 'Debit' },
      { $formkit: 'number', name: 'credit', label: 'Credit' },
    ],
  },
  // Auto-calculate balance indicator
  {
    $el: 'div',
    if: '$totalDebit !== $totalCredit',
    children: '$: "Balance: " + ($totalDebit - $totalCredit)',
    attrs: { style: { color: 'red' } },
  },
]
```

**Libraries**: FormKit (Vue/React), React Hook Form + JSON Schema, Ajv for validation, rjsf (React JSON Schema Form), Formly (Angular).

## 2. Dashboard Configuration Frameworks (Grafana Model)

### Grafana Dashboard JSON Model

Grafana dashboards are **fully defined as JSON** — panels, layouts, data sources, variables, and time ranges. This is the gold standard for configurable dashboards.

```json
{
  "id": null,
  "uid": "cashflow-overview",
  "title": "Cash Flow Overview",
  "tags": ["finance", "cashflow"],
  "timezone": "browser",
  "editable": true,
  "panels": [
    {
      "type": "timeseries",
      "title": "Monthly Cash Flow",
      "gridPos": { "x": 0, "y": 0, "w": 12, "h": 8 },
      "datasource": { "type": "postgres", "uid": "cashflow-db" },
      "targets": [
        {
          "rawSql": "SELECT date_trunc('month', date) AS time, SUM(amount) AS value FROM transactions GROUP BY 1",
          "format": "time_series"
        }
      ]
    },
    {
      "type": "stat",
      "title": "Total Revenue",
      "gridPos": { "x": 12, "y": 0, "w": 6, "h": 4 },
      "fieldConfig": {
        "defaults": {
          "unit": "currencyUSD",
          "thresholds": {
            "steps": [
              { "value": 0, "color": "red" },
              { "value": 100000, "color": "green" }
            ]
          }
        }
      }
    }
  ],
  "templating": {
    "list": [
      {
        "name": "period",
        "type": "custom",
        "query": "monthly,quarterly,yearly",
        "current": { "text": "monthly", "value": "monthly" }
      },
      {
        "name": "entity",
        "type": "query",
        "query": "SELECT DISTINCT entity_name FROM entities",
        "datasource": { "uid": "cashflow-db" }
      }
    ]
  },
  "time": { "from": "now-1y", "to": "now" }
}
```

### Key Configuration Patterns

**Grid layout system**: Panels positioned via `gridPos: { x, y, w, h }` on a 24-column grid. Width 1-24, height in 30px units. Negative gravity pulls panels up to fill gaps.

**Templating variables**: Dynamic dropdowns that filter dashboard data. Types: `custom` (static list), `query` (SQL from datasource), `interval`, `textbox`.

**Panel types for financial dashboards**:
- `timeseries` — trend lines, cash flow over time
- `stat` — single big-number KPIs (total revenue, outstanding AR)
- `table` — detailed line-item views
- `barchart` — category comparisons
- `piechart` — allocation breakdowns
- `gauge` — threshold-based metrics (burn rate)
- `text` — markdown/PDF embeds for narrative

**Model evolution**: Grafana now has V2 (Kubernetes-style resources with `apiVersion`, `kind`, `metadata`, `spec`) and V1 (classic JSON wrapped in K8s envelope). The classic model remains most common for import/export.

**Applicable to cashflow**: Define each dashboard as JSON. A dashboard registry file maps module IDs to their available dashboards. The UI renderer reads the JSON and instantiates panel components from a panel library.

## 3. Report Template Engines for Financial Statements

### Report Architecture Pattern

Financial reports follow a **template + data source + renderer** pipeline:

```
Template (layout/structure) + Data (query result) → Renderer → PDF/HTML/Excel
```

### Standard Financial Statement Templates

**Income Statement (P&L) Template Structure**:
```yaml
report: income_statement
title: "Profit & Loss Statement"
sections:
  - name: "Revenue"
    type: section_header
    indent: 0
  - name: "Product Revenue"
    type: line_item
    account_codes: ["4000-4099"]
    calculation: sum
  - name: "Service Revenue"
    type: line_item
    account_codes: ["4100-4199"]
    calculation: sum
  - name: "Total Revenue"
    type: subtotal
    formula: "product_revenue + service_revenue"
    style: bold
  - name: "Cost of Goods Sold"
    type: section_header
    indent: 0
  - name: "Total COGS"
    type: subtotal
    account_codes: ["5000-5999"]
  - name: "Gross Profit"
    type: calculated
    formula: "total_revenue - total_cogs"
    style: bold_double_underline
```

**Balance Sheet Template**:
```yaml
report: balance_sheet
title: "Balance Sheet"
sections:
  - name: "Assets"
    subsections:
      - name: "Current Assets"
        account_codes: ["1000-1999"]
        items: [cash, accounts_receivable, inventory, prepaid_expenses]
      - name: "Fixed Assets"
        account_codes: ["1500-1999"]
        items: [property, equipment, accumulated_depreciation]
  - name: "Liabilities"
    subsections:
      - name: "Current Liabilities"
        account_codes: ["2000-2999"]
      - name: "Long-Term Liabilities"
        account_codes: ["3000-3999"]
  - name: "Equity"
    account_codes: ["4000-4999"]
  validation:
    - equation: "total_assets = total_liabilities + total_equity"
      tolerance: 0.01
```

### Libraries and Patterns

| Library | Language | Strengths |
|---------|----------|-----------|
| **JasperReports** | Java | Enterprise-grade, XML templates (JRXML), subreports, cross-tabs |
| **Crystal Reports** | .NET | Visual designer, data grouping, formula engine |
| **jsPDF + jsPDF-AutoTable** | JS | Client-side PDF, good for invoices, lightweight |
| **React-PDF (@react-pdf/renderer)** | JS/React | Declarative React components → PDF, design-system friendly |
| **PDFKit** | Node.js | Low-level PDF generation, fine-grained control |
| **Apache POI** | Java | Excel/Word generation for financial exports |
| **ExcelJS** | Node.js | XLSX generation with formulas, styling, charts |
| **WeasyPrint** | Python | HTML/CSS → PDF, good for branded reports |

**Pattern for financial statement generation**:
1. Template defines structure (sections, line items, calculations)
2. Data provider maps account codes → queried balances
3. Renderer applies formatting (currency, thousands separator, parentheses for negatives)
4. Validation ensures accounting equation holds (assets = liabilities + equity)

## 4. Role-Based UI Rendering

### Module Manifest + Role Matrix Pattern

```typescript
// Module manifest — defines all available modules
interface ModuleManifest {
  id: string
  name: string
  icon: string
  routes: RouteConfig[]
  features: FeatureConfig[]
  permissions: PermissionConfig[]
}

// Role-to-module mapping
const roleModuleMatrix: Record<string, string[]> = {
  admin: ['*'], // all modules
  accountant: ['journal', 'reports', 'chart-of-accounts', 'bank-reconciliation'],
  viewer: ['dashboard', 'reports'],
  ap_clerk: ['accounts-payable', 'vendor-management', 'payment-processing'],
  ar_clerk: ['accounts-receivable', 'customer-management', 'invoicing'],
}

// Feature-level visibility
const roleFeatureMatrix: Record<string, Record<string, string[]>> = {
  admin: { '*': ['create', 'read', 'update', 'delete', 'export', 'settings'] },
  accountant: {
    journal: ['create', 'read', 'update'],
    reports: ['read', 'export'],
    'chart-of-accounts': ['read', 'update'],
  },
  viewer: {
    dashboard: ['read'],
    reports: ['read'],
  },
}
```

### React Rendering Pattern

```tsx
function useModuleVisibility(userRole: string, activeModules: string[]) {
  const modules = useModuleRegistry()
  
  return useMemo(() => {
    const allowedModules = modules.filter(mod => {
      const roleAllowed = roleModuleMatrix[userRole]?.includes('*') || 
                          roleModuleMatrix[userRole]?.includes(mod.id)
      const moduleActive = activeModules.includes(mod.id)
      return roleAllowed && moduleActive
    })
    return allowedModules
  }, [userRole, activeModules, modules])
}

function useFeaturePermission(moduleId: string, feature: string) {
  const { userRole } = useAuth()
  const allowed = roleFeatureMatrix[userRole]?.[moduleId]?.includes(feature)
    || roleFeatureMatrix[userRole]?.['*']?.includes(feature)
  return allowed
}

// Usage in component
function FinanceApp() {
  const modules = useModuleVisibility(user.role, config.activeModules)
  
  return (
    <AppShell sidebar={<DynamicNav modules={modules} />}>
      <Routes>
        {modules.map(mod => (
          <Route key={mod.id} path={mod.path} element={mod.component} />
        ))}
      </Routes>
    </AppShell>
  )
}
```

### Field-Level Visibility (Financial-Specific)

```typescript
// Hide/show specific fields based on role + module state
const fieldVisibility: Record<string, Record<string, string[]>> = {
  accountant: {
    journal_entry: ['date', 'account', 'debit', 'credit', 'description', 'reference'],
    // Accountants see all fields
  },
  ap_clerk: {
    journal_entry: ['date', 'vendor', 'amount', 'description', 'po_number'],
    // AP clerk sees vendor-oriented fields only
  },
  viewer: {
    journal_entry: ['date', 'account', 'amount', 'description'],
    // Viewer sees read-only summary fields
  },
}
```

**Key principle**: Two-layer access control — module visibility (can user access the section?) and field visibility (which fields within that section?).

**Sidebar component**:
```tsx
function DynamicSidebar({ modules }: { modules: NavModule[] }) {
  const pathname = usePathname()
  
  return (
    <aside className="w-64 border-r bg-muted/50">
      <nav className="p-4 space-y-1">
        {modules.sort((a, b) => a.order - b.order).map(mod => (
          <SidebarItem key={mod.id} module={mod} isActive={pathname.startsWith(mod.path)}>
            {mod.children?.map(child => (
              <SidebarItem key={child.id} module={child} nested />
            ))}
          </SidebarItem>
        ))}
      </nav>
    </aside>
  )
}

function SidebarItem({ module, isActive, children }) {
  const Icon = Icons[module.icon]
  return (
    <div>
      <Link
        href={module.path}
        className={cn(
          "flex items-center gap-2 px-3 py-2 rounded-md text-sm",
          isActive ? "bg-primary text-primary-foreground" : "hover:bg-muted"
        )}
      >
        <Icon className="h-4 w-4" />
        <span>{module.label}</span>
        {module.badge && <Badge>{module.badge}</Badge>}
      </Link>
      {children && <div className="ml-6 mt-1 space-y-1">{children}</div>}
    </div>
  )
}
```

**Key pattern**: Modules are config data, not file-system routes. The sidebar reads the manifest, filters by role + active modules, and renders navigation. Individual page components still live as files but are gated by the same manifest.

## 6. Financial Chart Components and Libraries

### Standard Financial Chart Types

| Chart Type | Use Case | Library |
|------------|----------|---------|
| **Waterfall** | Cash flow build-up (revenue → expenses → net), P&L bridging | Highcharts (built-in), Plotly.js, custom D3, Nivo |
| **Candlestick/OHLC** | Stock price, commodity prices, FX rates | Lightweight Charts (TradingView), Plotly.js, Highcharts |
| **Treemap** | Account hierarchy visualization, revenue by segment | Plotly.js, Highcharts, Recharts (custom) |
| **Sankey** | Fund flow, revenue source → destination, cost allocation | Plotly.js, Highcharts, D3-sankey |
| **Gauge** | KPI metrics (burn rate, utilization %, budget vs actual) | Highcharts, Nivo, RadialBar (Recharts) |
| **Stacked Bar** | Revenue by category over time, expense breakdown | Chart.js, Recharts, Nivo |
| **Line/Area** | Cash flow trends, P&L over periods | Chart.js, Recharts, Highcharts |
| **Pie/Donut** | Asset allocation, revenue mix | Chart.js, Recharts, Nivo |
| **Heatmap** | Daily cash positions, seasonal patterns | Plotly.js, D3, Nivo |

## 7. PDF Generation for Financial Documents

### Invoice Template Pattern (jsPDF + AutoTable)

```javascript
import jsPDF from 'jspdf'
import autoTable from 'jspdf-autotable'

function generateInvoice(invoiceData) {
  const doc = new jsPDF()

  // Header
  doc.setFontSize(24)
  doc.text('INVOICE', 14, 30)
  doc.setFontSize(10)
  doc.text(`Invoice #: ${invoiceData.number}`, 14, 40)
  doc.text(`Date: ${invoiceData.date}`, 14, 46)
  doc.text(`Due Date: ${invoiceData.dueDate}`, 14, 52)

  // Bill to
  doc.setFontSize(12)
  doc.text('Bill To:', 14, 70)
  doc.setFontSize(10)
  doc.text(invoiceData.customer.name, 14, 78)
  doc.text(invoiceData.customer.address, 14, 84)
  doc.text(invoiceData.customer.email, 14, 90)

  // Line items table
  autoTable(doc, {
    startY: 100,
    head: [['Description', 'Quantity', 'Unit Price', 'Amount']],
    body: invoiceData.items.map(item => [
      item.description,
      item.quantity.toString(),
      formatCurrency(item.unitPrice),
      formatCurrency(item.quantity * item.unitPrice),
    ]),
    foot: [
      ['', '', 'Subtotal', formatCurrency(invoiceData.subtotal)],
      ['', '', `Tax (${invoiceData.taxRate}%)`, formatCurrency(invoiceData.tax)],
      ['', '', 'Total', formatCurrency(invoiceData.total)],
    ],
    headStyles: { fillColor: [51, 51, 51] },
    footStyles: { fillColor: [245, 245, 245], textColor: [0, 0, 0], fontStyle: 'bold' },
  })

  doc.save(`invoice-${invoiceData.number}.pdf`)
}
```

### React-PDF Pattern (Declarative)

```tsx
import { Document, Page, Text, View, StyleSheet } from '@react-pdf/renderer'

const styles = StyleSheet.create({

  col1: { width: '50%' },
  col2: { width: '20%', textAlign: 'right' },
  col3: { width: '30%', textAlign: 'right' },
  total: { fontSize: 14, fontWeight: 'bold', marginTop: 20, textAlign: 'right' },

function BalanceSheetPDF({ data }) {
  return (
    <Document>
      <Page size="A4" style={styles.page}>
        <Text style={styles.header}>Balance Sheet</Text>
        <Text>{data.companyName} — As of {data.asOfDate}</Text>

          <View style={styles.row} key={item.id}>
            <Text style={styles.col1}>{item.name}</Text>
            <Text style={styles.col3}>{formatCurrency(item.amount)}</Text>
          </View>
        ))}
        
        <Text style={styles.total}>Total Assets: {formatCurrency(data.totalAssets)}</Text>
        
        <Text style={{ fontSize: 16, marginTop: 20 }}>Liabilities & Equity</Text>
        {data.liabilities.map(item => (
          <View style={styles.row} key={item.id}>
            <Text style={styles.col1}>{item.name}</Text>

```

### Financial Document Layout Patterns

**Invoice/Receipt**: Header (logo, number, dates) → Bill-to block → Line items table → Totals (subtotal, tax, total) → Payment terms → Footer (bank details, notes)

**Tax Document (1099/W-2)**: Fixed layout matching IRS specifications, box positions pre-defined, data injected into specific coordinates.

**Financial Statement**: Company header → Report title → Period → Sections with subsections → Totals → Footnotes → Audit trail.

**Key formatting rules**:
- Negative numbers in parentheses or red: `(1,234.56)` or `$-1,234.56`
- Currency formatting with thousands separator
- Right-aligned numeric columns
- Alternating row shading for readability
- Watermark for draft status

### Theme Provider Pattern (React)

```tsx
interface ThemeConfig {
  brand: string
  mode: 'light' | 'dark' | 'system'
  colors: Record<string, string>
  fonts: Record<string, string>
  borderRadius: 'none' | 'sm' | 'md' | 'lg'
}

const defaultThemes: Record<string, ThemeConfig> = {
  default: {
    brand: 'default',
    mode: 'light',
    colors: { primary: '#2563eb', secondary: '#7c3aed' },
    fonts: { sans: 'Inter', mono: 'JetBrains Mono' },
    borderRadius: 'md',
  },
  dark: {
    brand: 'default',
    mode: 'dark',
    colors: { primary: '#3b82f6', secondary: '#8b5cf6' },
    fonts: { sans: 'Inter', mono: 'JetBrains Mono' },
    borderRadius: 'md',
  },
}

function ThemeProvider({ config, children }: { config: ThemeConfig; children: ReactNode }) {
  useEffect(() => {
    const root = document.documentElement
    root.setAttribute('data-brand', config.brand)
    root.setAttribute('data-theme', config.mode)
    Object.entries(config.colors).forEach(([key, value]) => {
      root.style.setProperty(`--brand-${key}`, value)
    })
  }, [config])

  return <>{children}</>
}
```

### Design Token System (Multi-Brand)

```typescript
// tokens.config.ts — per-brand token definitions
export const brandTokens = {
  default: {
    colors: {
      primary: { 50: '#eff6ff', 500: '#2563eb', 900: '#1e3a5f' },
      success: { 500: '#16a34a' },
      danger: { 500: '#dc2626' },
    },
  },
  acme: {
    colors: {
      primary: { 50: '#fff1f2', 500: '#e11d48', 900: '#881337' },
      success: { 500: '#16a34a' },
      danger: { 500: '#dc2626' },
    },
  },
}

// CSS variable generator
function generateCSSVariables(brand: keyof typeof brandTokens) {
  const tokens = brandTokens[brand]
  return Object.entries(tokens.colors).flatMap(([colorName, shades]) =>
    Object.entries(shades).map(([shade, value]) => 
      `--color-${colorName}-${shade}: ${value};`
    )
  ).join('\n')
}
```

## Summary of Key Patterns for Cashflow Platform

| Concern | Pattern | Implementation |
|---------|---------|----------------|
| **Data Entry** | JSON Schema → Form Components | FormKit schema or custom JSON schema renderer with conditional fields |
| **Dashboard** | JSON Dashboard Model (Grafana-style) | Panels array + gridPos + templating variables + datasource config |
| **Reports** | Template + Data Source + Renderer | YAML/JSON report templates with account code mappings |
| **Access Control** | Module manifest + Role matrix | Two-layer: module visibility + field-level permissions |
| **Navigation** | Config-driven sidebar from module manifest | React component reads manifest, filters by role, renders nav tree |
| **Charts** | Recharts + Lightweight Charts + D3 | Recharts for standard, TradingView for candlestick, D3 for custom Sankey/waterfall |
| **PDF** | jsPDF + AutoTable (invoices) / React-PDF (statements) | Template-driven with financial formatting (currency, negatives in parens) |
| **Theming** | CSS custom properties + data attributes | `data-brand` + `data-theme` selectors, token system per tenant |
