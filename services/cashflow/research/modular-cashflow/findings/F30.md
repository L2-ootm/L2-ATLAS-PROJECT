## Finding 1: Contract Lifecycle Management (CLM) Stages

**Source:** Wikipedia, Juro, Coupa  
**Confidence:** High

Contract lifecycle management encompasses the systematic management of contracts from initiation to termination. The core stages are:

1. **Contract Request/Initiation** - Gathering requirements and data for contract creation
2. **Contract Authoring/Drafting** - Writing terms and clauses into formal documents
3. **Contract Negotiation** - Redlining and revising terms between parties
4. **Contract Approval** - Internal stakeholder sign-off based on risk thresholds
5. **Contract Execution/Signature** - Signing via wet ink or electronic signatures
6. **Contract Obligation Management** - Tracking deliverables and responsibilities
7. **Contract Compliance Monitoring** - Ensuring parties meet contractual obligations
8. **Contract Renewal/Amendment** - Extending or modifying terms before expiration
9. **Contract Termination** - Formal closure and archival

**Key Data Points per Stage:**
- Request: requestor, business justification, budget, timeline
- Authoring: template ID, clause library references, draft versions
- Negotiation: redline history, counterparty comments, version control
- Approval: approver roles, approval criteria, audit trail
- Execution: signature status, signer identities, timestamps
- Obligation: deliverable dates, performance metrics, milestones
- Compliance: check points, violation flags, remediation actions
- Renewal: renewal date, auto-renewal flag, notice periods
- Termination: termination reason, final settlements, archival metadata

- **Contract** (root entity): id, version, status, effective_date, expiration_date, created_at, updated_at

- Contract 1:N Term (multiple terms/conditions)

- Contract 1:N SLA (multiple SLAs)
- Contract 1:N PaymentSchedule (multiple payment milestones)
- Contract 1:1 RenewalConfig (renewal configuration)
- Contract 1:N Amendment (amendments over time)
- Contract 1:N Document (versioned documents)

### Notification Sequence Pattern
```
Day -90: Initial renewal assessment notification to contract owner
Day -60: Decision required - renew, renegotiate, or terminate
Day -45: Auto-renewal opt-out deadline (if applicable)
Day -30: Final reminder - renewal terms locked
Day -14: Escalation to management
Day -7: Last chance notification
Day 0:  Auto-renewal executed OR termination processed
Day +1: Confirmation notification with new terms
```

### Auto-Renewal vs Opt-In Renewal
| Feature | Auto-Renewal | Opt-In Renewal |
|---------|--------------|----------------|
| Default | Continues unless cancelled | Terminates unless renewed |
| Notice Required | Opt-out notice | Opt-in confirmation |
| Risk Level | Higher (accidental renewal) | Lower (intentional) |
| Common Use | SaaS subscriptions | Enterprise services |
| Legal Requirement | Varies by jurisdiction | Generally preferred |

### Penalty Calculation Example
```python
def calculate_sla_penalty(sla: SLA, actual_value: float, period: DateRange) -> PenaltyResult:
    target = sla.target_value
    actual = actual_value
    
    if sla.metric_type == 'percentage':
        performance = actual / target
    else:  # duration or count (lower is better)
        performance = target / actual if actual > 0 else 0
    
    breach_percentage = (1 - performance) * 100
    
    penalties = []
    cumulative_penalty = 0
    
    # Check each threshold
    if breach_percentage >= sla.penalties.critical_breach.threshold:
        penalty = sla.penalties.critical_breach.penalty_amount or (target_value * 0.25)
        penalties.append(("critical", penalty))
        cumulative_penalty += penalty
    elif breach_percentage >= sla.penalties.major_breach.threshold:
        penalty = sla.penalties.major_breach.penalty_amount or (target_value * 0.15)
        penalties.append(("major", penalty))
        cumulative_penalty += penalty
    elif breach_percentage >= sla.penalties.minor_breach.threshold:
        penalty = sla.penalties.minor_breach.penalty_amount or (target_value * 0.05)
        penalties.append(("minor", penalty))
        cumulative_penalty += penalty
    
    return PenaltyResult(
        breach_level=penalties[-1][0] if penalties else "none",
        total_penalty=cumulative_penalty,
        period=period,
        details=penalties
    )
```

### Alerting Pattern
```
SLA Monitoring Pipeline:
1. Data Collection → 2. Metric Calculation → 3. Threshold Check → 4. Alert Dispatch

Alert Channels:
- Warning (>90% of threshold): Email to contract owner
- Minor Breach: Email + Slack to ops team
- Major Breach: Email + Slack + PagerDuty escalation
- Critical Breach: All channels + automatic penalty calculation + management notification
```

### Invoice Validation Against Contract
```python
def validate_invoice_against_contract(invoice: Invoice, contract: Contract) -> ValidationResult:
    errors = []
    warnings = []
    
    # Check contract status
    if contract.status != 'active':
        errors.append(f"Contract {contract.id} is not active (status: {contract.status})")
    
    # Check invoice amount against schedule
    scheduled_amount = get_scheduled_amount(contract, invoice.period)
    if invoice.amount > scheduled_amount * 1.1:  # 10% tolerance
        errors.append(f"Invoice amount {invoice.amount} exceeds scheduled {scheduled_amount}")
    
    # Check payment terms
    if not is_within_payment_terms(invoice, contract.payment_terms):
        warnings.append("Invoice outside standard payment terms")
    
    # Check approval requirements
    if contract.requires_approval_for_invoice and not invoice.approved_by:
        errors.append("Invoice requires approval before submission")
    
    # Check duplicate invoices
    if duplicate_exists(invoice, contract.id):
        errors.append("Duplicate invoice detected")
    
    return ValidationResult(
        valid=len(errors) == 0,
        errors=errors,
        warnings=warnings
    )
```

### Contract Value Reporting
```sql
-- Contract value by status and period
SELECT 
  status,
  DATE_TRUNC('month', effective_date) as period,
  COUNT(*) as contract_count,
  SUM(contract_value) as total_value,
  AVG(contract_value) as avg_value
FROM contracts
WHERE effective_date >= DATEADD(year, -1, GETDATE())
GROUP BY status, DATE_TRUNC('month', effective_date)
ORDER BY period DESC, status;

-- Revenue at risk from expiring contracts
SELECT 
  contract_type,
  COUNT(*) as expiring_count,
  SUM(contract_value) as at_risk_value
FROM contracts
WHERE expiration_date BETWEEN GETDATE() AND DATEADD(day, 90, GETDATE())
  AND status = 'active'
  AND renewal_status != 'renewed'
GROUP BY contract_type
ORDER BY at_risk_value DESC;
```

### Risk Scoring Model
```python
def calculate_contract_risk_score(contract: Contract) -> RiskScore:
    factors = []
    
    # Financial risk
    if contract.value > 1000000:
        factors.append(("high_value", 0.3))
    
    # Term risk
    remaining_days = (contract.expiration_date - today()).days
    if remaining_days < 30:
        factors.append(("expiring_soon", 0.4))
    
    # Obligation risk
    overdue_obligations = count_overdue_obligations(contract)
    if overdue_obligations > 0:
        factors.append(("overdue_obligations", 0.2 * overdue_obligations))
    
    # SLA risk
    if contract.sla_breach_count > 0:
        factors.append(("sla_breaches", 0.15 * contract.sla_breach_count))
    
    # Amendment risk
    if contract.amendment_count > 3:
        factors.append(("frequent_amendments", 0.1))
    
    # Calculate composite score (0-100)
    raw_score = sum(weight for _, weight in factors)
    normalized_score = min(100, raw_score * 100)
    
    return RiskScore(
        overall=normalized_score,
        factors=factors,
        risk_level="high" if normalized_score > 70 else "medium" if normalized_score > 40 else "low"
    )
```
