# D-011 — Canonical repo layout

Date: 2026-06-04
Status: **locked** (ratified before Task 6).

## Context

Three committed docs proposed incompatible layouts:

- `RESEARCH_SYNTHESIS.md` — flat (`atlas-core/`, `atlas-runtime/`, `atlas-knowledge/`, `atlas-pulse/`, `atlas-native/`, `atlas-web/`, `atlas-skills/`).
- `FOUNDATION_STRATEGY.md` — `foundation/hermes-agent/`, `atlas/`, `apps/web/`, `packages/atlas-core/`, `services/wiki-runtime/`, `services/pulse-runtime/`.
- `SYSTEM_OVERVIEW.md` — `apps/web`, `apps/api`, `services/agent-runtime`, `services/wiki-runtime`, `packages/atlas-core`.

## Decision

Adopt a polyglot monorepo separating the Python runtime from web/native surfaces:

```txt
L2-ATLAS-PROJECT/
├── foundation/            # Hermes vendoring pointer (method deferred — see HERMES_FOUNDATION_PIN.md)
├── packages/
│   └── atlas-core/        # Python: Pydantic schemas, shared contracts, policy types
├── services/
│   ├── agent-runtime/     # enhanced Hermes/ATLAS runtime + event bus + audit capture
│   ├── wiki-runtime/      # LLM Wiki ingest/query/lint
│   └── pulse-runtime/     # monitors/briefings (later)
├── apps/
│   ├── api/               # mission/run/source/wiki API
│   └── web/               # cockpit WebUI (framework TBD — D-006)
├── infra/
│   └── migrations/        # SQLite schema
├── native/                # Rust sidecar/overlay (later)
├── wiki/                  # persistent markdown KB (exists)
└── docs/                  # planning/architecture/research (exists)
```

## Rationale

- Separates the Python runtime (`packages/`, `services/`) from web/native surfaces (`apps/`, `native/`), matching the real language boundary (Hermes + L2-Atlas are Python; web is JS/TS; sidecar is Rust).
- Reconciles `SYSTEM_OVERVIEW.md` and `FOUNDATION_STRATEGY.md`; supersedes the flat layout in `RESEARCH_SYNTHESIS.md` (those names survive as logical module labels, not directories).
- `infra/migrations/` and `packages/atlas-core/` match the paths already referenced in `NEXT_ACTION_PLAN.md`.

## Consequences

- Task 6 schemas land in `packages/atlas-core/atlas_core/schemas/`.
- Task 7 migration lands in `infra/migrations/0001_core.sql`.
- The flat names in `RESEARCH_SYNTHESIS.md` are deprecated as directory names; no doc rewrite required (kept as historical context).
