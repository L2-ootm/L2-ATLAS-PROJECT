# ATLAS v0.1.1 — Installer and update hardening

ATLAS `0.1.1` closes the first public-install feedback loop. It makes the
PowerShell bootstrap release-first, repairs legacy command shadowing, migrates
the retired `functions.actor_model` setting, and completes the embedded Python
dependency set with the pinned Claude Agent SDK.

## Install

```powershell
npm install --global @systemsl2/atlas
atlas install
atlas up --services gateway,cockpit
atlas doctor
```

Or use the public bootstrap. It installs Node.js LTS through winget when Node
20+ is not already available; no system Python, Git, Rust, Go, or Bun is needed:

```powershell
irm https://raw.githubusercontent.com/L2-ootm/L2-ATLAS-PROJECT/main/install/install.ps1 -OutFile $env:TEMP\atlas-install.ps1; powershell -ExecutionPolicy Bypass -File $env:TEMP\atlas-install.ps1
```

## Update from 0.1.0

```powershell
atlas update
atlas up --services gateway,cockpit
atlas doctor
```

The update installs `0.1.1` beside `0.1.0`, validates it, and switches the
active pointer. It preserves `ATLAS_HOME`, including the database, configuration,
credentials, wiki, logs, and user-created modules. `atlas rollback` returns to
the retained previous application version.

## Fixed

- repo-local and legacy source shims now delegate lifecycle commands to the npm
  launcher, so `atlas update` works from every directory;
- config schema v2 removes only the obsolete `functions.actor_model` key while
  retaining live function-model settings;
- the Windows platform package embeds the Claude Agent SDK and its transitive
  Python requirements;
- installer documentation now starts services before interpreting runtime
  `doctor` health;
- the internal `Prepare` / `Publish` / `Verify` npm release tool enforces exact
  platform-first publication.

## Boundaries

- self-contained binary distribution remains Windows x64;
- gateway and cockpit report `down` until started; optional sidecars may remain
  offline without making the installation incomplete;
- this is an open research preview and should not process sensitive production
  data until its operational boundaries fit the environment.
