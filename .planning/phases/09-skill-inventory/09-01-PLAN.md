# Phase 9 · Plan 09-01 — Skill Inventory & Classification

**Type:** doc / classification only (no service code, no UI, no registry runtime)
**Requirements:** SKILLS-01, SKILLS-02, SKILLS-03, SKILLS-04
**Primary artifact:** `docs/imports/SKILL_INVENTORY.md`

## Goal

Produce a complete, classified skill inventory so ATLAS can ship a curated default pack
rather than the undifferentiated union of every reachable skill, and so anything that cannot
ship publicly is identified with a reason and an action.

## Scope guard (What NOT to do — from CONTEXT.md)

- No new skills, no skill UI/marketplace, no auto-discovery service, no skill-registry
  runtime, no versioning infrastructure.
- No new classes beyond the fixed taxonomy.
- Do not copy private/L2 skill content into the repo; do not emit absolute local paths.
- Do not touch the parked `feat/integrations-module-catalog` work.

## Tasks

1. **Continuation-truth gate** — verify git status, STATE, ROADMAP, and Phase 9 CONTEXT.md
   before edits.
2. **Source grounding** — enumerate all skill source groups (vendored Hermes default +
   optional, GSD framework, OpenClaw design/meta, L2 packs) by direct inspection; map the
   Hermes catalog and pull frontmatter for Core/Operator candidates.
3. **Classify** — assign each skill/family the taxonomy class + the SKILLS-01 fields; build
   Core/Operator/L2 pack metadata (SKILLS-02/03/04).
4. **Public-safety pass** — identify release blockers and the redaction/polish queue.
5. **Write artifact** — `docs/imports/SKILL_INVENTORY.md` with the 10 required sections.
6. **Verify** — risky-string scan of the artifact; confirm pack members resolve to existing
   vendored SKILL.md files; write VERIFICATION.md.
7. **Handoff** — update STATE.md + ROADMAP.md; write phase SUMMARY.

## Acceptance criteria (goal-backward)

- `docs/imports/SKILL_INVENTORY.md` exists with all 10 sections and every required field.
- Core Pack (public_safe: true, requires_secrets: none) and Operator Pack (public_safe: true)
  carry full metadata; L2 Pack classified l2-internal/personal-private with public_safe: false.
- No personal/private absolute paths in the artifact (criterion 5).
- Release blockers (incl. `godmode` in the default tree) documented with actions.
- Repo stays in a clean, doc-only state.
