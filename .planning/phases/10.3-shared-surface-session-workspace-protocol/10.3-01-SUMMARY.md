# Plan 10.3-01 Summary — SurfaceSession spine

## What shipped

- **Frozen models** (`packages/atlas-core/atlas_core/schemas/surface_session.py`):
  `SurfaceSession` (composes the existing `SurfaceIdentity`/`WorkspaceIdentity`/`ModelIdentity`,
  copies `core.Run` id-gen + datetime `field_serializer`), `SurfaceEvent`, `SessionState`,
  `EventKind`. No `cancelled` session state by design.
- **Migration** `infra/migrations/0016_surface_sessions.sql`: `surface_sessions` table mirroring
  the model 1:1, soft session→run link (no FK), and a `BEFORE UPDATE` terminal-immutability
  trigger (`completed/failed/reclaimed` → `RAISE(ABORT)`).
- **Audit literals** appended to `AuditEvent.event_type` (TEXT column, no migration): the six
  planned (`surface_session_started/suspended/resumed/reclaimed`, `run_cancelled`,
  `permission_transition`) plus `surface_session_completed`/`surface_session_failed` so every
  terminal transition audits with a precise type.
- **Service** `services/agent-runtime/atlas_runtime/surface_session_service.py`:
  `create_session/get_session/list_sessions/transition_session`, mirroring `run_service`'s
  `with lock: with conn:` SELECT-precondition write + emit-after-lock. `_ALLOWED_FROM` pins the
  state machine (`cancelling→{completed,failed}`; terminals have empty outgoing sets).
- **conftest** `surface_session` fixture mirroring `run_id`.

## Key decisions / deviations

- **Operator sentinel run, seeded lazily.** `audit_events.run_id` is `NOT NULL REFERENCES
  runs(id)`, but a surface session can exist before any run. Rather than a global migration seed
  (which would pollute every `db`-fixture test's missions/runs and risk breaking suites),
  `create_session`/`transition_session` `INSERT OR IGNORE` an `operator` mission+run **only when**
  a lifecycle event is run-less — consistent with the `DEFAULT 'operator'` precedent in
  `tool_approvals`/`discord_approvals`. Real run-bound sessions never touch it.
- **Two extra audit literals** (`surface_session_completed`/`_failed`) beyond the planned six —
  additive, TEXT column, no migration; strengthens AUD-01. The parametrized literal test asserts
  acceptance (not an exact set), so this is non-breaking.
