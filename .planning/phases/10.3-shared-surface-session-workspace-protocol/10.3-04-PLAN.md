<objective>
Close the orphan-and-cancellation gap. Thread a cooperative `threading.Event` cancel token through the
executor and the native agent's watchdog so cancellation takes effect at the next checkpoint and emits a
terminal audited `cancelled` outcome (CPython cannot hard-kill a thread). Add owner-token + heartbeat-TTL
liveness to the session, and a startup reconciliation sweep that guarantees no unowned running execution
survives a restart.

Purpose: SURF-06 (cancellation propagates and emits a terminal audited outcome) + SURF-05
(disconnect/reconnect/restart preserves identity and leaves no unowned running execution) + AUD-01 +
AGNT-01 (reuse the existing executor/agent — no new runtime).
Output: `run_executor.py` + `native.py` edits (cancel token), `surface_session_service.py` extension
(heartbeat + `reconcile_orphans`), `cli/main.py` startup-sweep wiring, and the `test_cancellation.py` +
`test_surface_session_lifecycle.py` suites.

Depends on plan 01 for the session service + state machine. Runs in Wave 2.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cancel request → running execution | A cancel signal crosses into an in-flight run/agent thread |
| process restart → persisted session/run state | A crashed owner's authority over running rows crosses a restart boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.3-ORPHAN | Elevation of Privilege / Repudiation | active session/run with dead owner after crash | mitigate | startup `reconcile_orphans` reads DB state (not the in-process `_active_threads` dict), marks unheld sessions reclaimed, cancels their runs + crash-left running runs, all audited (SURF-05) |
| T-10.3-ORPHAN-FALSE | Denial of Service | reclamation of live work | mitigate | owner-token + heartbeat-TTL (3× interval); fresh-heartbeat sessions are excluded; PID-reuse-safe via owner_token (never `os.kill`, Pitfall 2) |
| T-10.3-CANCEL-DOS | Denial of Service | cancellation ignored → runaway run | mitigate | cooperative cancel token polled at watchdog checkpoints + existing max-runtime watchdog as backstop; subprocesses killable via terminate-in-poll |
| T-10.3-CANCEL-HARDKILL | Tampering | corrupting session state via force-kill | mitigate | cooperative checkpoints only (no CPython thread kill); cancel-wins invariant preserved so terminal state is consistent |
| T-10.3-RECLAIM-LEAK | Information Disclosure | reclamation audit payload | mitigate | reclaimed/cancel audit routed through `emit` (SECRET_PATTERNS redaction) |
| T-10.3-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages — stdlib owner-token/TTL design chosen over optional psutil (Open Question 1); zero install tasks |
</threat_model>
