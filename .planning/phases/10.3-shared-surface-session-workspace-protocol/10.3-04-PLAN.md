<objective>
Close the orphan-and-cancellation gap. Thread a cooperative `threading.Event` cancel token through the
executor, the native agent's watchdog, AND the ATLAS-owned tool-invocation gate (`tool_service.invoke`)
so cancellation takes effect at the next observable checkpoint and emits a terminal audited `cancelled`
outcome (CPython cannot hard-kill a thread). Add owner-token + heartbeat-TTL liveness to the session, and
a startup reconciliation sweep that guarantees no unowned running execution survives a restart.

SURF-06 cancellation granularity (bounded scope, stated honestly): cancellation is COOPERATIVE and takes
effect at three ATLAS-owned checkpoints — (a) the executor/native watchdog poll-join between turns, (b)
the `tool_service.invoke` tool gate (a tool/subprocess does not START once the token is set), (c) a
spawned subprocess is terminated in a poll-loop that checks the token. The single opaque foundation
`run_conversation(prompt)` model call cannot be interrupted MID-CALL (D-001 forbids editing foundation
internals; no per-turn hook is exposed — RESEARCH OQ-2 RESOLVED: between-turn + tool-gate + subprocess
granularity only). The max-runtime watchdog remains the hard backstop.

Purpose: SURF-06 (cancellation propagates to model stream between turns, the tool gate, and subprocesses,
emitting a terminal audited outcome) + SURF-05 (process restart leaves no unowned running execution) +
AUD-01 + AGNT-01 (reuse the existing executor/agent/tool chokepoint — no new runtime).
Output: `run_executor.py` + `native.py` + `tool_service.py` edits (cancel token), `surface_session_service.py`
extension (heartbeat + `reconcile_orphans`), `cli/main.py` startup-sweep wiring, and the
`test_cancellation.py` + `test_surface_session_lifecycle.py` suites.

Depends on plan 01 for the session service + state machine. Runs in Wave 2. (Resume after
disconnect/restart is plan 05.)
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cancel request → running execution | A cancel signal crosses into an in-flight run/agent thread and the tool gate |
| process restart → persisted session/run state | A crashed owner's authority over running rows crosses a restart boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.3-ORPHAN | Elevation of Privilege / Repudiation | active session/run with dead owner after crash | mitigate | startup `reconcile_orphans` reads DB state (not the in-process `_active_threads` dict), marks unheld sessions reclaimed, cancels their runs + crash-left running runs, all audited (SURF-05) |
| T-10.3-ORPHAN-FALSE | Denial of Service | reclamation of live work | mitigate | owner-token + heartbeat-TTL (3× interval); fresh-heartbeat sessions are excluded; PID-reuse-safe via owner_token (never `os.kill`, Pitfall 2) |
| T-10.3-CANCEL-DOS | Denial of Service | cancellation ignored → runaway run/tool/subprocess | mitigate | cooperative cancel token observed at THREE ATLAS-owned checkpoints (watchdog poll-join, `tool_service.invoke` gate, subprocess poll-terminate) + max-runtime watchdog backstop; bounded granularity stated honestly (no mid-call model interruption — OQ-2 resolved) |
| T-10.3-CANCEL-TOOLGATE | Elevation of Privilege | a tool/subprocess starting AFTER cancel requested | mitigate | `tool_service.invoke` short-circuits before adapter execution when the token is set; a cancelled run cannot start new side-effecting tools (Pattern 7c) |
| T-10.3-CANCEL-HARDKILL | Tampering | corrupting session state via force-kill | mitigate | cooperative checkpoints only (no CPython thread kill); cancel-wins invariant preserved so terminal state is consistent |
| T-10.3-RECLAIM-LEAK | Information Disclosure | reclamation audit payload | mitigate | reclaimed/cancel audit routed through `emit` (SECRET_PATTERNS redaction) |
| T-10.3-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages — stdlib owner-token/TTL design chosen over optional psutil (Open Question 1); zero install tasks |
