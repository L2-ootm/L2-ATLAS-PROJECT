## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| reconnect request → suspended session | A new owner claims a previously-suspended session and rebuilds its authority |
| resume → immutable contract snapshot | Resume reads (never mutates) the frozen 10.2 run-contract record |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.3-RESUME-DRIFT | Tampering / Elevation | resuming with a drifted prompt/tool/context version | mitigate | `replay_contract` asserts the stored prompt/tool-catalog/context-policy versions; a mismatch raises ContractCompatibilityError → fail closed, no resume (authority cannot silently widen across a version change) |
| T-10.3-RESUME-ORPHAN | Elevation of Privilege | a resumed session left without a live owner | mitigate | resume mints a fresh owner_token + refreshes heartbeat so the plan-04 sweep treats it as owned-alive; a resume that fails leaves the session suspended (reclaimable) — no unowned active session |
| T-10.3-RESUME-STATE | Tampering | resuming a terminal/illegal-state session | mitigate | `_ALLOWED_FROM` guard (plan 01) rejects resume from a non-resumable state with ValueError |
| T-10.3-RESUME-LEAK | Information Disclosure | resume audit payload | mitigate | `surface_session_resumed` emitted via `emit` (SECRET_PATTERNS redaction); no snapshot body re-persisted |
| T-10.3-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages — reuses existing agent_contract_service + stdlib |
</threat_model>
