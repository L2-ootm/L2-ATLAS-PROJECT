Implement a single shared `global_root() -> pathlib.Path` deriving from `pathlib.Path.home() / ".atlas"` (ATLAS_HOME-aware, matching db.DEFAULT_DB_PATH resolution) so the workspace root and DB home agree (Pitfall 4). Implement `resolve_workspace(conn, *, kind, project_id=None) -> str`: for `global` return `global_root()` resolved; for `project` call `project_service.get_project(conn, project_id)` and raise `WorkspaceError("unregistered")` on None, else use `Project.root_path`. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| surface request → path resolver | Untrusted target paths and project ids cross into filesystem authority |
| resolver denial → audit bus | Cross-project denial crosses into the persisted ledger |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.3-PATH | Elevation of Privilege / Tampering | assert_contained `..` traversal | mitigate | `.resolve()` + `os.path.commonpath` containment maps any non-containment to typed `traversal`; string-prefix matching explicitly NOT used |
| T-10.3-PATH-SYM | Elevation of Privilege | symlink escaping the workspace root | mitigate | resolve both root and target with `.resolve()` BEFORE commonpath (collapses symlinks); typed `symlink_escape` via pre/post-resolve comparison (Pitfall 5) |
| T-10.3-PATH-STALE | Tampering | moved/deleted registered project root | mitigate | fail closed with `stale_root`, NO auto-reregister/auto-mutation (SURF-03) |
| T-10.3-PATH-DRIVE | Denial of Service / error leak | mixed-drive commonpath ValueError | mitigate | wrap commonpath in try/except ValueError → typed error; raw traceback never escapes (Pitfall 1) |
| T-10.3-XPROJ | Elevation of Privilege | undeclared cross-project write | mitigate | `assert_write_allowed` denies a contained-but-different-project target with the distinct typed reason `cross_project` (not mislabeled traversal) and audits the denial (SURF-03 / AUD-01) |
| T-10.3-PATHLEAK | Information Disclosure | denial audit payload | mitigate | denial data routed through `emit` (SECRET_PATTERNS redaction); typed errors do not leak paths beyond the contained root |
| T-10.3-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed — stdlib `os.path`/`pathlib` only |
</threat_model>
