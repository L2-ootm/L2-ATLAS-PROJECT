# Phase 10.3 Verification — Shared Surface Session & Workspace Protocol

**Verdict:** Passed — all 8 phase requirement IDs and all four ROADMAP success criteria are
delivered with automated tests, on the existing agent runtime (AGNT-01), with no new event bus,
no new snapshot format, and no new dependency.

## Goal-backward: ROADMAP success criteria

1. **Session records full identity + lifecycle** — `SurfaceSession` (frozen, composes
   `SurfaceIdentity`/`WorkspaceIdentity`/`ModelIdentity`) + `surface_sessions` table (migration
   0016, terminal-immutable) + `create_session`/`transition_session` state machine. Records
   surface kind/instance, workspace kind, project/root, mission/run/session IDs, agent/model,
   permission mode, prompt/tool-catalog/context-policy versions, and lifecycle state.
   **Verified:** `test_surface_session_service.py` (16) — persistence of every SURF-01 field,
   guarded transitions, terminal immutability.
2. **Global/project workspace resolution + path safety** — `workspace_service.resolve_workspace`
   (global bound to db home; project via the existing registry) + `assert_contained` /
   `assert_write_allowed` fail-closed with typed reasons traversal / symlink_escape / stale_root /
   unregistered / cross_project (mixed-drive ValueError mapped). **Verified:**
   `test_workspace_service.py` (9).
3. **One normalized agent event stream** — `surface_events.normalize_surface_events` projects
   the audited ledger (`get_events_for_session`, per-session ordered) + terminal `RunOutcome` into
   `SurfaceEvent` with a discriminated kind covering text/reasoning/tool_call/tool_result/task/
   retry/retrieval/approval/error/completion and a monotonic per-session seq + `replay_since`.
   Pure read-projection, no new bus (AGNT-01). **Scope note (per locked CONTEXT decision):** 10.3
   delivers the shared contract + normalizer + conformance tests; TUI/WebUI *rendering* of this
   stream is wired in 10.6/10.7. **Verified:** `test_surface_events.py` (30) incl. contiguous seq
   across two runs of one session and the no-writes purity gate.
4. **Cancel / disconnect / reconnect / resume / restart with no orphan** — cooperative
   `cancel_token` (executor + native watchdog poll-join + tool gate) → terminal audited
   `cancelled`; `heartbeat` + `reconcile_orphans` startup sweep (stdlib owner-token/TTL, no
   os.kill) reclaims orphaned sessions and crash-left running runs; `resume_session` re-binds
   ownership and rebuilds context via the immutable 10.2 `RunContractSnapshot` replay (fail closed
   on version drift). **Verified:** `test_cancellation.py` (5), `test_surface_session_lifecycle.py`
   (5), `test_surface_session_resume.py` (5).
