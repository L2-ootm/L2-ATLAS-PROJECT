# Phase 10.3: Shared Surface Session & Workspace Protocol - Context

**Gathered:** 2026-06-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Deliver one ATLAS session contract that TUI, WebUI, CLI, API, and future native clients share
over the **existing** agent runtime (AGNT-01 â€” no donor-specific runtime/provider/tool-executor/
memory backend). A surface session binds a surface (kind/instance) and a workspace (ATLAS global
or a registered Project root) to mission/run/session identity, agent/model, permission mode, and
the frozen prompt/context/tool-catalog versions from Phase 10.2, with an explicit, audited
lifecycle.

In scope: the persisted `SurfaceSession` entity and lifecycle state machine; canonical workspace
resolution and fail-closed path safety; a normalized `SurfaceEvent` contract + normalizer +
conformance tests; orphan-free disconnect/reconnect/resume/restart; cooperative cancellation.

Out of scope (deferred to owning phases): actual TUI rendering (10.6) and WebUI rendering/queue
UX (10.7); the global config control plane (10.4); the permission broker routing approvals to
surfaces (10.5). Phase 10.3 defines the contract those phases consume.

</domain>

<decisions>
## Implementation Decisions

### Surface-Session Persistence & Identity (SURF-01)
- New **frozen `SurfaceSession`** Pydantic v2 model in `packages/atlas-core/atlas_core/schemas/`,
  composing the existing frozen `SurfaceIdentity` and `WorkspaceIdentity` (do not duplicate fields).
- New **`surface_sessions`** table via migration `infra/migrations/0016_surface_sessions.sql`;
  all writes go through a service layer using the Pydantic-first â†’ validate â†’ SQL-write pattern.
- A surface session is **long-lived and owns 0..N runs** (a session is a client connection;
  runs are executions within it). Records surface kind/instance, workspace kind, project/root,
  mission/run/session IDs, agent/model, permission mode, prompt/context/tool-catalog versions,
  and lifecycle state.
- **Reuse the existing `core.py` id-generation scheme** for session IDs (match Mission/Run).

### Workspace Resolution & Path Safety (SURF-02, SURF-03)
- New **`workspace_service`** (resolver) wrapping `project_service._resolve_dir()`; the ATLAS
  **global workspace root is `~/.atlas` (ATLAS_HOME-aware)**, project sessions resolve through the
  existing Project registry/model.
- **Fail-closed canonical validation**: after `.resolve()`, enforce containment with
  `os.path.commonpath`; raise **typed errors** â€” `traversal`, `symlink_escape`, `stale_root`,
  `unregistered`. No silent widening.
- **Stale root** (registered project dir moved/deleted) â†’ fail closed with `stale_root`; **no
  auto-mutation/auto-reregister**.
- **Undeclared cross-project writes are blocked** (SURF-03) and the denial is audited.

### Normalized Event Stream (SURF-04)
- New **frozen `SurfaceEvent`** projection derived from `AuditEvent` + `RunOutcome`, with a stable
  **discriminated `kind`**: text, reasoning, tool_call, tool_result, task/subagent, retry,
  retrieval, approval, error, completion.
- **One normalizer function** is the single source; **WebUI consumes via gateway SSE**, **TUI via
  an in-process iterator** â€” same normalized sequence, two transports (AGNT-01).
- **Phase 10.3 scope:** deliver the `SurfaceEvent` contract + normalizer + conformance tests.
  TUI/WebUI rendering is wired in 10.6/10.7.
- Events carry a **monotonic per-session sequence number** for reconnect gap-detection/ordering.

### Lifecycle & Orphan Prevention (SURF-05, SURF-06)
- Explicit **session state machine**: starting â†’ active â†’ suspended(disconnected) â†’ resuming â†’
  cancelling â†’ completed/failed, plus reclaimed; transitions are tested.
- **Orphan reclamation:** owner-PID/heartbeat on the session; a **startup reconciliation sweep**
  marks unheld running sessions interrupted and cancels/fails their runs â€” audited. No unowned
  running execution may persist.
- **Cancellation (SURF-06):** a **cooperative cancel token** threaded into the run/session context
  that the existing executor, model stream, active tools, subprocesses, and child agents observe;
  emits a terminal **audited `cancelled` outcome** (not a hard kill).
- **Resume after disconnect/compaction/restart:** rebuild session identity and context by reusing
  the Phase 10.2 `RunContractSnapshot` replay/resume invariants.

### Audit (AUD-01)
- Every surface session, model call, retrieval, tool call, permission transition, lifecycle
  change, and cancellation emits through the existing `audit_service.emit()` bus with redaction;
  reuse `AuditEvent`/`ToolCall`. Add new `event_type` literals as needed (TEXT column, no schema
  migration for the enum).

</decisions>

  tools, results, tasks/subagents, retries, retrieval, approvals, errors, completion.
