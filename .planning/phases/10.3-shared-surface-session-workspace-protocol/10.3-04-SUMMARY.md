# Plan 10.3-04 Summary — cancellation + orphan reconciliation

## What shipped

**Task 1 — cooperative cancel token (watchdog):**
- `AgentRuntime.execute` (ABC), `native.execute`, `claude_code.execute`, and the executor
  (`execute_run`/`start_and_execute_async`) now accept `cancel_token: threading.Event | None`.
- `native.execute` replaces the single `worker.join(max_runtime_s)` with a cancel-aware
  poll-join: a set token emits `run_cancelled` and returns
  `RunOutcome(status="failed", stop_reason="cancelled")`. No-token behavior is byte-identical
  (max-runtime watchdog still governs). Cancel-wins invariant preserved.

**Task 2 — tool-gate short-circuit:**
- `tool_service.invoke` reads `ctx["cancel_token"]` (public signature unchanged, D-013-safe);
  a set token short-circuits BEFORE adapter execution, emits a `tool_failed` cancellation
  audit, and returns `ToolResult(ok=False, error="cancelled")` — a tool/subprocess never
  starts after cancel. Subprocess `proc.terminate()/kill()` poll contract documented for adapters.

**Task 3 — heartbeat + reconciliation:**
- `surface_session_service.heartbeat()` refreshes `heartbeat_at` for an active session.
- `surface_session_service.reconcile_orphans(*, ttl_seconds)` reads DB state (not the
  restart-lost `_active_threads` dict): reclaims non-terminal sessions with null/stale
  heartbeat, cancels their running runs, and cancels crash-left running runs whose session is
  absent/reclaimed — all audited (`surface_session_reclaimed`). Idempotent; never reclaims
  fresh-heartbeat work. Stdlib owner-token/TTL — no `os.kill`.
- Wired into `cli/main.py`: `atlas runtime serve` runs the sweep at startup; `atlas runtime
  reconcile [--ttl]` exposes it directly. TTL default 90s (3× a 30s heartbeat interval).

## Decisions

- **`_ALLOWED_FROM` extended** so `reclaimed` is reachable from `active`/`resuming` (was only
  from `suspended`) — reconciliation must reclaim an orphan regardless of the state it died in.
  No plan-01 test asserted those edges were forbidden, so non-breaking.
- **Cancellation granularity stated honestly (RESEARCH OQ-2 resolved):** between-turn watchdog
  + tool-gate + subprocess only. The single opaque foundation `run_conversation` call cannot be
  interrupted mid-call (D-001 forbids foundation edits); the max-runtime watchdog is the backstop.
