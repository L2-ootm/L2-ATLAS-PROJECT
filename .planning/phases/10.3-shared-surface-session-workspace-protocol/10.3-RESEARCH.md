# Phase 10.3: Shared Surface Session & Workspace Protocol - Research

**Researched:** 2026-06-25
**Domain:** Session/workspace contract layer over an existing SQLite-backed Python agent runtime (Pydantic v2 frozen contracts, cooperative cancellation, cross-platform process liveness, fail-closed path containment)
**Confidence:** HIGH (almost entirely codebase-internal; the two external-knowledge findings — Windows `os.kill`/`commonpath` semantics — were verified live on this machine)

## Summary

Phase 10.3 is overwhelmingly an **internal-pattern-replication phase**, not an ecosystem-discovery phase. Every required capability has a directly analogous, already-shipped implementation in `services/agent-runtime/atlas_runtime/`: frozen Pydantic v2 contracts (`agent_contract.py`, `core.py`), the Pydantic-first → validate → atomic SQL-write service pattern (`run_service.py`, `project_service.py`, `audit_service.py`), immutable SQL-trigger snapshots (`0015_agent_contract_snapshots.sql`), the background-daemon executor (`run_executor.py`), and SSE event delivery (`api_server.py`). The plan should reuse these patterns verbatim and introduce **no new architecture** — this also satisfies AGNT-01 (no second runtime) and D-022 (gateway stays dispatch-only).

There are exactly **three areas where careful technical judgment is required** beyond pattern-copying, and all three have Windows-specific traps that this research resolved with live verification:

1. **Cooperative cancellation (SURF-06).** The current `NativeAtlasAgent.execute()` runs `agent.run_conversation(prompt)` synchronously on a daemon thread with only a join-timeout watchdog — there is **no cancel hook today**. A `threading.Event`-based cancel token must be threaded into the run/session context and polled at observable boundaries (executor loop, between model turns, tool-invocation gate, subprocess wait). A hard thread kill is impossible in CPython and is explicitly out of scope; cancellation is checkpoint-cooperative and emits a terminal audited `cancelled` outcome.

2. **Owner-PID / heartbeat liveness (SURF-05).** The POSIX idiom `os.kill(pid, 0)` is **broken on Windows**: a dead PID raises `EINVAL` (errno 22), not `ESRCH`, so the standard liveness check silently misclassifies. Verified live. Use `psutil.pid_exists(pid)` **plus a stored `create_time` guard** to defeat PID reuse, or a stdlib-only fallback. The startup reconciliation sweep marks unheld `active` sessions interrupted and cancels their runs.

3. **Fail-closed path containment (SURF-02/03).** `os.path.commonpath` raises `ValueError` on mixed Windows drives (verified) — which is a useful fail-closed signal but must be caught and mapped to a typed error, not allowed to escape as a raw `ValueError`. Symlink escape requires `.resolve()` (already used by `_resolve_dir`) **before** the `commonpath` check, and the stale-root case must fail closed without auto-reregister.

**Primary recommendation:** Model 10.3 as ~5 work packages, each mirroring an existing service: (1) `SurfaceSession` frozen model + migration `0016` + `surface_session_service`; (2) `workspace_service` containment resolver with typed errors; (3) `SurfaceEvent` frozen projection + single normalizer over `AuditEvent`+`RunOutcome`; (4) cooperative cancel token threaded through executor/agent; (5) heartbeat + startup reconciliation sweep. Reuse `RunContractSnapshot` for resume. Prefer **stdlib-only** for liveness if avoiding a new dependency is valued (decision flagged below); `psutil` is already installed and is the more robust option.

## Project Constraints (from CLAUDE.md / STATE.md)

No `./CLAUDE.md` exists at the project root (verified — `claude_md_path: "./CLAUDE.md"` in config but file absent). Constraints are sourced from STATE.md decisions and the v1.1 roadmap:

- **D-013 (JSON-stable contracts):** All public model fields are `str`/`tuple`/number/`bool`/nested frozen models only. **No `dict[str, Any]`, no `pathlib.Path`** in public schema. JSON payloads (audit `data`, snapshot bodies) are stored as JSON **strings**, not dicts. `datetime` fields use a `field_serializer` to emit ISO-8601 strings. The `SurfaceSession` and `SurfaceEvent` models MUST follow this exactly.
- **D-022 (gateway dispatch-only / Rust-first cementation):** The Rust gateway reads SQLite directly and writes via the `atlas` CLI contract; SSE is rowid-poll. The gateway **must not author session/event state** — it forwards the normalizer output. All session/workspace authority lives in Python/SQLite.
- **AGNT-01:** No second `AgentRuntime`, provider layer, tool executor, or memory backend. Reuse the existing `native` runtime and `audit_service` bus.
- **Authority monotonicity:** Capabilities/paths may **narrow but never silently widen** (mirrors the existing `narrow_capabilities` invariant in `tool_catalog.py`). Workspace containment is the path-level expression of this rule.
- **Deterministic, network-free assembly:** Session/context assembly must not touch the network (mirrors Phase 10.2 `prepare_run_contract`).
- **Migrations are sequential, additive, idempotent:** `infra/migrations/NNNN_*.sql`, `CREATE ... IF NOT EXISTS`. Next file is **`0016`**. The `db.py` runner is `executescript`-based and drift-tolerant — trigger bodies (`BEGIN ... END`) must not be `;`-split (this is why 0015's triggers work).

### Pattern 1: Frozen model + 1:1 migration + Pydantic-first service write
**What:** Define `SurfaceSession` as a frozen Pydantic v2 model; mirror its fields 1:1 in `0016_surface_sessions.sql`; all writes construct+validate the model **before** any SQL, inside `with lock: with conn:`.
**When to use:** SURF-01 persistence.
**Example (model — follows `core.py` exactly):**
```python
# Source: packages/atlas-core/atlas_core/schemas/core.py (Run) + agent_contract.py (_FrozenContract)
class SurfaceSession(BaseModel):
    model_config = ConfigDict(frozen=True, str_strip_whitespace=True)
    id: str = Field(default_factory=lambda: str(uuid.uuid4()))     # reuse core id-gen
    surface: SurfaceIdentity            # composed, not duplicated
    workspace: WorkspaceIdentity        # composed, not duplicated
    agent: str
    model: ModelIdentity
    permission_mode: PermissionMode
    prompt_version: str
    tool_catalog_version: str
    context_policy_version: str
    state: SessionState = "starting"
    owner_token: str = ""               # PID-reuse-safe ownership (recommended)
    owner_pid: Optional[int] = None
    heartbeat_at: Optional[datetime.datetime] = None
    created_at: datetime.datetime = Field(default_factory=lambda: datetime.datetime.now(datetime.timezone.utc))
    updated_at: datetime.datetime = Field(default_factory=lambda: datetime.datetime.now(datetime.timezone.utc))
    @field_serializer("created_at", "updated_at", "heartbeat_at")
    def _ser_dt(self, dt): return None if dt is None else dt.isoformat()
```
**Service write (mirror `run_service.start_run`):** SELECT precondition + INSERT inside one `with lock: with conn:`, then `emit()` AFTER the lock releases (emit re-acquires the lock internally — emitting inside the block deadlocks).

### Pattern 2: Fail-closed path containment with typed errors
**What:** Resolve with `.expanduser().resolve()` (already in `_resolve_dir`), then enforce `os.path.commonpath([root_resolved, target_resolved]) == root_resolved`. Map every failure mode to a typed error, never leak a raw `ValueError`.
**When to use:** SURF-02/03.
**Example:**
```python
# Source: extends services/agent-runtime/atlas_runtime/project_service.py _resolve_dir
class WorkspaceError(ValueError): ...
# reason ∈ {"traversal","symlink_escape","stale_root","unregistered"}

def assert_contained(root: str, target: str) -> str:
    root_p = pathlib.Path(root).expanduser().resolve()
    if not root_p.exists():
        raise WorkspaceError("stale_root")        # registered dir moved/deleted; NO auto-reregister
    target_p = pathlib.Path(target).expanduser().resolve()  # .resolve() collapses symlinks
    try:
        common = pathlib.Path(os.path.commonpath([str(root_p), str(target_p)]))
    except ValueError:
        raise WorkspaceError("traversal")         # mixed Windows drives raise ValueError → fail closed
    if common != root_p:
        # distinguish symlink escape from plain traversal by comparing pre/post-resolve
        raise WorkspaceError("symlink_escape" if _had_symlink(target) else "traversal")
    return str(target_p)
```
Project sessions resolve `root` through `project_service.get_project(...).root_path`; a missing project → `unregistered`. Global sessions use `ATLAS_HOME or ~/.atlas` (match `db.py` `DEFAULT_DB_PATH` resolution so the workspace root and DB home agree).

### Pattern 3: Frozen `SurfaceEvent` discriminated projection
**What:** A frozen model with a discriminated `kind` Literal covering SURF-04's full enumeration, plus a monotonic per-session `seq`. JSON-stable per D-013 (payload is a JSON string, not a dict).
**Example:**
```python
EventKind = Literal[
    "text","reasoning","tool_call","tool_result","task",   # task == subagent
    "retry","retrieval","approval","error","completion",
]
class SurfaceEvent(_FrozenContract):     # reuse agent_contract._FrozenContract (extra="forbid")
    session_id: str
    seq: int                              # monotonic per session; reconnect gap-detect
    kind: EventKind
    run_id: str | None = None
    occurred_at: str                      # ISO-8601 str (D-013)
    payload_json: str = "{}"              # JSON string, not dict (D-013)
```

### Pattern 4: One normalizer, two transports
**What:** A single pure function `normalize_surface_events(audit_events, run_outcome) -> tuple[SurfaceEvent, ...]` is the only place `AuditEvent.event_type` maps to `SurfaceEvent.kind`. WebUI consumes via gateway SSE (rowid-poll, D-022); TUI consumes via an in-process iterator over the same function output. The `seq` is assigned in normalizer order so both transports see identical numbering.
**event_type → kind mapping (derive from existing `AuditEvent.event_type` Literals):**
```
llm_call            → text / reasoning   (split on data payload shape)
tool_call / tool_requested → tool_call
tool_completed      → tool_result
tool_failed         → error (or tool_result with ok=false)
subagent_run        → task
approval            → approval
failure             → error
memory_change / artifact / wiki_update → retrieval/activity as applicable
RunOutcome (terminal) → completion        (carries status/summary/claims)
```
**Reconnect:** the client sends `last_seq`; the normalizer/transport replays rows with `seq > last_seq` (audit rows are immutable and time-ordered — `get_events_for_run` already returns `ORDER BY timestamp ASC`).

### Pattern 5: Owner-token + heartbeat liveness (RECOMMENDED, stdlib-only)
**What:** Each session row stores `owner_token` (uuid generated by the owning process), `owner_pid`, and `heartbeat_at`. The owning process refreshes `heartbeat_at` on a timer. A session is "owned-alive" iff `heartbeat_at` is within a TTL (e.g., 3× the heartbeat interval). This is fully deterministic and testable without process introspection, sidesteps the Windows `os.kill` trap entirely, and is PID-reuse-safe (a reused PID won't carry the old `owner_token`).
**Optional hardening:** combine with `psutil.pid_exists(owner_pid)` + stored `create_time` for immediate (non-TTL-delayed) detection. Only add psutil if the TTL latency is unacceptable.
**Startup reconciliation sweep (SURF-05):**
```python

        ...
```

No `active` session may survive a restart without a live heartbeat owner.

### Pattern 6: Session state machine (mirror `run_service` transition guards)
**What:** Enforce `starting → active → suspended → resuming → cancelling → completed|failed` plus `reclaimed`, each transition guarded by a SELECT-status precondition inside the lock (exactly like `complete_run`'s `if row[0] != "running": raise ValueError`). Terminal states have no outgoing transitions. Every transition emits an audit event.

### Pattern 7: Cooperative cancel token (SURF-06)
**What:** A `threading.Event` (`cancel_token`) created per run/session, passed into `execute_run` and the agent. Checkpoint polling at: (a) the executor loop, (b) between model turns inside the agent driver, (c) the tool-invocation gate (before each tool runs), (d) `subprocess` waits (use `proc.poll()` loops checking the token, then `terminate()`), (e) child-agent spawn points. On set: stop at the next checkpoint, emit `event_type` cancellation audit, call `cancel_run`, return `RunOutcome(status="failed", stop_reason="cancelled")` (or a dedicated cancelled outcome). **Not a hard kill** — CPython cannot force-terminate a thread.
**Threading point:** `run_executor.start_and_execute_async` currently builds the worker closure (`_worker`) and `NativeAtlasAgent.execute` runs `agent.run_conversation(prompt)` on its own inner daemon thread (`_drive`) joined with a watchdog timeout. The cancel token must be checked in the watchdog join loop (poll-join in small intervals instead of one long `join(max_runtime_s)`) and forwarded to the agent so cooperative agents can break between turns.

### Pattern 8: Audit `event_type` extension without schema migration
**What:** AUD-01 needs new event types (session lifecycle, cancellation, permission transition). The `audit_events.event_type` column is **TEXT**; the enum is enforced by the Pydantic `AuditEvent.event_type` Literal, not by SQL. Add new literals (e.g., `surface_session_started`, `surface_session_reclaimed`, `run_cancelled`, `permission_transition`) to the Literal in `core.py` — **no migration for the enum**. Note `emit()`'s signature types `event_type: str`, so it forwards anything the model accepts.

### Anti-Patterns to Avoid
- **New event bus / pub-sub for SurfaceEvent.** Violates AGNT-01. The normalizer is a pure read-projection over `audit_events` rows.
- **Authoring session/event state in the Rust gateway.** Violates D-022. Gateway forwards normalizer output only.
- **`dict[str, Any]` / `pathlib.Path` in any public model field.** Violates D-013. Use JSON strings and `str` paths with `field_serializer` for datetimes.
- **Hard thread kill for cancellation.** Impossible in CPython; out of scope. Cancellation is cooperative.
- **`os.kill(pid, 0)` for liveness.** Broken on Windows (Pitfall 2).
- **Auto-reregister a moved project root.** SURF-03 requires fail-closed `stale_root`; no auto-mutation.
- **`;`-splitting migration SQL with trigger bodies.** `db.py` uses `executescript`; keep `BEGIN ... END` intact (0015 is the template).

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Cross-platform process liveness | Custom `os.kill`/signal probing | `psutil.pid_exists()`+`create_time()` **or** owner-token+heartbeat-TTL | `os.kill(pid,0)` raises `EINVAL` not `ESRCH` on Windows (verified) — the POSIX idiom silently misclassifies dead PIDs |
| Path containment | String prefix matching (`target.startswith(root)`) | `.resolve()` + `os.path.commonpath` with typed-error mapping | Prefix matching is defeated by `..`, symlinks, and `C:\foo` vs `C:\foobar`; `commonpath` is component-aware |
| Immutable terminal records | App-level "don't update" discipline | SQL `BEFORE UPDATE/DELETE` triggers (0015 pattern) | DB-enforced immutability survives buggy callers; matches `agent_contract_snapshots` |
| Atomic dual-table transitions | Two separate writes | One `with lock: with conn:` block (run_service pattern) | TOCTOU-safe; rolls back both tables on any exception |
| Secret redaction in audit | New regex set | `SECRET_PATTERNS` + `audit_service._redact` | Already battle-tested; AUD-01 routes through `emit()` which redacts |
| Resume state reconstruction | New snapshot format | `RunContractSnapshot` / `ResumeSnapshot` replay | 10.2 already froze the resume invariants; SURF-05 reuses them |

**Key insight:** The single highest-risk "hand-roll" temptation is process liveness. The codebase has no precedent for it, the obvious stdlib idiom is Windows-broken, and getting it wrong produces either orphaned running sessions (SURF-05 failure) or spurious reclamation of live work. The owner-token + heartbeat-TTL design avoids OS-level process introspection entirely and is the lowest-risk default.

## Common Pitfalls

### Pitfall 1: `os.path.commonpath` raises on mixed Windows drives
**What goes wrong:** `commonpath(["C:\\a", "D:\\b"])` raises `ValueError: Paths don't have the same drive` instead of returning a non-containing result. An uncaught `ValueError` escapes the resolver as an untyped error.
**Why it happens:** Windows paths on different drives have no common ancestor; `commonpath` treats this as an error, not a false containment.
**How to avoid:** Wrap `commonpath` in `try/except ValueError` and map to a typed `traversal`/`unregistered` error (fail closed). This is actually convenient — the raise IS the fail-closed signal.
**Warning signs:** A workspace on a different drive than the project root surfaces a raw traceback instead of a clean denial. [VERIFIED live on this machine: `ValueError: Paths don't have the same drive`]

### Pitfall 2: `os.kill(pid, 0)` is not a liveness check on Windows
**What goes wrong:** The POSIX `try: os.kill(pid, 0); except ProcessLookupError: dead` idiom returns the wrong answer on Windows: a **dead** PID raises `OSError(EINVAL)` (errno 22, "incorrect parameter"), not `ESRCH`/`ProcessLookupError`, and the code may misclassify it as alive or crash on the wrong exception type.
**Why it happens:** Windows `os.kill` only meaningfully supports `CTRL_C_EVENT`, `CTRL_BREAK_EVENT`, and `SIGTERM`; signal 0 is not a no-op probe as on POSIX.
**How to avoid:** Use `psutil.pid_exists(pid)` (already installed) **or** the recommended owner-token + heartbeat-TTL design that never probes the OS. Guard PID reuse with `create_time` (psutil) or the owner-token.
**Warning signs:** Reconciliation never reclaims orphans on Windows, or reclaims live sessions. [VERIFIED live: `os.kill(999999, 0)` → `OSError errno=22` on win32; `psutil.pid_exists(999999)` → `False` correctly]

### Pitfall 3: Cancellation that assumes a force-kill
**What goes wrong:** A design that expects to terminate the run thread immediately. CPython has no safe thread-kill; the `run_conversation` call inside `NativeAtlasAgent` is opaque and blocking.
**Why it happens:** Mental model from languages with cancellable threads/futures.
**How to avoid:** Cooperative checkpoints only (Pattern 7). For subprocesses spawned by tools, use `proc.terminate()`/`proc.kill()` in a poll-loop that checks the token — those ARE killable. For the model stream and in-process agent, rely on between-turn polling. Document that an opaque single blocking model call cannot be interrupted mid-call; cancellation takes effect at the next checkpoint.
**Warning signs:** "cancel" appears to hang until `max_runtime_s` watchdog fires.

### Pitfall 4: Global workspace root drift between DB home and workspace root
**What goes wrong:** `workspace_service` resolves global root differently from where `db.py` opens the DB, so a "global" session's containment root and the actual `~/.atlas` data dir diverge (e.g., one honors `ATLAS_HOME`, the other does not — the known CLI caveat).
**How to avoid:** Resolve the global root from the same source as `db.DEFAULT_DB_PATH` (`Path.home()/".atlas"`, ATLAS_HOME-aware). Add a single shared resolver so both agree. Test under a temp `ATLAS_HOME`.
**Warning signs:** Global-workspace writes denied as `traversal` because the containment root points elsewhere.

### Pitfall 5: Symlink escape masked by `.resolve()` ordering
**What goes wrong:** Checking containment on the **pre-resolved** target lets a symlink inside the workspace point outside it pass the check; checking only the resolved target loses the ability to label the failure `symlink_escape` vs `traversal`.
**How to avoid:** Resolve both root and target with `.resolve()` (collapses symlinks) **before** `commonpath`. To distinguish `symlink_escape` from `traversal` for the typed error, compare the un-resolved vs resolved target (a path component that changed identity under resolution indicates a symlink). `.resolve()` is already the convention in `_resolve_dir`.
**Warning signs:** A symlink pointing outside the project root is accepted, or every escape is labeled generically.

### Pitfall 6: Emitting audit inside the write lock → deadlock
**What goes wrong:** Calling `audit_service.emit()` inside the `with lock: with conn:` block deadlocks, because `emit()` re-acquires the same lock internally.
**Why it happens:** The established pattern (run_service) deliberately releases the lock before emitting. New session/workspace services must follow it.
**How to avoid:** Do the SQL transition inside the lock, release, **then** `emit()`. This is documented in `run_service.py` ("Emit-after-lock pattern prevents deadlock").
**Warning signs:** Session-transition tests hang.

## Open Questions (RESOLVED)

*All three open questions were resolved during planning (Phase 10.3 plan set 01-05). Resolutions are binding planning decisions.*

1. **psutil dependency vs stdlib owner-token for liveness — RESOLVED: stdlib owner-token + heartbeat-TTL.**
   - Resolution: adopt the **stdlib owner-token + heartbeat-TTL (3× interval)** design (plan 04). No psutil dependency is added. Reclamation latency is bounded by the TTL, which is acceptable for SURF-05 (a fresh deploy reconciles at startup; live sessions heartbeat). `os.kill(pid, 0)` is never used (Windows-broken, Pitfall 2). psutil hardening is explicitly out of scope for 10.3.

2. **Cancellation granularity inside the foundation harness (A2) — RESOLVED: between-turn + tool-gate + subprocess only; no mid-call interruption.**
   - Resolution: cancellation is COOPERATIVE at three ATLAS-owned checkpoints — (a) the executor/native watchdog poll-join between turns, (b) the `tool_service.invoke` tool gate (a tool/subprocess does not START once the token is set), (c) subprocess `proc.terminate()` in a poll-loop (plan 04 tasks 1–2). The single opaque foundation `run_conversation(prompt)` model call CANNOT be interrupted mid-call: D-001 forbids editing foundation internals and no per-turn hook is exposed (verified by inspection scope — the foundation harness exposes no cancel callback the ATLAS wrapper can observe). SURF-06's "model stream" wording is satisfied at between-turn granularity; this bound is stated honestly in plan 04's objective and SURF-06 acceptance criteria. The max-runtime watchdog is the hard backstop.

3. **Should `surface_sessions` FK to `runs`, or `runs` FK to a session? — RESOLVED: soft link, no FK.**
   - Resolution: keep the session→run link **soft (no FK)** (plan 01 migration 0016), mirroring the `tool_approvals.run_id` stored-not-FK-enforced precedent. Existing runs carry arbitrary/None `session_id`; a hard FK would break them. Integrity is enforced in the service layer (`surface_session_service`).
