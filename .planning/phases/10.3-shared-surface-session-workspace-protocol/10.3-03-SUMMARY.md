# Plan 10.3-03 Summary — normalized surface event stream

## What shipped

- `audit_service.get_events_for_session(conn, session_id)` — orders AuditEvents across
  ALL of a session's runs (`JOIN runs r ON ae.run_id = r.id WHERE r.session_id=? ORDER BY
  ae.timestamp ASC, ae.rowid ASC`), the per-session source the normalizer requires.
- `services/agent-runtime/atlas_runtime/surface_events.py`:
  - `_KIND_MAP` covering every `AuditEvent.event_type` member (a test asserts completeness),
    with `llm_call` refined to text/reasoning by payload and a `surface_kind` producer hint
    that makes the full `EventKind` vocabulary (incl. `retry`) reachable.
  - `normalize_surface_events(audit_events, run_outcome=None, *, session_id, start_seq=0)` —
    pure projection assigning a monotonic per-session `seq` over the whole sequence, appending
    a `completion` event from the terminal `RunOutcome`. Returns a tuple; writes nothing.
  - `replay_since(events, last_seq)` — reconnect gap-detection primitive.

## Decisions

- **Producer `surface_kind` hint** added so the SURF-04 enumeration is fully reachable — the
  ledger has no native "retry" event_type, so a producer tags it in the (already-redacted)
  `data` payload and the normalizer honors it over the default map. Payload is parsed only for
  the kind decision; `payload_json` stays the original redacted string (D-013).
- Pure read-projection — no new event bus (AGNT-01), enforced by a no-writes test and grep gate.
