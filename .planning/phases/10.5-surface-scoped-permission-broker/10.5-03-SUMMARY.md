# Phase 10.5 Plan 03: Surface-Scoped Permission Broker Core Summary

The broker core that routes deferred tool approvals to the initiating surface session,
exposes the strictly-scoped actionable queue, and resolves approvals via an
authority-checked, at-most-once `claim()` that delegates the only status-flip to
`tool_service.approve()/reject()` — losers receive a typed `AlreadyDecided` carrying the
winning decision; restart reconciliation fails orphaned `executing` rows closed.

## What Was Built

### Task 1 — `invoke()` surface binding seam (`tool_service.py`)
- Added keyword-only `surface_session_id` / `surface_kind` to `invoke()`. Existing
  callers omit both and persist NULL surface fields (back-compatible; the 7 existing
  tool_service tests still pass).
- In the `requires_approval` branch the persisted `ToolApproval` is now anchored to the
  initiating surface session: `workspace_root` (from `ctx`), a fresh `uuid4` nonce,
  `expiry_at = now + approval_ttl_seconds`, and `args_normalized` via the shared
  `_normalize_args` helper (single redaction path, D-002).
- TTL resolver `_resolve_approval_ttl_seconds(ctx)`: `ctx['approval_ttl_seconds']`
  override → `PermissionConfig.approval_ttl_seconds` → `300` fallback, wrapped so a
  missing/locked/invalid config never breaks `invoke()`.
- The `approval` audit emit now carries `session_id` + `surface_kind` + `workspace_root`
  provenance. The PERM-05 fail-closed gate was intentionally NOT added here (Plan 04).

### Task 2 — `permission_broker.py` (new) + conftest fix
- `list_actionable(conn, *, surface_session_id)` — `WHERE surface_session_id=? AND
  status='pending' AND expiry_at > now`, strict scope, never another session's rows.
- `claim(conn, lock, *, approval_id, surface_session_id, decision, nonce, ctx, reason)`:
  1. ownership check (`approval.surface_session_id == caller` else `WrongSessionError`);
  2. active-state authority (`SELECT state FROM surface_sessions` == 'active' else
     `NotActiveSessionError`; column read only, never a PID probe);
  3. write the non-status `decision` column (cannot race the guard while still pending);
  4. delegate the atomic pending→terminal claim+execute to `tool_service.approve()`
     (approve) or `reject()` (reject) — the single at-most-once authority;
  5. loser path: catch `ToolApprovalError` (rowcount==0), re-read, raise `AlreadyDecided`;
  6. winner emits routing/claim provenance AFTER releasing the lock.
- `reconcile_executing_orphans(conn, lock)` — flips `executing` + NULL-result rows to
  `failed` (reason `reconciled_fail_closed`), leaves `pending` intact, idempotent.
- `list_outcomes(conn)` — read-only terminal projection (no mutating verb).
- Typed `AlreadyDecided` (Exception), `WrongSessionError`, `NotActiveSessionError`, `__all__`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] conftest `db` fixture was thread-affine**
- **Found during:** Task 2 (`test_concurrent_claim_at_most_once`)
- **Issue:** The shared in-memory `db` fixture connected without `check_same_thread=False`,
  so the at-most-once TOCTOU test's worker threads raised
  `sqlite3.ProgrammingError: SQLite objects created in a thread can only be used in that
  same thread`. This contradicts the production concurrency contract (`audit_service`:
  "may be shared across threads with check_same_thread=False" + a `threading.Lock`).
- **Fix:** Connect the `db` fixture with `check_same_thread=False`. This does not weaken any
  assertion — the test still proves exactly one winner reaches a terminal status and exactly
  one loser receives `AlreadyDecided`.
- **Files modified:** `services/agent-runtime/tests/conftest.py`
- **Commit:** 3e4d9e3

### Contract clarification (not a code deviation)

The plan's prose described `AlreadyDecided` as "a returned result dataclass — NOT an
exception." The executable Wave-0 contract, however, both `except broker.AlreadyDecided`
(concurrent + replay tests) and `pytest.raises((..., broker.AlreadyDecided))`. Tests are the
binding contract, so `AlreadyDecided` is an `Exception` subclass (still carrying
`status`/`decision`/`reason` + `already_decided=True`) and the loser RAISES it. This
satisfies every test verbatim while preserving the "loser receives the winning decision"
intent (PERM-06).
