<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| headless/API surface → invoke() | An automated/api caller must not silently auto-approve a write/shell action without an explicit approval channel |
| stale/replayed client → claim() | A re-submitted or expired decision must never resolve a pending approval |
| client claim contract → broker | Malformed/unknown decision input must fail closed before reaching SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.5-04-HEADLESS | Elevation of Privilege | invoke() PERM-05 gate | mitigate | surface_kind=='api' with no unrevoked approval_channels row → immediate deny (reason 'no_approval_channel'); never creates a pending row (PERM-05) |
| T-10.5-04-REPLAY | Tampering | claim() nonce | mitigate | Per-row nonce echo; mismatch → StaleApprovalError, no row touched (SEC-02) |
| T-10.5-04-STALE | Tampering | claim() expiry | mitigate | now > expiry_at → deny before atomic UPDATE; fail-closed (SEC-02, PERM-01 expiry) |
| T-10.5-04-MALFORMED | Tampering | claim contract | mitigate | Closed decision-label set + field validation before any SQL; typed error (SEC-02 V5 input validation) |
| T-10.5-04-RACE | Tampering | stale-then-claim ordering | mitigate | nonce + expiry checks run BEFORE the rowcount-gated UPDATE so a stale claim cannot win the race |
| T-10.5-04-DEADLOCK | Denial of Service | deny audit emit | mitigate | Deny/expiry audit emits occur AFTER releasing the lock (Pitfall 1/6) |
| T-10.5-04-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages; stdlib + existing atlas_core/atlas_runtime only; slopcheck N/A |
</threat_model>
