### `packages/atlas-core/atlas_core/schemas/tool.py` (model, in-place extension)

**Analog:** self — the existing frozen `ToolApproval` (lines 77-99). Mirror the 0017 DDL 1:1; add new fields as `Optional` so legacy NULL rows still load (RESEARCH Runtime State Inventory).

**Frozen model + ISO serializer pattern** (lines 77-99 — `ConfigDict(frozen=True)`, `Field(default_factory=...)` for id/timestamp, `@field_serializer` for datetimes → ISO-8601):
```python
class ToolApproval(BaseModel):
    model_config = ConfigDict(frozen=True, str_strip_whitespace=True)

    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    tool_name: str
    risk_level: ToolRiskLevel
    args: str = "{}"  # JSON string, secret-redacted before persistence (D-013)
    status: ToolApprovalStatus = "pending"
    reason: Optional[str] = None
    result: Optional[str] = None
    run_id: str = "operator"
    requested_at: datetime.datetime = Field(
        default_factory=lambda: datetime.datetime.now(datetime.timezone.utc)
    )
    decided_at: Optional[datetime.datetime] = None

    @field_serializer("requested_at", "decided_at")
    def serialize_dt(self, dt: datetime.datetime | None) -> str | None:
        return None if dt is None else dt.isoformat()
```

New fields to add (all `Optional[...] = None` for legacy back-compat): `surface_session_id`, `surface_kind`, `workspace_root`, `decision`, `nonce`, `args_normalized` (all `Optional[str]`), `expiry_at: Optional[datetime.datetime]` (add to `@field_serializer` list). Add `ApprovalChannel` (and optional `SessionAllowRule`) frozen models in the same file with the same `field_serializer` discipline; export both via `__all__` (lines 102-109). Status Literal (line 30) — RESEARCH recommends reusing the existing 5 values and carrying cause in `reason`/`decision`; adding `expired` is a one-line Literal change with no migration.

**Lockstep warning (Pitfall 2):** any field added here MUST be mirrored in `tool_service._COLS`, the INSERT column list/placeholders, and `_row_to_approval` — they zip column order into kwargs and silently mismap on drift.

## Shared Patterns

### Atomic at-most-once claim (PERM-06)
**Source:** `services/agent-runtime/atlas_runtime/tool_service.py:232-244` (`approve`), `:288-295` (`reject`)
**Apply to:** every state transition out of `pending` in `permission_broker.claim()`
```python
with lock:
    with conn:
        cur = conn.execute(
            "UPDATE tool_approvals SET status = ? WHERE id = ? AND status = 'pending'",
            (new_status, approval_id),
        )
        claimed = cur.rowcount == 1
if not claimed:
    current = _load(conn, approval_id)   # losers re-read → typed AlreadyDecided (10.5)
```

### Construct + validate frozen model BEFORE INSERT (D-013)
**Source:** `surface_session_service.py:173-224`
**Apply to:** every INSERT in the broker / extended `invoke()`
```python
model = ToolApproval(...)         # Pydantic validates BEFORE any SQL
dumped = model.model_dump()       # datetimes → ISO strings
with lock:
    with conn:
        conn.execute("INSERT INTO ... VALUES(...)", (...))
```

### Emit AFTER releasing the lock (Pitfall 6 — deadlock guard)
**Source:** `surface_session_service.py:226-238`, `audit_service.py:157-158` (`emit` re-acquires the lock)
**Apply to:** every audit emission in the broker / `tool_service`
```python
with lock:
    with conn:
        conn.execute("UPDATE ...")
# lock released here
audit_service.emit(conn, lock, run_id=..., event_type="approval",
                   session_id=surface_session_id,
                   data={"approval_id":..., "status":..., "decision":...,
                         "surface_kind":..., "workspace_root":...})
```

### Audit provenance enrichment (AUD-02 / D-002)
**Source:** `audit_service.py:63-128` (`emit`), `tool_service.py:152-159` (existing `approval` emit)
**Apply to:** all approval lifecycle transitions. `_redact()` runs automatically inside `emit`; reuse `event_type="approval"` (already in the `AuditEvent` Literal) and carry surface provenance in the `data` payload. Do NOT add a second redaction path.

### Single redaction at the audit boundary (D-002)
**Source:** `audit_service.py:36-55` (`_redact`), reused in `tool_service.py:111`
**Apply to:** `args_normalized` storage and all `data`/`args`/`result` JSON strings — redact ONCE, never a second sanitizer.

## No Analog Found

None. Every primitive this phase needs already exists in the codebase. This is an additive
extension (columns, one registration table, a routing kwarg, an authority-checked claim API,
a read-only projection). The risk is *introducing a parallel mechanism* (a second lock, second
redaction path, second migration runner) that diverges from established invariants.

| Pattern | Use existing, do NOT build |
|---------|----------------------------|
| At-most-once execution | rowcount-gated `UPDATE ... WHERE status='pending'` (tool_service) |
| Migration apply/track | `db.apply_migrations()` + `schema_migrations` |
| Secret redaction | `audit_service._redact()` |
| Session liveness | `surface_sessions.state` + heartbeat TTL (never `os.kill(pid,0)` — broken on Windows) |
| Config persistence / TTL | `PermissionConfig` field in control_plane.py |
| Cross-surface visibility | audit-event read projection (no pub/sub bus — D-022) |
| Identity contracts | compose `SurfaceIdentity`/`WorkspaceIdentity` (AGNT-01) |
