<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| surface session → claim() | A surface client requests resolution; the broker must verify ownership + live-active state before touching any row |
| concurrent approvers → SQLite | Multiple replies/reconnects race the same pending row; only one deferred execution may occur |
| crashed process → restart | An interrupted execution leaves an orphaned 'executing' row that must fail closed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.5-03-SPOOF | Spoofing / Elevation | claim() ownership | mitigate | claim() requires approval.surface_session_id == caller AND surface_sessions.state=='active'; cross/terminal session → typed reject, no row touched (PERM-02) |
| T-10.5-03-DOUBLE | Tampering | concurrent claim | mitigate | Verbatim rowcount-gated UPDATE WHERE status='pending' inside with lock: with conn:; loser gets AlreadyDecided; deferred adapter runs at most once (PERM-06) |
| T-10.5-03-ORPHAN | Denial of Service | restart reconcile | mitigate | reconcile_executing_orphans flips executing+NULL-result → failed (fail-closed); pending rows survive intact (PERM-06) |
| T-10.5-03-PIDPROBE | Spoofing | liveness check | mitigate | Authority reads surface_sessions.state column only; os.kill PID probing forbidden (broken on Windows; RESEARCH anti-pattern) |
| T-10.5-03-DEADLOCK | Denial of Service | audit emit | mitigate | emit() called AFTER releasing the lock (Pitfall 1/6); test hangs are the warning sign |
| T-10.5-03-LEAK | Information disclosure | audit data payload | mitigate | _redact() runs inside emit(); args_normalized already redacted at insert; no second sanitizer |
| T-10.5-03-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages; stdlib + existing modules only; slopcheck N/A |
</threat_model>
