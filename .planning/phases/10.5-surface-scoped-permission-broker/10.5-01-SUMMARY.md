# Phase 10.5 Plan 01: Wave 0 Permission-Broker Failing-First Test Contract Summary

Authored the Wave 0 RED-first test module `test_permission_broker.py` plus three conftest
helpers, locking the surface-scoped permission broker's public API and full behavioral
contract (PERM-01..07, SEC-02, AUD-02) as executable assertions before any broker code exists.

## What Was Built

- **conftest helpers (Task 1)** — three fixture-returned callables added to
  `services/agent-runtime/tests/conftest.py`:
  - `make_active_session` — flips the existing `surface_session` row to `state='active'`
    via a direct column UPDATE and returns its id (PERM-02 authority precondition).
  - `seed_pending_approval` — INSERTs a surface-anchored `pending` `tool_approvals` row
    carrying the migration-0017 columns (`surface_session_id`, `surface_kind`,
    `workspace_root`, `expiry_at`, `nonce`, `args_normalized`) alongside the base 0013
    columns; returns the approval id.
  - `register_test_channel` — INSERTs an unrevoked `approval_channels` row (PERM-05
    fail-closed precedent).
  - Liveness is set purely by column UPDATE; no PID-signal probing anywhere (PATTERNS
    "do NOT build" table). stdlib only (sqlite3 via the passed conn, datetime, uuid).

- **Broker contract module (Task 2)** — `test_permission_broker.py` (459 lines) imports
  `from atlas_runtime import permission_broker as broker` (RED: module absent until Wave 2+)
  and defines the nine named tests, mapped 1:1 to the RESEARCH Test Map:
  - `test_approval_record_fields` (PERM-01)
  - `test_claim_requires_matching_active_session` (PERM-02: WrongSessionError +
    NotActiveSessionError + active-match success)
  - `test_headless_fail_closed_without_channel` (PERM-05)
  - `test_concurrent_claim_at_most_once` (PERM-06: `threading.Barrier`, exactly one winner
    + one `AlreadyDecided` loser, DB terminal-once)
  - `test_restart_reconcile_executing_orphan` (PERM-06: `executing` orphan -> `failed`,
    `pending` untouched)
  - `test_allow_rules_cannot_widen_scope` (PERM-07: session-A rule never resolves session-B
    / other-workspace approvals)
  - `test_replay_and_stale_rejected` (SEC-02: nonce mismatch + expiry -> StaleApprovalError;
    malformed -> typed error)
  - `test_outcome_projection_read_only` (AUD-02: `list_outcomes` mutates no status)
  - `test_audit_provenance_emitted` (PERM-01/AUD-02: `audit_events` row, `event_type='approval'`,
    `session_id == surface_session_id`, `approval_id` in the data payload)

The module references the exact public API names Waves 2-4 must implement verbatim:
`list_actionable`, `claim`, `register_channel`, `revoke_channel`,
`reconcile_executing_orphans`, `list_outcomes`, and typed errors `AlreadyDecided`,
`NotActiveSessionError`, `WrongSessionError`, `StaleApprovalError`,
`ApprovalChannelMissingError`.
