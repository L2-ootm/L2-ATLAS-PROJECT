## Decisions Made

- Migration is purely additive: `model_registry` (0003) stays live for the gateway until the 10.3 reader cutover. The compat VIEW (which maps the six named columns the gateway selects) is drafted as a SQL comment in 0004 as the recommended 10.3 mechanism.
- `route_policy` table created now to avoid a v1.2 migration churn, but marked SCHEMA-ONLY / no reader-writer in v1.1 (LANDMINE 4 embedded in DDL comment per plan requirement).
- FK enforcement deferred to v1.2: discovery may write model rows before provider rows exist; logical relationship documented in comments only.
- `ModelV2.auth_status` is a derived `Literal` field (not a stored secret); the credential boundary is documented in both the class docstring and inline comment.
