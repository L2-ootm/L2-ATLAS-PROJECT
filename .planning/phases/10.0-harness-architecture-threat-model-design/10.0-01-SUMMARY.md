# Phase 10.0 Plan 01: Architecture Design Docs Summary

**One-liner:** Three load-bearing design documents (auth-store layout, adapter/foundation boundary, fallback-cascade contract) committed with seven embedded landmines — de-risking AUTH-01/02/SEC-03 (10.1), AGNT-01..06 (10.2), and AUD-01 (10.2) before any harness code is written.

## What was built

**Task 1 — AUTH_STORE_DESIGN.md** (f6a29ec)

Committed the ATLAS auth-store layout decision: flat `~/.atlas/auth.json` as a peer of `atlas.db`, a single `auth_store_path()` resolver honoring `ATLAS_HOME`, JSON `{"version":1,"providers":{…}}` format, atomic-write pattern mirroring Hermes (`O_EXCL` temp + fsync + `os.replace`), and `icacls /inheritance:r /grant:r` Windows permissions. Embeds:
- **LANDMINE 1:** `os.replace` on Windows is not guaranteed atomic (`MoveFileEx` can fall back to `CopyFile`); the cross-process lock, not the rename, is the no-corruption guarantee.
- **LANDMINE 2:** the lock must be an OS-handle advisory lock (`fcntl.flock`/`msvcrt.locking` on an open fd), not a presence-based lock — auto-release on process death is load-bearing for crash-safety.

**Task 2 — ADAPTER_BOUNDARY_DESIGN.md + DIVERGENCE_LOG D-LOG-002** (0c3378e)

Committed the adapter/foundation boundary: `AIAgent` wrapped by composition in `services/agent-runtime/atlas_runtime/`, one-way dependency, five adapter responsibilities (credential resolution, adapter-driven cascade, audit emission, approval-gate preservation, transcript redaction), and AGNT-01 hard gate ("wrap, never rewrite"). Back-filled `D-LOG-002` in `foundation/atlas-hermes/DIVERGENCE_LOG.md` for the `atlas_audit` plugin shim. Embeds:
- **LANDMINE 3 (reconciliation):** the `DIV-F-*` label scheme referenced in CONTEXT docs is incorrect and superseded; the canonical id scheme is `D-LOG-NNN`; atlas_audit was not previously recorded in the log.

**Task 3 — FALLBACK_CASCADE.md** (093ffdb)

Committed the canonical halt-vs-cascade contract: error-classification markdown table (401/403 HALT, network/5xx/429 CASCADE, 404 model-scoped, 400 body-inspected, refusal not-an-error), OpenAI-first lane by `api_mode` membership, health-ordered remaining providers, bounded attempt budget (N=3), `provider_fallback{from,to,reason}` audit on every advance, provider-used surfaced on response. Embeds:
- **LANDMINE 6:** `400` cannot be classified by status code alone — body/`code`/`message` inspection required; ambiguous defaults to HALT.
- **LANDMINE 7:** non-conformant/garbled provider responses (200-with-error-body, bare connection error, non-standard codes) are CASCADE-class transport failures; defensive posture inherited from v1 `model_registry.py`.
