<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| auth material at rest -> filesystem | secrets stored in `~/.atlas/auth.json`; the design must keep them owner-only and out of SQLite/audit |
| ATLAS adapter -> vendored foundation | one-way import boundary; foundation must stay upstream-mergeable |
| provider error surface -> cascade logic | untrusted provider responses classified into HALT/CASCADE; misclassification leaks errors or burns budget |

## STRIDE Threat Register

This phase ships NO runtime code; the residual surface is documentation correctness. The threats below are the ones the designed artifacts mitigate (mitigation = the design recording the correct invariant).

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.0-01 | Information Disclosure | auth-store at-rest secrets | mitigate | AUTH_STORE_DESIGN.md mandates 0600/icacls owner-only, O_EXCL create (closes umask TOCTOU), redacted_hint-only display, secrets never in SQLite/audit (SEC-01/03) |
| T-10.0-02 | Tampering / DoS | concurrent torn write of auth.json | mitigate | Design records the OS-handle cross-process lock (not os.replace) as the no-corruption guarantee + fsync + .corrupt recovery (LANDMINE 1/2) |
| T-10.0-03 | Tampering | foundation forked off-upstream | mitigate | ADAPTER_BOUNDARY_DESIGN.md enforces wrap-not-rewrite + extension-points-only via DIVERGENCE_LOG (D-LOG-002 back-filled) |
| T-10.0-04 | Repudiation / Information Disclosure | silent auth-failure cascade | mitigate | FALLBACK_CASCADE.md makes 401/403 HALT-and-audit; provider_fallback{from,to,reason} emitted on every cascade (AGNT-03, AUD-01) |
| T-10.0-05 | DoS | malformed 400 cascaded to N providers | mitigate | LANDMINE 6: 400 requires body inspection; ambiguous defaults to HALT; bounded attempt budget |
| T-10.0-DOC | Tampering | documentation drift (DIV-F vs D-LOG) | mitigate | LANDMINE 3 reconciled: D-LOG-NNN is canonical; atlas_audit back-filled as D-LOG-002 |

No package-manager installs occur in this plan (RESEARCH "Package Legitimacy Audit": not applicable — 10.0 installs nothing). No `high` residual threat is introduced: the plan produces only documents and one log entry.
</threat_model>
