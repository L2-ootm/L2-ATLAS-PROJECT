    - Implementation hint for the future spike: stdlib `http.server` bound to `("127.0.0.1", 0)`, read the chosen port from `server.server_address[1]`, single-request handler captures `code`+`state` then `server.shutdown()`. No third-party dependency.
    - EMBED the constant-time-state landmine as a HARD GATE: the `state` comparison MUST be constant-time (`hmac.compare_digest`) and the listener MUST reject any callback whose `state` does not match the issued value BEFORE processing `code`. A naive `==` plus "first callback wins" is the classic loopback-hijack vector.
    - Record the port-flexibility precondition: the redirect URI registered with the authorization server must tolerate the port varying (RFC 8252 §7.3); list "provider supports loopback port flexibility" as a precondition not all providers honor.
    Render the gates as a numbered list and/or a requirement table; do NOT inline a full implementation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webview (untrusted) -> Tauri IPC core | every IPC command crosses from untrusted frontend into privileged native code |
| OAuth authorization server -> loopback listener | the redirect callback carries `code`+`state` into a local listener; interception/CSRF is the threat |
| frontend -> PTY child | bytes vs command strings — the load-bearing distinction for NAT-03 |

## STRIDE Threat Register

Both deliverables are prose threat-model drafts (no runtime code). The threat model is partly the deliverable itself — the table below enumerates the threats the drafted specs mitigate.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.0-OAUTH | Spoofing / Tampering | OAuth code interception / CSRF | mitigate | OAUTH draft requires PKCE-S256 + constant-time `state` (hmac.compare_digest) validated before code + exact redirect match + one-shot loopback listener + no-secret-in-logs |
| T-10.0-IPC | Elevation of Privilege | arbitrary-exec via IPC command string | mitigate | NATIVE_IPC draft: PTY transports bytes to a fixed `atlas tui` child; no command-string param (LANDMINE 5 / NAT-03); deny-by-default Tauri 2 capabilities |
| T-10.0-IPC2 | Elevation of Privilege | open_external scheme abuse | mitigate | Tauri scope scheme allowlist (https/mailto only) |
| T-10.0-IPC3 | Elevation of Privilege | compromised remote origin invoking IPC | mitigate | NAT-01: no remote origin in production; cockpit from local bundle; gateway stays loopback HTTP, not an IPC surface |

No package-manager installs occur in this plan (the `openai`/Tauri/PTY-plugin installs are owned by 10.2/10.5, which run the Package Legitimacy Gate at install time). No `high` residual threat introduced: the plan produces only threat-model documents.
</threat_model>
