## Pattern Assignments

### `infra/migrations/0004_registry_v2.sql` (migration, CRUD)

**Analog:** `infra/migrations/0003_model_registry.sql` (closest — same file family, same purpose) plus `infra/migrations/0001_core.sql` (for pragma + multi-table + comment-header conventions).

**DDL house style to copy column-for-column** (from `0003`, lines 1-13):

```sql
-- 0003: model registry for the AI router (D-017).
-- Mirrors atlas_runtime/model_registry.py ensure_schema(); both are
-- CREATE TABLE IF NOT EXISTS so apply order is idempotent.
-- The Rust gateway (D-022) reads this table directly for /models.

CREATE TABLE IF NOT EXISTS model_registry (
    model_id    TEXT PRIMARY KEY,
    provider    TEXT NOT NULL DEFAULT '',
    source      TEXT NOT NULL,
    first_seen  TEXT NOT NULL,
    last_seen   TEXT NOT NULL,
    active      INTEGER NOT NULL DEFAULT 1
);
```

**Conventions the `0004` draft must reproduce exactly** (extracted from `0001`/`0003`):
- **Header comment:** `-- 000N: <purpose> (<decision-id>).` followed by a `-- Mirrors <python module> ensure_schema(); ... CREATE TABLE IF NOT EXISTS so apply order is idempotent.` note. The `0004` header must add the two landmine notes (see below).
- **All tables `CREATE TABLE IF NOT EXISTS`** — idempotent re-apply is the migration contract (sorted-glob `executescript`, no new runner).
- **Column alignment:** name column left-aligned, type padded to a fixed column (see `0001` `mission_id  TEXT NOT NULL REFERENCES missions(id)`). Match the visual padding.
- **Types:** timestamps are `TEXT NOT NULL` storing ISO-8601 strings (never a SQLite DATE/DATETIME type — see every `*_at`/`*_seen` column). Booleans are `INTEGER NOT NULL DEFAULT 1`. JSON blobs are `TEXT ... DEFAULT '{}'` (see `0001` `data TEXT NOT NULL DEFAULT '{}'`).
- **Pragmas:** WAL + foreign_keys are declared ONCE in `0001` (`PRAGMA journal_mode = WAL; PRAGMA foreign_keys = ON;`). `0003` does NOT repeat them. `0004` should follow `0003` and NOT redeclare pragmas (they are connection/db-level, already set by `0001`). Do not add an FTS5 virtual table — `0004` has no full-text need (FTS5 only appears in `0001` for `wiki_pages`).
- **Composite PK syntax:** `0001`/`0003` only show single-column `PRIMARY KEY` inline. For the new composite key use a table-level constraint: `PRIMARY KEY(model_id, provider_id, source)` as the last item in the column list (standard SQLite form — no analog in-repo, follow seed §8 verbatim).

**Three tables the draft must define** (column-for-column per CONTEXT Area 3 / seed §8):
1. `provider_registry(provider_id PK, display_name, auth_type, default_base_url, api_modes_json, source, status, last_checked, last_error)`
2. `model_registry_v2(model_id, provider_id, source, api_mode, status, auth_status, context_window, metadata_json DEFAULT '{}', first_seen, last_seen, deactivated_at, PRIMARY KEY(model_id, provider_id, source))`
3. `route_policy(task_class PK, provider_id, model_id, fallback_policy_json DEFAULT '{}', updated_at)`

**Recommended indexes** (RESEARCH Area 3 OQ2, cheap, support the source-scoped deactivation pass): `CREATE INDEX IF NOT EXISTS ... ON model_registry_v2(source)` and `ON model_registry_v2(source, deactivated_at)`.

**LANDMINES that change how this file is written (from RESEARCH):**
- **Do NOT `DROP` or `ALTER` `model_registry` (the `0003` table).** Migration is purely additive. The gateway still reads `model_registry` directly until the 10.3 cutover. (RESEARCH Area 3 + CONTEXT Area 3 backward-compat.)
- **`route_policy` is schema-only in v1.1 — no reader/writer.** The migration COMMENT must say so explicitly (LANDMINE 4) so a future executor doesn't assume it's wired (`atlas route` / enforcement is v1.2+).
- **Record the compatibility-VIEW as the recommended 10.3 cutover, not applied now.** The gateway reads named columns `model_id, provider, source, first_seen, last_seen, active FROM model_registry` (verified `db.rs:255`), so a view exposing those six columns over `model_registry_v2` is a drop-in — but a view and a table cannot share a name, so it can only land at the 10.3 cutover. Draft the view DDL as a commented option / design note, do not include it as live DDL in `0004`. (RESEARCH Area 3 VIEW option.)
- **Do not enforce FK** on `model_registry_v2.provider_id` / `route_policy.provider_id` in v1.1 (discovery may write a model before its provider row exists). Document the logical relationship in a comment; defer hard FK to v1.2. (RESEARCH Area 3 OQ1.)
- **UPSERT pattern is a 10.3 contract, not 10.0 DDL** — but the draft should document it so the DDL is justified: `INSERT ... ON CONFLICT(model_id, provider_id, source) DO UPDATE SET ... last_seen = excluded.last_seen, deactivated_at = NULL` with `first_seen` intentionally NOT in the SET clause (preserves original). (RESEARCH Area 3 + Don't-Hand-Roll.)

### Pydantic models `Provider` / `ModelV2` / `RoutePolicy` (model, transform)

**Analog:** `packages/atlas-core/atlas_core/schemas/core.py` (frozen Pydantic v2 models + `SECRET_PATTERNS`), with `schemas/__init__.py` as the barrel-export analog.

**Model-definition pattern to copy** (from `core.py`, `Mission` lines 37-57 — the canonical shape):

```python
class Mission(BaseModel):
    """Top-level unit of intent — a goal the agent is working toward."""

    model_config = ConfigDict(frozen=True, str_strip_whitespace=True)

    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    title: str
    intent: str = ""
    status: Literal["pending", "running", "succeeded", "failed", "cancelled"] = "pending"
    project: str = ""
    created_at: datetime.datetime = Field(
        default_factory=lambda: datetime.datetime.now(datetime.timezone.utc)
    )
    updated_at: datetime.datetime = Field(
        default_factory=lambda: datetime.datetime.now(datetime.timezone.utc)
    )

    @field_serializer("created_at", "updated_at")
    def serialize_dt(self, dt: datetime.datetime | None) -> str | None:
        """Return ISO 8601 string so model_dump() is JSON-safe."""
        return None if dt is None else dt.isoformat()
```

**Imports / module-header pattern** (from `core.py`, lines 10-17):

```python
from __future__ import annotations

import datetime
import re
import uuid
from typing import Literal, Optional

from pydantic import BaseModel, ConfigDict, Field, field_serializer
```

**House conventions the new models must follow (D-012 / D-013):**
- **Every model `model_config = ConfigDict(frozen=True, str_strip_whitespace=True)`** — all domain models are frozen.
- **DDL mirrors fields 1:1.** The module docstring states "DDL in infra/migrations/000N mirrors these fields 1:1." The `0004` columns and these Pydantic fields must match name-for-name and type-intent-for-type (TEXT↔str, INTEGER↔int/bool, `TEXT DEFAULT '{}'`↔a JSON-string field, never `dict[str, Any]` in a public field — see `AuditEvent.data: str = "{}"`, lines 110 + 80-86).
- **Enums use `Literal[...]`**, not Python `Enum` (see `status` / `event_type` / `artifact_type`). Map the status vocabulary (`available`/`offline`/...) and `auth_status` vocabulary (`auth_present`/`needs_api_key`/`needs_login`/`rate_limited`/...) to `Literal` unions.
- **Datetimes are `datetime.datetime` fields with a `@field_serializer` returning `.isoformat()`** so `model_dump()` is JSON-safe (D-013). Apply to `first_seen`/`last_seen`/`deactivated_at`/`last_checked`/`updated_at`. Note `deactivated_at`/`finished_at`-style nullable timestamps use `Optional[datetime.datetime] = None` (see `Run.finished_at`, line 72).
- **Default-factory IDs:** `id: str = Field(default_factory=lambda: str(uuid.uuid4()))` — but note the v2 registry uses a NATURAL composite key (`model_id`+`provider_id`+`source`), so `ModelV2` should NOT auto-generate a UUID id; its identity is the composite. `Provider.provider_id` and `RoutePolicy.task_class` are likewise natural string keys, not UUIDs.
- **JSON blob fields** (`api_modes_json`, `metadata_json`, `fallback_policy_json`): per D-013, keep them as `str` (a JSON string, default `"{}"`) to avoid `dict[str, Any]` in the public schema — mirrors `AuditEvent.data` / `ToolCall.args`.

**Barrel export** (from `schemas/__init__.py`, lines 3-23): re-export the new models from `atlas_core.schemas` and add them to `__all__`, matching the existing alphabetical re-export block.

**Reuse, don't re-create:** `SECRET_PATTERNS` (`core.py`, lines 25-29) is the existing redaction primitive for SEC-01 — the auth-store / adapter designs reference it; do NOT define a new regex set.

**LANDMINE that changes these models:** the **credential boundary** (CONTEXT Area 1/3) — `Provider`/`ModelV2` must NOT carry secret material. `auth_status` is a DERIVED `Literal` field computed at discovery time from the file auth store, never a stored secret. Only `provider_id` + derived status live in the registry models. (RESEARCH Area 1 + Area 3.)

### UPSERT / discovery write path (service, CRUD) — reference only, owned by 10.3

**Analog:** `services/agent-runtime/atlas_runtime/model_registry.py` (`ensure_schema()` lines 61-63, `refresh()` lines 102-172).

10.0 does NOT write this code. The analog matters because the `0004` DDL must be consistent with how 10.3 will evolve this module. Two facts to carry into the design docs:
- **`ensure_schema()` parity convention** (lines 61-63): the Python module embeds a `_SCHEMA` string identical to the migration DDL and applies it idempotently — `0004`'s tables get a parallel `ensure_schema()` in the v2 discovery module (10.3). The migration and the Python `_SCHEMA` must stay byte-aligned.
- **Source-scoped deactivation precedent** (lines 143-166): v1 already scopes deactivation by `source` (`WHERE source=? AND active=1`) and soft-deletes (`active=0`, "deactivated, not deleted, so routing history stays resolvable"). v2 replaces `active=0` with a `deactivated_at` timestamp but keeps the source-scoping invariant (MOD-04). The v1 read-diff-then-write three-pass (lines 132-166) is the pattern UPSERT replaces.
