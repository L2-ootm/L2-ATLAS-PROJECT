# Phase 10.7: Web Agent Surface & Permission Queue UX - Context

**Gathered:** 2026-06-29
**Status:** Ready for planning
**Mode:** Autonomous defaults approved by operator

<domain>
## Phase Boundary

Make the existing React Console the canonical WebUI client of ATLAS surface sessions, normalized
events, configuration, projects, models, and surface-owned permissions. Extend the existing
ATLAS policy/broker into one operator-configurable authority shared by all currently usable
surfaces. Deliver the conditional Web session header and owned queue sidebar. Do not build the
future desktop shell or make the Phase 10.8 cutover decision.

</domain>

<decisions>
## Implementation Decisions

### Policy composition

- **D-01:** Keep `permission_broker` plus the frozen ATLAS contracts as the authoritative
  persisted lifecycle; adapt Hermes guard concepts into ATLAS-owned modules rather than running
  `tools/approval.py` as a second authority.
- **D-02:** Evaluate policy in fixed precedence: hardline deny → explicit master deny → master
  ceiling/preset → narrower surface/workspace/project/agent profile → scoped allow → ask/allow.
  A later layer can never overturn an earlier deny.
- **D-03:** Model policy as versioned, JSON-stable data: master preset plus ordered rules with
  typed selectors and deterministic explain output. Do not embed executable callbacks or
  surface UI concerns in policy records.
- **D-04:** Support manual, smart, and full-autonomy presets. Smart mode may advise only; the
  deterministic hardline/path/tool rules remain authoritative and failures/timeouts deny.
- **D-05:** Full autonomy means "execute every non-hardline action not denied by a matching
  rule." It never means bypassing audit, path validation, secret redaction, cancellation, or
  exactly-once execution.

### Scope, maintenance, and approval lifecycle

- **D-06:** Surface/workspace/project/agent/channel profiles only narrow the global master
  policy. Invalid widening is rejected at config validation time with a typed conflict.
- **D-07:** Workspace-only profiles deny paths outside the canonical active root. ATLAS
  installation/update/non-secret config roots are a separate protected maintenance scope,
  reachable only for explicit user-directed maintenance actions.
- **D-08:** Reuse the existing broker's immutable ownership anchor, nonce, expiry, atomic claim,
  allow-once/session/durable scoped rules, restart recovery, and read-only cross-surface outcome
  projections.
- **D-09:** Add Hermes-quality blocking semantics at the adapter boundary: one-second bounded
  waits/heartbeats for live channels, clean cancellation, and timeout-to-deny. Do not block the
  Rust async runtime thread or introduce a message bus.
- **D-10:** Every current surface uses the shared non-visual decision contract. CLI, Go TUI,
  WebUI, API/headless, and enabled messaging adapters can choose different presentations, but
  they cannot implement their own risk or policy semantics.

### Canonical Web surface

- **D-11:** Replace the React Console's `/v1/console/stream` path with shared session/run/event
  APIs. Keep the existing Console route and workbench identity; do not add a parallel Agent page.
- **D-12:** Introduce a shell-level session coordinator/context so conditional chrome survives
  route composition and reconnect. Keep server state in API/session hooks and transient visual
  state in React; do not add Redux or another state framework.
- **D-13:** The compact agent header renders only for active, suspended, resuming, cancelling, or
  reconnectable sessions. It shows workspace, agent/model, permission preset, lifecycle, queue
  count, and cancel/resume affordances without duplicating full System/Models pages.
- **D-14:** The right queue sidebar opens only for actionable items owned by the matching WebUI
  surface session or when explicitly pinned. Foreign pending approvals never render as
  actionable; terminal foreign outcomes remain available only through read-only audit history.
- **D-15:** On narrow viewports the right sidebar becomes an inert modal sheet with focus trap and
  restoration; on wide viewports it is a fixed-width obsidian slab that changes content width
  without overlaying the transcript.

### Queue interaction and visual language

- **D-16:** Preserve the existing L2 Topographic cockpit language: dark-only void, liquid-glass
  slabs, 2px radii, compact HUD labels, JetBrains Mono data, Lucide 1.5px icons, and semantic
  glow. Reuse existing `GlassPanel`, `TopoField`, HUD, and CSS tokens.
- **D-17:** Treat visual effects as state signals only. Pending uses amber, active/streaming uses
  cyan, approved/executed uses green, rejected/failed/hardline uses crimson. No decorative
  animation; all motion stops under reduced-motion.
- **D-18:** Queue rows expose risk, action/tool, redacted normalized arguments, workspace,
  requester/provenance, expiry, and decision scope. Destructive actions require an explicit
  selection and never make the dangerous choice the default focused action.
- **D-19:** Keyboard order is header → queue toggle → queue items → decision controls. Opening
  the sidebar moves focus to its heading/first actionable item; resolving/closing restores focus
  to the originating control and announces the terminal result through a polite live region.

### Migration, verification, and budgets

- **D-20:** Introduce shared gateway/session endpoints before switching the Console. Keep the
  legacy console endpoint temporarily available only as a hidden rollback seam until Phase 10.8;
  new Web code must not call it.
- **D-21:** Build test-first at each contract boundary: policy matrix and hardline corpus,
  broker lifecycle/concurrency, Rust gateway API/contract tests, TypeScript API/hooks, React
  visibility/accessibility, and Go/CLI adapter conformance.
- **D-22:** Add no production framework, ORM, message bus, or permission store. Prefer frozen
  Pydantic models, Rust DTOs/handlers, existing SQLite tables/migrations, standard React hooks,
  and existing test libraries.
- **D-23:** Capture gateway startup/idle memory and Web bundle baselines before implementation;
  reject unexplained regressions greater than 5%. Shared policy reads remain local and
  network-free.
- **D-24:** Preserve provenance for adapted Hermes mechanisms and test behavior, while all
  runtime symbols, config keys, API shapes, and UI copy remain ATLAS-owned.
- **D-25:** Keep Phase 10.7 as one traceable product phase but execute it through four gated
  waves: (1) freeze policy/session/permission contracts and implement the policy core; (2) expose
  gateway endpoints and converge non-visual surface adapters; (3) migrate the React Console and
  add conditional chrome; (4) close accessibility, conformance, provenance, and budget gates.
  No frontend migration begins before the Wave 1 contract fixtures are frozen.

### the agent's Discretion

- Exact module/file boundaries, hook names, endpoint paths, and internal rule-indexing strategy.
- Exact sidebar width and responsive breakpoint within existing cockpit layout conventions.
- Exact smart-mode evaluator interface, provided deterministic policy remains authoritative and
  no external call is required for manual/full-autonomy decisions.

</decisions>

<specifics>
## Specific Ideas

- Hermes is the quality floor: one guard core, hardline floor below YOLO/full autonomy,
  once/session/always scopes, blocking channel queues with heartbeats, ContextVar/thread-local
  isolation, timeout-deny, and native channel adapters.
- WebUI is expected to be the richest working surface, but every capability must remain wired
  through non-visual contracts so CLI, TUI, channels, and the future desktop can use it.
- Operators must be able to express "allow everything except X and Y" and "allow everything
  inside this workspace" without losing the immutable catastrophic-action floor.
- Explicit user requests may direct ATLAS to update itself or change its non-secret config
  through a protected maintenance scope.

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets

- `surface_session_service`: frozen session creation, transitions, heartbeat, reconciliation,
  resume, and cancellation boundaries.
- `surface_events`: one audit-derived normalizer and replay cursor shared by surfaces.
- `permission_broker`: persisted channel registration, scoped actionable query, atomic claim,
  scoped allow rules, outcome projection, and orphan reconciliation.
- `config_service` / `/v1/config`: versioned masked control-plane contract and optimistic patch.
- Rust `atlas-gateway`: loopback-only Axum adapter with SSE, dispatch boundaries, API and contract
  test harnesses.
- React `api.ts`, `useRunStream`, `Console`, `Layout`, `GlassPanel`, `TopoField`, HUD components,
  and existing gateway-health patterns.
- Go TUI client and permission pane: existing thin-client adapter and HTTP test patterns.

### Established Patterns

- Frozen Pydantic v2 contracts are schema authority; JSON-safe dumps cross Rust/TypeScript/Go.
- Rust gateway remains a thin, loopback-only adapter over ATLAS runtime/CLI contracts.
- Audit events are the cross-surface visibility mechanism; no pub/sub bus is introduced.
- React uses local hooks and typed API functions, inline/CSS-token styling, and no global state
  framework.
- L2 product surfaces are compact, dark-only, topographic liquid glass with semantic color.

### Integration Points

- Add policy schema/config settings beside the existing frozen config registry.
- Extend tool execution before broker creation/claim so hardline and master/profile decisions
  share one path.
- Add Rust gateway session/permission endpoints and contract tests before Web migration.
- Replace `consoleChatStream` usage in `Console.tsx` with the shared session coordinator.
- Mount conditional header/sidebar from `Layout.tsx` through a shell-level provider.
- Narrow or remove actionable approval controls from `System.tsx`; retain read-only outcomes.
- Update Go/CLI/channel adapters to consume the same decision and scope vocabulary.

</code_context>
