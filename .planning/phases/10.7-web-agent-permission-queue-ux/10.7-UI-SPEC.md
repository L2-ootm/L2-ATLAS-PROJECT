# Phase 10.7 — UI Design Contract

> Canonical Web agent/session chrome and surface-owned permission queue for the existing L2
> ATLAS Topographic cockpit.

## Design Intent

The foreground stays calm while the runtime state remains legible. Agent/session chrome is not
permanent navigation: it materializes only while work is active, reconnectable, awaiting a
decision, or explicitly pinned. The queue is an obsidian control slab, not a notification center.
It shows enough evidence to make a safe decision without forcing the operator into System or
Audit pages.

The memorable interaction is the queue's relationship to the living terrain: pending approval
raises a restrained amber contour signal beneath the queue toggle and owning item; the field
cools immediately after a terminal decision. No other decorative motion is introduced.

## Design System

| Property | Value |

### Reuse Contract

- Reuse `TopoField`, `GlassPanel`, `StatePill`, HUD label treatment, and existing token names.
- Extend `Layout.tsx`; do not create a second page shell.
- Keep `Console.tsx` as the canonical route and visual workbench.
- Use Lucide `Bot`, `ShieldCheck`, `Clock3`, `PanelRight`, `Pin`, `PinOff`, `X`,
  `Check`, `Ban`, `RotateCcw`, and `Square` where their semantics match.
- Do not add shadcn, Radix, Tailwind component registries, animation libraries, or a new icon set.

## Shell Composition

### Wide viewport — `>= 1180px`

```text
┌ LEFT NAV ┬──────────────────── MAIN CONTENT ────────────────────┬ QUEUE ┐
│          │ [conditional 44px agent/session header]              │ 360px │
│          ├───────────────────────────────────────────────────────┤       │
│          │ Console / current route                              │ owned │
│          │                                                       │ items │
└──────────┴───────────────────────────────────────────────────────┴───────┘
```

- Left nav keeps its existing 58px collapsed / 248px expanded widths.
- Session header sits inside `main`, above the route outlet, after any mock-mode banner.
- Queue width is 360px, clamped to 32vw. It is a sibling of `main`, never an overlay.
- Opening/closing the queue animates width and opacity only; the Console reflows.
- Main content keeps a minimum usable width of 620px.

### Compact viewport — `768–1179px`

- Session header remains inline and scroll-sticky at the top of `main`.
- Queue becomes a fixed modal sheet from the right, width `min(420px, calc(100vw - 32px))`.
- A `rgba(3,3,5,.72)` scrim is click-dismissible only when no destructive confirmation is open.
- Background content is inert while the sheet is open.

### Narrow viewport — `< 768px`

- Header becomes two rows: identity/lifecycle first; queue/cancel/resume controls second.
- Queue sheet occupies the viewport below the safe-area top, with 16px outer inset and 8px
  earned radius.
- Queue evidence stacks vertically; decisions remain a fixed bottom action rail.
- The operator can complete every flow without horizontal scrolling.

## Components

### `AgentSessionHeader`

- Hidden for no session and terminal sessions with no reconnect/queue state.
- Visible states: `starting`, `active`, `suspended`, `resuming`, `cancelling`, reconnectable.
- Left cluster: agent icon, `AGENT SESSION`, lifecycle pill, short session ID.
- Center cluster: workspace/project, provider/model, effective permission preset.
- Right cluster: owned queue count, queue toggle, context-sensitive cancel/resume.
- Queue toggle uses `aria-expanded` and `aria-controls`.
- Cancel is never the first focusable control and uses a square stop icon, not a red primary CTA.

### `PermissionQueueSidebar`

- `<aside aria-labelledby="permission-queue-title">`.
- Heading: `PERMISSION QUEUE`, owned pending count, pin toggle, close control in sheet mode.
- Sections: `ACTION REQUIRED`, `QUEUED WORK`, then optional `RECENT OUTCOMES`.
- Empty pinned state remains useful: `NO ACTION REQUIRED` plus current policy preset and scope.
- Unpinned empty state closes automatically after the terminal receipt is announced.

### `PermissionQueueItem`

- Summary row: risk/state signal, tool/action, expiry countdown.
- Evidence grid: workspace, requester/agent, run/session provenance, redacted argument summary.
- Expand control reveals formatted redacted arguments and matched policy explanation.
- Decision group:
  - `ALLOW ONCE`
  - `ALLOW SESSION`
  - `ALLOW SCOPED` only when the server advertises a valid bounded rule
  - `DENY`
- No decision is preselected. `Enter` on the row expands evidence; it does not approve.
- A terminal receipt replaces actions in place, then moves to recent outcomes.

### `PolicyReceipt`

- States the exact outcome and source: `DENIED · HARDLINE RULE H-004`,
  `ALLOWED ONCE · OPERATOR`, `DENIED · TIMEOUT`.
- Shows timestamp, decision surface, and safe remediation link when relevant.
- Receipt is read-only and never presents a retry that could bypass policy.

## State Matrix

| Session | Owned pending | Pinned | Header | Sidebar |
|---|---:|---:|---|---|
| none | 0 | no | hidden | hidden |
| active | 0 | no | visible | hidden |
| active | >0 | no | visible, amber count | open |
| active | 0 | yes | visible | open empty |
| suspended/reconnectable | any | any | visible, cyan/warn | reflects owned queue |
| terminal | 0 | no | hidden after receipt | hidden |
| terminal | 0 | yes | hidden unless recent receipt | open recent/empty |
| foreign surface pending | any | any | unchanged | never actionable |

## Interaction Contract

1. A matching pending approval arrives.
2. Queue toggle receives an amber state signal; sidebar opens unless the operator explicitly
   closed it during this session.
3. Focus moves to the queue heading, then the first actionable item.
4. Operator expands evidence and chooses a decision.
5. Client disables that item's controls while the nonce-bound claim is in flight.
6. Server receipt replaces the controls; a polite live region announces the outcome.
7. Focus moves to the next actionable item, or returns to the queue toggle when empty.
8. An unpinned empty sidebar closes after the receipt remains visible for at least 800ms.

Network/config conflicts keep the item actionable, restore controls, and focus the error summary.
Stale/foreign/already-decided responses become terminal receipts, never silent failures.

## Copywriting Contract

| Element | Copy |
|---|---|
| Header label | `AGENT SESSION` |
| Queue toggle | `PERMISSION QUEUE · {n}` |
| Queue heading | `PERMISSION QUEUE` |
| Empty heading | `NO ACTION REQUIRED` |
| Empty body | `This Web session has no pending decisions.` |
| Loading state | `SYNCING SESSION STATE` |
| Reconnecting | `RECONNECTING · ATTEMPT {n}` |
| Foreign outcome notice | `READ-ONLY OUTCOME · RESOLVED BY {surface}` |
| Timeout receipt | `DENIED · RESPONSE WINDOW EXPIRED` |
| Hardline receipt | `DENIED · HARDLINE SAFETY FLOOR` |
| Conflict error | `DECISION NOT APPLIED · STATE CHANGED. REFRESHED FROM AUTHORITY.` |
| Offline error | `GATEWAY UNAVAILABLE · DECISIONS REMAIN FAIL-CLOSED.` |
| Cancel confirmation | `CANCEL RUN · Active tools and child agents will unwind cooperatively.` |

The interface states; it does not congratulate, apologize, or use exclamation marks.

## Accessibility Contract

- All functionality works with keyboard only.
- Sheet mode uses a real focus trap, `aria-modal="true"`, inert background, Escape close when safe,
  and focus restoration.
- Pending count changes and terminal decisions use one `aria-live="polite"` region.
- Hardline/offline failures use `role="alert"` only when immediate intervention is required.
- Decision controls have visible focus rings at least 2px and meet WCAG AA contrast.
- Countdown text announces at meaningful thresholds only (60s, 30s, 10s, expired), not every
  second.
- Risk/state never relies on color alone.
- `prefers-reduced-motion` disables sliding/parallax/glow transitions; visibility and focus
  behavior remain identical.
- Touch targets are at least 40×40px in sheet mode and 32×32px on dense desktop.

## Motion and Topographic Effects

- Header enters as one 200ms obsidian slab transition; it does not animate child-by-child.
- Sidebar uses a 250ms width/translate transition with the canonical easing.
- Pending terrain glow is caused by queue state and operator presence, then cools on resolution.
- No infinite polling indicator, shimmer, bouncing badge, or decorative loop.
- Existing ambient topography remains one shared animation loop and stops under reduced motion.

## Error and Recovery States

- **Gateway offline:** keep evidence visible, disable decisions, state fail-closed behavior.
- **Session ownership changed:** replace item with read-only terminal/foreign receipt.
- **Nonce stale/already decided:** refresh authority state and show the winning outcome.
- **Config revision conflict:** retain edited values, show current effective revision, require
  explicit reapply.
- **Resume incompatible:** state the exact snapshot/version mismatch and provide a new-session
  action; never silently discard reproducibility.
- **Approval expired:** disable controls and show timeout denial source.

## Performance and Density

- No new production UI dependency.
- Queue polling/stream updates are session-scoped and deduplicated by sequence/revision.
- Keep at most 50 recent outcome rows mounted; paginate or virtualize only after measurement.
- Header/sidebar state updates must not rerender the topographic engine or unrelated routes.
- Web production bundle may not regress more than 5% without a recorded explanation.
