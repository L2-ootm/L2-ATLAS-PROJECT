# Phase 10.7 Verification

## Outcome

WEB-01 through WEB-06 pass. Python, Rust, Go, and React consume the same session, event,
permission, ownership, and receipt contracts. Actionable approvals are surface-owned and
nonce-bound; the global approval projection is read-only.

## Security and conformance

- Frozen policy/event fixtures are consumed by Python, Rust, Go, and React tests.
- A rebuilt release gateway returned 403 for a queue read without the surface owner token.
- The same live session returned 200 for owned queue and event reads and for close.
- The global session-list projection redacts owner tokens; a live probe confirms none are exposed.
- Foreign, stale, nonce-mismatched, timed-out, terminal, and concurrent decision behavior is
  covered by broker, gateway, adapter, and frontend tests.
- Static provenance checks confirm one policy/broker authority, no actionable global mutation,
  no raw secrets or chain-of-thought in surface contracts, and no donor identity in product UI.
- Gen 1 OpenTUI, Gen 2 Rich, legacy Console chat/stream, and their callers/dependencies are gone.
