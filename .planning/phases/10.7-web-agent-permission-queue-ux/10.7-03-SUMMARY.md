# 10.7-03 Summary: Surface and Scoped-Permission CLI Bridge

**Completed:** 2026-06-29  
**Requirements:** WEB-01, WEB-04, WEB-05  
**Status:** Green

## Delivered

- Added `atlas surface` as a thin machine-readable adapter over existing services:
  - create global/project sessions;
  - get/list;
  - owner-bound heartbeat and suspend;
  - resume with owner-token rotation and channel re-registration; and
  - cancel/close with audited terminal lifecycle and channel revocation.
- Session creation resolves registered workspace, configured agent/provider/model, permission mode,
  prompt version, tool-catalog version, and context-policy version through shared contracts.
- Read-only session projections mask owner tokens; create/resume return the current token only to
  the creating/resuming client.
- Added typed JSON errors for invalid workspace, missing session, stale owner, transition conflict,
  and resume incompatibility.
- Migrated `atlas tools` actionable decisions to the broker:
  - guarded calls require surface session, kind, and workspace root;
  - actionable list is strictly owner-scoped;
  - approve/reject require owner session id and nonce;
  - once/session/durable maps to existing broker allow-rule scopes; and
  - the global approvals command remains read-only compatibility visibility.

## Stable JSON Contract Examples
