### Batch 3: Remove legacy console endpoints (after Wave 3)

| Item | Path | Reason |
|------|------|--------|
| Gateway stream endpoint | `/v1/console/stream` in `lib.rs` (handler `console_stream`, line ~1079) | Replaced by surface-session + run-stream |
| Gateway chat endpoint | `/v1/console/chat` in `lib.rs` (handler `console_chat`, line ~1042) | Dead after React migration; CLI can use tools directly |
| Rust handler functions | `console_stream`, `console_chat`, `ConsoleChatBody` in `lib.rs` | Dispatch-only wrappers for removed endpoints |
| React transport | `consoleChatStream` in `api.ts` (line 406, in_degree=1) | Replaced by `AgentSurfaceContext` coordinator |
| React Console.send | `Console.tsx` `send()` function (line 296, calls consoleChatStream) | Replaced by surface-session submit |
| React stream hook | Any hook consuming the old NDJSON stream from `/v1/console/stream` | Replaced by normalized event consumption |
| CLI console chat command | `console_chat` in `cli/main.py` (line 170) | Orphaned if gateway endpoints removed |
| Python console service | `console_service.py` (run_chat, _run_claude_code_chat) | Orphaned; runs agent outside policy chokepoint |

**Verification:** grep for `console/stream`, `consoleChatStream`, `console_chat`, `console_service` in production code, full test suites, `npm run build` green.

**Note:** `console_service.py` runs agent execution directly without going through `tool_service.invoke` (the policy chokepoint). This is not a security issue for chat, but removing it eliminates an orphan code path. The Rust gateway `console_chat` handler also dispatches to `atlas console chat` — a separate CLI path that bypasses the surface-session protocol entirely.

## Cleanup ordering

1. Batch 1 (Gen 1) — can execute immediately, no active dependencies
2. Batch 2 (Gen 2) — can execute immediately, hidden rollback kept separately
3. Batch 3 (console stream) — only after Wave 3 (Plan 05) is verified green
4. Batch 4 (global approvals) — only after Wave 2 (Plan 04) is verified green
5. Batch 5 (dead deps) — after Batches 1 and 2

Each batch is atomic: remove, verify no callers, run full test suites, commit.
