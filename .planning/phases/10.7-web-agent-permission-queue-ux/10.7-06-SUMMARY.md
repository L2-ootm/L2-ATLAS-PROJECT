# Plan 10.7-06 Summary

The React shell now exposes operational agent state through a conditional session header and an
owned permission queue. Inactive sessions stay visually quiet; active and reconnectable sessions
show compact status, model, policy, queue count, and cancellation controls. The queue opens for
matching actionable work or an explicit operator pin, and terminal or foreign approvals never
gain decision controls.

Wide layouts use a 360px complementary sidebar. Narrow layouts use a modal sheet with focus
entry, focus trapping, Escape dismissal, trigger restoration, and inert background content.
Decision rows expose risk, redacted arguments, workspace, requester, expiry, and server-authored
policy receipts. Destructive and durable choices require deliberate confirmation; hardline
denials render no allow action.
