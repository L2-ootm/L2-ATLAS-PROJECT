# Plan 10.0.4-04 — Summary

**Wave:** 4 (CLI + gateway) · **Status:** complete · **Executed:** 2026-06-22 (inline)

## What shipped
The tool layer is now reachable via CLI and HTTP (SC1/SC2/SC4 via CLI; SC3 endpoints for Plan 05).

- **`cli/tools.py`** (`atlas tools` Typer sub-app) — `list`, `manifests`, `call`, `approvals`,
  `approve`, `reject`, each with `--json`. Mirrors `cli/discord.py` (`_get_connection`/`_get_lock`
  monkeypatch seams). `call` routes through `tool_service.invoke`; the exit-code convention is
  preserved (processed outcome → exit 0 so JSON reaches the gateway; unknown tool / bad approval id →
  stderr + exit 1).
- **`cli/main.py`** — registered `app.add_typer(tools_app, name="tools")` next to discord.
- **`tests/cli/test_tools_cli.py`** — CliRunner with `_get_connection` monkeypatched to an injected
  temp DB (never the live path). 5 tests: manifests/list JSON, write-tool short-circuit to pending,
  unknown-tool non-zero exit, reject round-trip.
- **`native/atlas-core-rs/crates/atlas-gateway/src/lib.rs`** — five dispatch-only handlers +
  routes: `GET /v1/tools/manifests`, `POST /v1/tools/calls`, `GET /v1/tools/approvals` (optional
  `?status`), `POST /v1/tools/approvals/{id}/approve`, `.../reject`. `require_arg` guards tool name +
  approval id (empty → 400); user values passed AFTER `--`; **no DB/state/policy in Rust** (D-022).
- **`tests/api.rs`** — `tool_call_empty_tool_is_rejected` (400) + `tool_manifests_returns_cli_json`
  (stub dispatch).
