**Mapped:** 2026-06-22

## Orientation

This phase is ~80% mechanical generalization of the reviewed Phase C `discord_*` pipeline into a tool-agnostic `tool_*` pipeline, plus a `ToolManifest` contract and four stdlib/`gh`-only adapter bodies. The dominant rule: **clone the Phase C shape verbatim** (state machine, redaction-once, atomic TOCTOU claim, operator-run FK bootstrap, dispatch-only gateway, CLI `--json`, cockpit ApprovalsPanel). The only genuinely novel surface is `ToolManifest`/`ToolResult`, the YAML registry, `policy.decide()`, and the four adapter bodies.

## Pattern Assignments

### `packages/atlas-core/atlas_core/schemas/tool.py` (model, contract)

**Analog:** `packages/atlas-core/atlas_core/schemas/discord.py` (frozen-model + Literal taxonomy + JSON-string field + `__all__` export) and `core.py` (`AuditEvent`/`ToolCall` field-serializer pattern).

Why closest: `discord.py` is the most recent frozen-schema module added for an approval pipeline. It already demonstrates (a) a `Literal` action/status taxonomy, (b) `model_config = ConfigDict(frozen=True, str_strip_whitespace=True)`, (c) a JSON-string `params` field documented as secret-redacted, (d) ISO-8601 `field_serializer` for datetimes, (e) `id` default-factory uuid, (f) `run_id: str = "operator"`, and (g) an `__all__` list. The new `ToolManifest`/`ToolResult`/`ToolApproval` should mirror this exactly.

**Frozen-model + Literal taxonomy** (`discord.py:25-64`):
```python
DiscordAction = Literal["create_channel", "edit_channel", ...]
DiscordApprovalStatus = Literal["pending", "executing", "executed", "rejected", "failed"]

class DiscordApproval(BaseModel):
    model_config = ConfigDict(frozen=True, str_strip_whitespace=True)
    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    action: DiscordAction
    params: str = "{}"  # JSON string, secret-redacted before persistence (D-013)
    status: DiscordApprovalStatus = "pending"
    run_id: str = "operator"
    requested_at: datetime.datetime = Field(default_factory=lambda: datetime.datetime.now(datetime.timezone.utc))
    decided_at: Optional[datetime.datetime] = None

    @field_serializer("requested_at", "decided_at")
    def serialize_dt(self, dt): return None if dt is None else dt.isoformat()
```

For the new file:
- `ToolRiskLevel = Literal["read", "write", "shell"]` (maps 1:1 to SC1 read=auto-allow).
- `ToolApprovalStatus` reuse the exact 5-value Literal from `DiscordApprovalStatus` (`pending|executing|executed|rejected|failed`) so the TOCTOU claim and terminal states carry over unchanged.
- `ToolManifest` fields: `name`, `description`, `risk_level: ToolRiskLevel`, `permissions: list[str]`, `inputs: dict[str,str]`, `outputs: dict[str,str]`, `audit_events: list[str]` (default the 3 verbs). All frozen.
- `ToolResult` fields: `tool_name`, `ok: bool`, `output: str` (JSON string, D-013), `error: str|None`, `exit_code: int|None`. Frozen.
- `ToolApproval` mirrors `DiscordApproval` column-for-column against the 0013 DDL (`id, tool_name, risk_level, args, summary, status, reason, result, run_id, requested_at, decided_at`).
- Add an `__all__` list as in `discord.py:67-71`.

**Pitfall (Pitfall 5):** all these are `frozen=True` — never mutate; construct new instances.

### `services/agent-runtime/atlas_runtime/tools/registry.py` (service, transform)

**Analog:** `atlas_runtime/agents/registry.py` (the `_REGISTRY` dict + `get_*(name)`/`known_*()` resolve-or-raise pattern) plus `config_service.load_config` for `yaml.safe_load`.

Why closest: `agents/registry.py` is the canonical "resolve a thing by name, raise `ValueError` on unknown" registry in the runtime. The new tool registry adds YAML manifest loading on top of that exact shape.

**Resolve-or-raise pattern** (`agents/registry.py:8-31`):
```python
_REGISTRY: dict[str, type[AgentRuntime]] = {
    NativeAtlasAgent.name: NativeAtlasAgent,
    ClaudeCodeAgent.name: ClaudeCodeAgent,
}

def get_agent(name: str) -> AgentRuntime:
    try:
        cls = _REGISTRY[name]
    except KeyError:
        raise ValueError(f"Unknown agent runtime {name!r}. Known: {known_agents()}") from None
    return cls()

def known_agents() -> list[str]:
    return sorted(_REGISTRY)
```

For the new registry: eager-load `manifests/*.yaml` via `yaml.safe_load`, validate each with `ToolManifest.model_validate(...)` (a `ValidationError` is a fail-fast load error per CONTEXT), build a `{name: (manifest, adapter)}` map, and expose `resolve(name) -> (ToolManifest, adapter)` that raises `ValueError` on unknown — same shape as `get_agent`. Bind each manifest `name` to its adapter module's `run` callable. The github adapter must NOT crash registry load if `gh` is absent (handle at call time, returning `ToolResult(ok=False, ...)`).

### `tools/adapters/web_fetch.py` and `webhook_notify.py` (adapter, HTTP)

**Analog:** `services/agent-runtime/atlas_runtime/discord_api.py:29-56` — the stdlib `urllib.request` HTTP client with `HTTPError` handling. This is the proven anti-bloat HTTP pattern (no `requests`/`httpx`).

`web_fetch` adds the SSRF guard from RESEARCH Pattern 4: `urlparse` scheme allowlist (`http`/`https` only), block `localhost`/`0.0.0.0`/`::1`, resolve via `socket.getaddrinfo` and reject `is_private`/`is_loopback`/`is_link_local`/`is_reserved`/`is_multicast`, plus a size cap (`_MAX_BYTES`) and timeout. `webhook_notify` is write-class (approval-gated) and reuses the same SSRF guard for POST. Document the DNS-rebinding TOCTOU as a known v0 limitation; do not over-engineer.

### `services/agent-runtime/atlas_runtime/tool_service.py` (service, state machine) — PRIMARY ANALOG

**Analog:** `services/agent-runtime/atlas_runtime/discord_service.py` (the entire propose/approve/reject/_set_terminal state machine). Clone this 1:1, renaming `discord_*` → `tool_*`.

Why closest: this is the reviewed, concurrency-correct, audit-first template the whole phase generalizes. Copy verbatim: the redact-once discipline, the atomic TOCTOU claim, the operator-run FK bootstrap, the `_load`/`_row_to_approval`/`_set_terminal` helpers, and the custom `*ApprovalError(ValueError)`.

**(1) Redact ONCE at propose/invoke** (`discord_service.py:126-127`):
```python
# Redact ONCE: stored + audited + executed copy are all this redacted JSON.
params_json = _redact(json.dumps(params))
```
For tools: `args_json = _redact(json.dumps(args or {}))`. Store/audit/execute that single string. Never redact a dict in place; never add a second redact path (Pitfall 3 / anti-pattern). `_redact` is round-trip-safe (`audit_service.py:36-55`).

**(2) Operator-run FK bootstrap** (`discord_service.py:138`, `mission_service.py:23-60`):
```python
run_id = mission_service.ensure_operator_run(conn, lock)   # returns "operator"
```
`audit_events.run_id` is `NOT NULL REFERENCES runs(id)`; this idempotently inserts the synthetic operator mission+run (Pitfall 4). Call before any `emit`.

**(3) Atomic TOCTOU claim on approve** (`discord_service.py:248-260`) — copy verbatim:
```python
with lock:
    with conn:
        cur = conn.execute(
            "UPDATE tool_approvals SET status = 'executing' "
            "WHERE id = ? AND status = 'pending'",
            (approval_id,),
        )
        claimed = cur.rowcount == 1
if not claimed:
    current = _load(conn, approval_id)
    raise ToolApprovalError(f"approval {approval_id!r} is {current.status!r}, not pending")
```
Two concurrent approvers: only the one whose UPDATE affects exactly one row executes.

**(4) Terminal write + audit on success/failure** (`discord_service.py:264-290`):
```python
try:
    result = _execute(approval, audit_reason)
except discord_api.DiscordSidecarError as exc:
    result_json = json.dumps({"error": str(exc)})
    _set_terminal(conn, lock, approval_id, "failed", result_json, now)
    emit(conn, lock, run_id=run_id, event_type="failure", ...)
    return _load(conn, approval_id)
result_json = _redact(json.dumps(result))
_set_terminal(conn, lock, approval_id, "executed", result_json, now)
emit(conn, lock, run_id=run_id, event_type="discord_action", ...)
```
For tools, map to the three verbs: emit `tool_requested` at the start of `invoke`; on the read-class path or post-approve run, emit `tool_completed` (with `tool_call_kwargs` for the `ToolCall` row) or `tool_failed` on adapter exception. See RESEARCH Pattern 3 for the exact `invoke()` skeleton.

**(5) `_set_terminal` helper** (`discord_service.py:329-343`) — copy as `tool_service._set_terminal`.

**The new `invoke()` chokepoint** (no analog — novel, but built from the pieces above): emit `tool_requested` → `policy.decide(manifest, mode)` → if `requires_approval` INSERT a pending `tool_approvals` row + emit `event_type="approval"` and return the `ToolApproval`; else run the adapter now and emit `tool_completed`/`tool_failed`. This is the single policy chokepoint.

async fn discord_propose(State(state): State<AppState>, Json(body): Json<DiscordWriteBody>) -> ApiResult {

    require_arg(&body.guild, "guild must be non-empty")?;
