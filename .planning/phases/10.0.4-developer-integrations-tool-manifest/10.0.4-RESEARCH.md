## Summary

This phase generalizes the proven Phase C Discord write-surface into a tool-agnostic, manifest-driven
integration harness. The entire end-to-end template already exists and works: `discord_service.propose/
approve/reject/list` (the two-phase audit-first state machine), `0012_discord_approvals.sql` (additive
migration), the `atlas discord propose|approvals|approve|reject` CLI group, the dispatch-only gateway
routes in `lib.rs`, and the cockpit `ApprovalsPanel` Pending-Approvals UI in `Discord.tsx`. The work is to
copy that shape into a `tool_*` equivalent and add four stdlib/`gh`-only adapters behind a `ToolManifest`
contract.

Two questions in the research brief resolve cleanly against the code: **(1) PyYAML is already available**
(`config_service.py` does `import yaml`; `python -c "import yaml"` reports `yaml 6.0.3` in the runtime venv),
so no new dependency is incurred by loading manifests — but it is a *transitive* dep, not declared in
`services/agent-runtime/pyproject.toml`, so the plan should add `pyyaml` to that file's `dependencies` to make
the manifest loader's import explicit and supply-chain-honest. **(2) `gh` 2.92.0 is installed** at
`C:\Program Files\GitHub CLI\gh` and is the correct GitHub path — it carries the user's existing auth, so
ATLAS handles no token (D-002 satisfied trivially).

The single largest risk is **redaction-ordering and the operator run_id FK** — both already solved in
Phase C and must be copied verbatim, not re-invented. The second risk is the **ATLAS_HOME-vs-real-DB CLI
gotcha** (in project memory): the discord CLI hardcodes `~/.atlas/atlas.db` in its own `_get_connection`,
bypassing `ATLAS_HOME` — tests must inject a temp DB, never exercise the live CLI.

**Primary recommendation:** Clone the Phase C pipeline 1:1 into `tool_service.py` + `tools/` package +
`0013_tool_approvals.sql` + `atlas tools` CLI + `/v1/tools/*` gateway routes + a System-page POLICY/TOOLS/
APPROVALS surface. Add `pyyaml` to runtime pyproject. Use only stdlib (`urllib`, `subprocess`, `pathlib`)
plus the `gh` CLI for the four adapters.

### Pattern 1: Frozen ToolManifest in atlas-core, validated at load
**What:** A `ToolManifest` Pydantic model (frozen, D-012/D-013) is the contract; `tool_registry` reads each
`manifests/*.yaml` via `yaml.safe_load`, calls `ToolManifest.model_validate(data)`, and a `ValidationError`
is a fail-fast load error (CONTEXT decision).
**When to use:** At registry construction (module import / first access), mirroring `agents/registry.py`'s
eager `_REGISTRY` dict.
**Example:**
```python
# packages/atlas-core/atlas_core/schemas/tool.py
# Source pattern: core.py lines 240-307 (frozen models), config_service.py (yaml.safe_load)
from typing import Literal
from pydantic import BaseModel, ConfigDict, Field

ToolRiskLevel = Literal["read", "write", "shell"]  # maps 1:1 to SC1 (read=default-allow)

class ToolManifest(BaseModel):
    model_config = ConfigDict(frozen=True, str_strip_whitespace=True)
    name: str
    description: str = ""
    risk_level: ToolRiskLevel
    permissions: list[str] = Field(default_factory=list)   # e.g. ["fs:read", "net:get"]
    inputs: dict[str, str] = Field(default_factory=dict)    # arg_name -> type/desc
    outputs: dict[str, str] = Field(default_factory=dict)
    audit_events: list[str] = Field(default_factory=lambda: ["tool_requested","tool_completed","tool_failed"])

class ToolResult(BaseModel):
    model_config = ConfigDict(frozen=True)
    tool_name: str
    ok: bool
    output: str = ""          # JSON string (D-013) — redacted at the audit boundary
    error: str | None = None
    exit_code: int | None = None
```
```python
# services/agent-runtime/atlas_runtime/tools/registry.py — mirror agents/registry.py lines 8-31
import pathlib, yaml
from atlas_core.schemas.tool import ToolManifest
_MANIFEST_DIR = pathlib.Path(__file__).resolve().parent / "manifests"

def _load_manifests() -> dict[str, ToolManifest]:
    out: dict[str, ToolManifest] = {}
    for p in sorted(_MANIFEST_DIR.glob("*.yaml")):
        m = ToolManifest.model_validate(yaml.safe_load(p.read_text(encoding="utf-8")) or {})
        out[m.name] = m          # fail-fast: ValidationError propagates
    return out
```

### Pattern 2: policy.decide() — the read/write/shell → allow/deny/needs-approval mapping
**What:** Extend `PolicyDecision` with `requires_approval: bool` (the `ToolCall` docstring at core.py:284-285
already promises it) and add an aggregator. `read` is allowed when global mode is read-only or write; `write`/
`shell` require approval.
**Example:**
```python
# services/agent-runtime/atlas_runtime/policy.py — EXTEND existing dataclass (lines 25-35)
@dataclass
class PolicyDecision:
    allowed: bool
    reason: str
    requires_approval: bool = False     # NEW (back-compat default)

def decide(manifest, mode: str = "read_only") -> PolicyDecision:
    """mode: 'read_only' (global default) | 'unrestricted'. risk read=allow now;
    write/shell=needs-approval (never auto-run). SC1."""
    if manifest.risk_level == "read":
        return PolicyDecision(True, "read_class_allowed", requires_approval=False)
    return PolicyDecision(False, f"{manifest.risk_level}_requires_approval", requires_approval=True)
```

### Pattern 3: tool_service.invoke() — the single chokepoint with the three audit verbs
**What:** Every tool call flows through `invoke()`. It emits `tool_requested`, calls `policy.decide`, and
either (a) runs the adapter and emits `tool_completed`/`tool_failed`, or (b) short-circuits to a pending
`tool_approvals` row. `approve()` runs the deferred adapter and emits `tool_completed`/`tool_failed`. Copy
the redaction/run_id/atomic-claim discipline from `discord_service` verbatim.
**Example (skeleton — mirror discord_service.py propose/approve):**
```python
# services/agent-runtime/atlas_runtime/tool_service.py
# Source pattern: discord_service.py propose (112-167), approve (229-290), _set_terminal (329-343)
from atlas_runtime import mission_service
from atlas_runtime.audit_service import _redact, emit
from atlas_runtime import policy
from atlas_runtime.tools import registry

def invoke(conn, lock, *, tool_name, args, mode="read_only", reason=None):
    manifest, adapter = registry.resolve(tool_name)       # raises ValueError on unknown
    run_id = mission_service.ensure_operator_run(conn, lock)   # operator run_id FK bootstrap
    args_json = _redact(json.dumps(args or {}))           # redact ONCE (Phase C discipline)
    emit(conn, lock, run_id=run_id, event_type="tool_requested",
         tool_name=tool_name, data={"tool": tool_name, "args": json.loads(args_json)})
    decision = policy.decide(manifest, mode)
    if decision.requires_approval:
        # INSERT pending tool_approvals row; emit event_type="approval"; return ToolApproval
        ...
        return approval
    # read-class: run now, emit completed/failed with tool_call_kwargs for the ToolCall row
    try:
        result = adapter.run(json.loads(args_json), ctx={"conn": conn, "run_id": run_id})
        emit(conn, lock, run_id=run_id, event_type="tool_completed", tool_name=tool_name,
             data={"tool": tool_name}, policy_result=decision.reason,
             tool_call_kwargs={"tool_name": tool_name, "args": args_json,
                               "result": result.output, "policy_allowed": True,
                               "requires_approval": False})
        return result
    except Exception as exc:
        emit(conn, lock, run_id=run_id, event_type="tool_failed", tool_name=tool_name,
             data={"tool": tool_name, "error": str(exc)}, policy_result="tool_failed")
        raise
```

### Pattern 4: web_fetch SSRF guard (stdlib only)
```python
# services/agent-runtime/atlas_runtime/tools/adapters/web_fetch.py
import ipaddress, socket, urllib.request
from urllib.parse import urlparse
_MAX_BYTES = 1_000_000          # size cap
_TIMEOUT = 10.0

def _assert_safe(url: str) -> None:
    u = urlparse(url)
    if u.scheme not in ("http", "https"):
        raise ValueError(f"web_fetch: scheme {u.scheme!r} not allowed (http/https only)")
    host = u.hostname or ""
    if host in ("localhost", "0.0.0.0", "::1"):
        raise ValueError("web_fetch: localhost blocked")
    # Resolve and reject RFC1918 / loopback / link-local / reserved (SSRF guard)
    for fam, _, _, _, sa in socket.getaddrinfo(host, None):
        ip = ipaddress.ip_address(sa[0])
        if ip.is_private or ip.is_loopback or ip.is_link_local or ip.is_reserved or ip.is_multicast:
            raise ValueError(f"web_fetch: target IP {ip} is non-public (SSRF blocked)")

def run(args, ctx):
    url = args["url"]; _assert_safe(url)
    req = urllib.request.Request(url, method="GET", headers={"User-Agent": "atlas-web-fetch/0"})
    with urllib.request.urlopen(req, timeout=_TIMEOUT) as resp:
        body = resp.read(_MAX_BYTES + 1)
        if len(body) > _MAX_BYTES:
            raise ValueError("web_fetch: response exceeds size cap")
        return ToolResult(tool_name="web_fetch", ok=True, output=body.decode("utf-8","replace"))
```
> Note the TOCTOU caveat: DNS can rebind between `_assert_safe` and `urlopen`. For v0 this guard is
> acceptable (read-class, operator-driven). Document it as a known limitation; a hardened version pins the
> resolved IP into the connection. Do not over-engineer for v0.

### Anti-Patterns to Avoid
- **Re-litigating redaction**: do NOT add a second redact path. Reuse `audit_service._redact` and redact
  args ONCE at the boundary, exactly as `discord_service.propose` does (line 127). Double-redaction or
  redact-after-execute is the bug class Phase C already closed.
- **State in Rust**: the gateway must stay dispatch-only (D-022). No `tool_approvals` reads/writes in Rust —
  it shells to `atlas tools ... --json` and returns parsed JSON, like `discord_propose` (lib.rs 683-717).
- **Adapter doing its own policy check**: policy lives ONLY in `tool_service.invoke` via `policy.decide`.
  Adapters assume they have been authorized.
- **Mutating frozen models**: `PolicyDecision` is a dataclass (mutable, fine to extend), but `ToolManifest`/
  `ToolResult`/`AuditEvent` are `frozen=True` — build a new instance, never set an attribute.

## Common Pitfalls

### Pitfall 1: Windows console flash on subprocess spawn
**What goes wrong:** A bare `subprocess.run(["gh", ...])` on Windows can flash a console window per call.
**Why it happens:** Without creation flags, GUI-spawned children get a console.
**How to avoid:** Short-lived, non-detached calls (`subprocess.run` with `capture_output=True`) generally do
NOT flash because no new console is allocated when stdout/stderr are pipes — but to be safe and match repo
convention, pass `creationflags=subprocess.CREATE_NO_WINDOW` on `os.name == "nt"` for `gh`/webhook calls. Do
NOT use the full detached triad (`DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP`) used by `discord_control.start`
(lines 164-167) — that is for long-lived daemons; for capture-and-wait you only want `CREATE_NO_WINDOW`.
**Warning signs:** A black window blinks during `atlas tools call github ...` on Windows.
```python
def _no_window_flags():
    import os, subprocess
    return subprocess.CREATE_NO_WINDOW if os.name == "nt" else 0  # type: ignore[attr-defined]
```

### Pitfall 2: ATLAS_HOME-vs-real-DB CLI gotcha (from project memory)
**What goes wrong:** Tests that exercise the live `atlas tools` CLI hit the real `~/.atlas/atlas.db`, not an
isolated temp DB, even with `ATLAS_HOME` set.
**Why it happens:** `cli/discord.py::_get_connection` (lines 28-35) hardcodes `pathlib.Path.home()/".atlas"/
"atlas.db"` — it is NOT `ATLAS_HOME`-aware. A copied `cli/tools.py` will inherit the same gotcha.
**How to avoid:** Test `tool_service`/`tool_registry` directly with an injected temp `sqlite3` connection
(via `db.connect(tmp_path/"t.db")` + `db.apply_migrations`), never through the CLI's `_get_connection`. The
CLI exposes `_get_lock` (line 38) as a monkeypatch seam — mirror that for the connection in tests.
**Warning signs:** Test pollution; rows appearing in your real `~/.atlas/atlas.db`.

### Pitfall 3: Redaction ordering / round-trip safety
**What goes wrong:** Storing un-redacted args, or redacting after JSON parse such that `json.loads` fails.
**Why it happens:** `_redact` replaces JSON values with `"[REDACTED]"` (a valid JSON string literal) for the
key pattern, but `[REDACTED]` (unquoted) for querystring/Bearer patterns (audit_service.py 46-55).
**How to avoid:** Redact the JSON *string* once via `_redact(json.dumps(args))` exactly like
`discord_service.propose` line 127; store that string; the audit `emit` will redact again harmlessly (it is
idempotent on already-redacted text). Never redact a dict in place.

### Pitfall 4: FK integrity for operator-initiated tool calls
**What goes wrong:** Emitting an audit event for a tool call on a fresh DB fails the
`audit_events.run_id REFERENCES runs(id)` FK.
**Why it happens:** Operator tool calls carry no real run.
**How to avoid:** Call `mission_service.ensure_operator_run(conn, lock)` first (returns the synthetic
`OPERATOR_RUN_ID`); use its return as `run_id` for both the row and the emit — exactly as Phase C does.

### Pitfall 5: Frozen-model immutability
**What goes wrong:** `manifest.risk_level = "read"` raises `ValidationError`/`TypeError` at runtime.
**Why it happens:** All atlas-core schemas are `frozen=True` (core.py 248).
**How to avoid:** Treat `ToolManifest`/`ToolResult`/`AuditEvent` as read-only; construct new instances.
`PolicyDecision` is a plain dataclass (mutable) — safe to extend with a field.

### Pitfall 6: Dotted vs snake_case event names (SC traceability)
**What goes wrong:** SC4 says `tool.requested`; the enum must be `tool_requested` to match
`tool_call`/`discord_action` convention and pass the Literal.
**How to avoid:** Use snake_case enum values; document in the manifest `audit_events` field and in the
phase docs that the dotted form is the conceptual/external label (CONTEXT specifics). State this explicitly
so `/gsd-verify-work` traces the SC unambiguously.

#[derive(Deserialize)]
struct ToolCallBody { tool: String, args: Option<Value>, mode: Option<String>, reason: Option<String> }

async fn tool_call(State(state): State<AppState>, Json(body): Json<ToolCallBody>) -> ApiResult {
    require_arg(&body.tool, "tool must be non-empty")?;
    let mut args: Vec<String> = vec!["tools".into(), "call".into(), "--json".into()];
    if let Some(m) = &body.mode { args.push("--mode".into()); args.push(m.clone()); }
    if let Some(a) = &body.args { args.push("--args".into()); args.push(a.to_string()); }
    args.push("--".into()); args.push(body.tool.clone());
    let arg_refs: Vec<&str> = args.iter().map(String::as_str).collect();
    let out = dispatch_atlas(&state.atlas_cmd, &arg_refs).await?;
    Ok(Json(serde_json::from_str::<Value>(&out)
        .map_err(|e| ApiError::Internal(format!("tool call parse failed: {e}")))?))
}
