## What shipped
The four integration adapters (SC1) + their manifests (SC2), stdlib + `gh` only — no new deps.

- **`tools/` package** — `tools/__init__.py`, `tools/adapters/__init__.py`, `tools/manifests/__init__.py`.
- **`tools/adapters/base.py`** — `ToolAdapter` protocol (`run(args, ctx) -> ToolResult`) + `no_window_flags()`
  (Windows CREATE_NO_WINDOW only, not the detached triad — RESEARCH Pitfall 1).
- **`workspace.py`** (read) — read/list/grep, every path gated through `policy.check_workspace_boundary`
  (real signature `target_path, workspace_root`) BEFORE any I/O; escape → ok=False, no read.
- **`github.py`** (read) — argv-vector `gh` calls (never shell=True) for repo_view/issue_list/pr_view;
  gh-absent / non-zero / timeout → ToolResult(ok=False), never raises out.
- **`web_fetch.py`** (read) — `_assert_safe` SSRF guard (non-http(s), localhost, RFC1918/loopback/
  link-local/reserved/multicast/unspecified blocked); `_read_capped` size cap; GET-only; 10s timeout;
  DNS-rebind TOCTOU documented as a v0 limitation.
- **`webhook_notify.py`** (write) — POST via urllib, reuses web_fetch's `_assert_safe` before sending.
- **Four manifests** (`workspace`/`github`/`web_fetch` = read, `webhook_notify` = write), each declaring
  name/risk_level/permissions/inputs/outputs/audit_events and validating against `ToolManifest`.
