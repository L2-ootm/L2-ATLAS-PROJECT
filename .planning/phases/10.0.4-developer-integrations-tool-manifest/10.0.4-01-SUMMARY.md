**Wave:** 1 (contracts) · **Status:** complete · **Executed:** 2026-06-22 (inline)

## What shipped
The interface-first wave — every downstream layer implements against these contracts.

- **`packages/atlas-core/atlas_core/schemas/tool.py`** (new) — frozen Pydantic v2 models mirroring
  `discord.py`: `ToolRiskLevel = Literal["read","write","shell"]`, `ToolApprovalStatus` (reuses the
  exact 5-value Discord set), `ToolInput`, `ToolManifest`, `ToolResult`, `ToolApproval` (column-for-
  column with the future 0013 DDL), all with `__all__`. `ToolApproval.args` is a JSON string
  (redacted before persist); datetimes serialize to ISO.
  - Deviation: `inputs` is `list[ToolInput]` (not the plan's suggested `dict[str,str]`) to match the
    Plan-00 test's `m.inputs[0].name` assertion. CONTEXT left manifest field shapes to discretion.
- **`core.py`** — extended `AuditEvent.event_type` Literal with `tool_requested`/`tool_completed`/
  `tool_failed` (snake_case, like `discord_action`; SC4 dotted form is the external label). No
  migration (event_type is TEXT, A2).
- **`policy.py`** — `PolicyDecision` gained `requires_approval: bool = False` (back-compat); new
  `decide(manifest, mode="read_only")`: read → allowed/no-approval; write|shell → not-allowed/
  requires_approval. Single decision function; adapters never check policy themselves.
- **`services/agent-runtime/pyproject.toml`** — declared `pyyaml>=6,<7` (already resolved 6.0.3;
  declaration only, no install). No other deps (anti-bloat: no requests/PyGithub).
