# Phase 10.2: Agent Contract, Tool Semantics & Context Intelligence - Context

**Gathered:** 2026-06-24
**Status:** Ready for planning
**Source:** Accepted D-023 + existing AI-SPEC and prompt/context audit

<domain>
## Phase Boundary

Freeze and prove the versioned behavior contract used by every future ATLAS
surface: stable ATLAS identity/prompt layers, generated tool capabilities,
structured dynamic context, Brain/wiki retrieval, replay provenance,
compaction/resume invariants, and a deterministic/adversarial evaluation suite.

This phase does not build a second agent, a TUI session protocol, a WebUI
session, surface-scoped permission routing, or a new provider framework.
</domain>

<decisions>
## Implementation Decisions

### Prompt and identity

- D-01: Compile prompts as ordered, versioned layers; L0-L3 are the stable cache prefix.
- D-02: ATLAS identity, policy, evidence, verification, uncertainty, and permission invariants are immutable across provider and surface adapters.
- D-03: Session bootstrap and dynamic context are generated structured envelopes, never model-authored authority.
- D-04: Provider-family compatibility text is isolated from ATLAS product identity and snapshot-tested.

### Tools

- D-05: One frozen `ToolCapability` schema normalizes ATLAS manifests, the Hermes registry, MCP tools, and future surface rendering metadata.
- D-06: The catalog is generated from live registries/manifests; duplicate hand-maintained catalogs are prohibited.
- D-07: Parent/child capabilities only narrow, and unavailable/malformed/disallowed calls normalize to explicit errors.

### Context and retrieval

- D-08: The Brain graph is the retrieval spine; wiki, runs, observations, artifacts, failures, and skills are evidence expansions.
- D-09: Retrieval has explicit confidence, freshness, provenance, budgets, and abstention; irrelevant context is a test failure.
- D-10: Retrieved documents, repository files, tool output, and web content are evidence, not instructions.

### Replay and evaluation

- D-11: Prompt, tool-catalog, context-policy, instruction-source, and selected-source versions are persisted without secrets or hidden reasoning.
- D-12: Deterministic safety gates outrank calibrated LLM judges; no external tracing or general agent/RAG framework enters the shipped runtime.

### the agent's Discretion

- Internal module boundaries and helper names.
- Exact SQLite table split for version/snapshot records, provided schemas remain additive and JSON-stable.
- Exact deterministic scoring weights, provided labeled precision/recall/abstention gates are met and recorded.
</decisions>

<specifics>
## Specific Ideas

- Keep new Python inside the Hermes-derived/LLM adapter contract surface.
- Use frozen Pydantic v2 models and JSON-safe `model_dump()`.
- No new runtime dependency is expected.
- Stable core prompt target: <=4,000 estimated tokens before tool schemas.
- Dynamic context default: <=8,000 estimated tokens.
- Local retrieval overhead target: <=250 ms p95.
- Start with 30 curated evaluation scenarios and grow from real failures.
</specifics>
