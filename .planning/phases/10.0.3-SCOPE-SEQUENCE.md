# 10.0.3 In-Flight Scope — Six-Item Sequence (operator-directed, 2026-06-20)

Operator added six items to scope and asked for execution "in the best sequence possible,"
formalized as GSD phases. These are **in-flight, ahead-of-spine** work under the 10.0.3 umbrella
(same pattern as `10.0.3-command-center` and `10.0.3-graphify-living-graph`), not new numbered
ROADMAP spine phases. The v1.0.5 wedge spine (10.0.1→10.0.6) is unchanged.

## Sequence & rationale

| # | Phase dir | Item | Why here | Depends on |
|---|-----------|------|----------|------------|
| 1 | `10.0.3-memory-router` | Memory router (FTS5/semantic → context assembly) | Highest leverage, fully self-contained, no foundation edits; directly compounds the just-shipped Command Center loop. The wiki FTS5 search already exists; only the wiring is missing. | — |
| 2 | `10.0.3-setup-wizard` | `atlas setup` + config-service ✅ **DONE** | Establishes `~/.atlas/config.yaml` + a config service. **Already shipped** in commit `76df72d` (config_service.py, cli/config.py, `atlas setup`, masked `GET /v1/config`, tests) — confirmed during 10.0.2 planning. #3 channel-cockpit is now unblocked. | — |
| 3 | `10.0.3-channel-cockpit` | Channel cockpit UI (System page expansion) | Exposes the 22+ foundation messaging adapters. Needs the config-service (read/write) and new gateway endpoints. | #2 config-service |
| 4 | `10.0.3-console-tiling` | Console hyprland BSP auto-tiling | Independent UI polish on the React Console workbench; slots in after the functional backend items. | — (console workbench exists) |
| 5 | `10.0.3-harness-cherrypick` | Cherry-pick from PI / OpenCode | Research intake (no local clones exist). Informs the harness direction and the de-brand that follows. Output is a doc, not code. | — |
| 6 | `10.0.7-foundation-debrand` | Gateway/foundation rebrand (hermes → atlas) | Release-gate, mechanical, large (~12.9k `hermes_cli` refs, foundation-locked tree, test-gated). Held last; it is a prerequisite for 10.0.6 public release, not a feature unlock. Already scoped in its own dir. | green foundation suite |

## Notes
- Items #1, #4, #5 are independent and could parallelize; #3 hard-depends on #2.
- #6 (de-brand) is the only item touching the D-001-locked foundation tree; it is a permitted,
  test-gated exception scoped to its own phase before public ship.
- Memory router semantic/embedding retrieval (sqlite-vec) is deferred within #1 — FTS5 first.
