## Decisions Made

- Mock branch lives in `native.py`'s `execute()`, not `config_service.resolve_provider` — keeps the credential-presence decision at the single runtime call site that already owns factory selection, leaving `resolve_provider` a pure config resolver.
- Sentinel-file idempotency (not `INSERT OR IGNORE` on a business key) for `demo_seed.py`, since `mission_service` has no `find_by_title` and missions lack a natural unique key — confirmed via direct source read before implementing, per the plan's explicit guidance.
- Demo wiki content resolves under `ATLAS_HOME/wiki` (not `atlas_wiki`'s own cwd-relative default) so demo data always lands in the ATLAS-owned home directory and is trivially cleanable.

- **Found during:** Task 1 full-suite verification (after committing Task 2)
