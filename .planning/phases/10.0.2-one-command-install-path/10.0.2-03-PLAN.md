<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `atlas doctor` -> config_service.resolve_provider() output | The resolved dict contains a dereferenced secret (`api_key`); doctor's output rendering is the boundary where that secret must never cross into stdout/logs |
| `atlas doctor` -> gateway/cockpit health probes | Local-loopback-only HTTP GETs (127.0.0.1), no untrusted network input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.0.2-09 | Information Disclosure | `atlas doctor` provider check echoing `resolved["api_key"]` | mitigate | Doctor only ever echoes `"configured"`/`"mock"`, never the dict's `api_key` value — Task 1 Test 4 explicitly asserts a fake key-shaped string injected into the patched return dict never appears in captured output |
| T-10.0.2-10 | Denial of Service | A single check raising an unhandled exception aborts the whole `atlas doctor` run, hiding other failing checks | mitigate | Each of the 5 checks is independently try/except-wrapped (Task 1 Test 5) so one broken subsystem (e.g. invalid config.yaml) does not prevent reporting on the other four |
</threat_model>
