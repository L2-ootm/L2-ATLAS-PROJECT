port-authority open question): the gateway binary reads `ATLAS_GATEWAY_PORT` (env var,
default 8484) for its bind port — it does NOT read `config.yaml`'s `gateway.rust_port`.
This means env-var (`ATLAS_GATEWAY_URL`/`ATLAS_GATEWAY_PORT`) is ALREADY the de facto
authoritative mechanism for the gateway; `config.yaml`'s `gateway.rust_port`/`cockpit.port`

existing `gateway_control.py` precedent); `cockpit_control.py` mirrors this with
`ATLAS_COCKPIT_URL` (default `http://127.0.0.1:5173` — Vite's actual served default, NOT
the unused `CockpitConfig.port=3000` default — see RESEARCH.md Pitfall 5/A3). Document
this precedence explicitly in code comments; the install/docs plan (10.0.2-04) writes the
operator-facing explanation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI -> subprocess (gateway binary, npm) | `atlas up` spawns two local child processes; argv is constructed from internal config/env, not user-supplied free text |
| CLI -> filesystem (PID files) | `~/.atlas/{gateway,cockpit}.pid` are read/written by the CLI under the current user's home dir |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.0.2-01 | Tampering | `cockpit_control.start()` subprocess argv | mitigate | argv is a fixed list built from `COCKPIT_URL`'s parsed port (an int) and hardcoded `npm`/`run`/`preview` tokens — never `shell=True`, never string-concatenated from external input; `_parse_port` only extracts an int from an env-var-or-default URL the operator controls locally |
| T-10.0.2-02 | Denial of Service | `atlas up` re-run while cockpit/gateway already running | mitigate | both `start()` functions check `health_ok()` first (idempotent no-op) per Task 1 Test 3/Task 2 Test 3 — prevents EADDRINUSE crashes documented in RESEARCH.md Pitfall 3 |
| T-10.0.2-03 | Information Disclosure | PID files under `~/.atlas/` | accept | PID values are not secret; file is current-user-readable only by default OS permissions, consistent with existing `gateway.pid` posture (no new exposure surface) |
| T-10.0.2-04 | Tampering | Windows console-window flash exposing process activity to onlookers | mitigate | `CREATE_NO_WINDOW` flag asserted by Task 1 Test 4 — closes the exact regression class fixed in commit `62e9456` for the gateway, now extended to the cockpit's npm-shim spawn |
</threat_model>
