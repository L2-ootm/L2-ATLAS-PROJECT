**Files modified:**

- `services/agent-runtime/tests/test_config_service.py`
- `services/web-ui-react/src/lib/api.ts`
- `services/web-ui-react/src/shell/Layout.tsx`

**Commits:** `2942dbb` (Python), `c45745c` (cockpit)

**Applied fix:** Wired the mock-mode signal end-to-end with no Rust change
(confirmed `lib.rs:548-553`'s `GET /v1/config` is a thin pass-through of
`atlas config json` → `masked_dict()`).
- `config_service.masked_dict()` now adds a derived `mock_mode` boolean
  computed via the existing `resolve_provider()` — True when no effective
  credential resolves. No secret value is added; only the boolean
  presence/absence of a resolved `api_key`.

  blank `api_key`, `mock_mode=True` when the `env:VAR` is unset
  (monkeypatched), `mock_mode=False` when it's set — and confirmed the
  resolved secret value never leaks into the masked dict.
- Cockpit: `AtlasConfigView` in `api.ts` gains `mock_mode?: boolean`
  (optional, so older gateways degrade gracefully).
- The actual cockpit app shell is `shell/Layout.tsx` (rendered on every
  route via react-router's `<Outlet/>` — `App.tsx` is the route manifest,
  not a rendered component). Layout now fetches config once on mount
  (mirroring the existing `useEffect`/`useState` pattern used in
  `System.tsx`, since this cockpit has no query-hook library) and renders
  `<MockModeBanner mockMode={...} />` above the routed content. Failure to
  fetch (gateway offline) degrades to `mockMode=false` (banner hidden).
- Verified: `npm run check` (tsc -b --noEmit) clean; new Python tests pass.

### WR-01: PID-file kill has no liveness/identity check before signaling

**File modified:** `services/agent-runtime/atlas_runtime/cockpit_control.py`,
`services/agent-runtime/tests/test_cockpit_control.py`

**Commit:** `e541a62`

**Applied fix:** Added `_pid_alive(pid)` with no new dependency (confirmed
`psutil` is not in `pyproject.toml` and was not added). POSIX uses
`os.kill(pid, 0)` (raises `ProcessLookupError` for a dead PID without
signaling); Windows shells out to `tasklist /FI "PID eq <pid>" /NH` and
checks whether the PID string appears in the output. `stop()` now checks
liveness before signaling — if the PID is already gone, it unlinks the
stale PID file and returns `"cockpit process already gone (pid N, removed)"`
instead of blindly calling `taskkill`/`os.kill`. Mirrors
`gateway_control.py`'s message conventions. Added 6 new tests covering the
already-gone branch, the alive-and-signaled branch, and `_pid_alive` on
both platforms (mocked).
