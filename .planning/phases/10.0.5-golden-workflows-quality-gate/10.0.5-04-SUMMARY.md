# Plan 10.0.5-04 — Quality Gate Smoke + Demo-Reset — SUMMARY

## What shipped

**Task 1 — the quality gate** (`services/agent-runtime/tests/test_golden_workflows_smoke.py`):
- `test_each_golden_workflow_runs_three_times_with_consistent_structure` — 3× each for
  repo_triage + research_brief; per call asserts the artifact row accrues (append-only,
  `== rep`), the wiki page exists, and the audit trail carries
  `golden_workflow_started`/`golden_workflow_completed` (plus `tool_requested`/`tool_completed`
  for repo_triage, which goes through the tool_service chokepoint). Cross-rep consistency =
  same `artifact_type` + a present non-null `sha256` each run (structure, not byte-equality).
- `test_self_review_three_runs_never_auto_writes` — the phase's most important assertion:
  self_review yields **exactly 3** pending approvals across 3 calls (`==`, not `>=`) and the
  proposed `golden/self-review-<date>.md` never lands on disk. A regression that silently
  auto-executes the gated write fails the count.
- `test_full_3x3_sequence_raises_no_exceptions` — the full 9-dispatch sequence runs clean
  ("repeated failures fixed" = gate green, not merely present).

**Task 2 — demo-reset** (`atlas golden reset` in `cli/golden.py`):
- Dry-run by default (reports would-delete counts, deletes nothing); `--confirm` to delete.
- Scoped to golden-tagged rows ONLY: `artifacts.path LIKE 'golden/%'`,
  `wiki_pages.slug LIKE 'repo-triage-%'/'research-brief-%'/'self-review-%'`,
  `tool_approvals.reason LIKE 'golden_workflow:%'`. Never touches audit_events/missions/runs.
- `--json` summary `{dry_run, artifacts_deleted, wiki_pages_deleted, tool_approvals_deleted}`.
