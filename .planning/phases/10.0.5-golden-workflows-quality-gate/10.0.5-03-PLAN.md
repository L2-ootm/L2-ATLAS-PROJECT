actual need rather than overloading the goal-bound `operation_app`.

the approval gate holds for the one workflow whose nature is "propose a write".

`cli/golden.py` wired into `cli/main.py`.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-1005-07 | Elevation of Privilege | self_review write path | mitigate | risk_level="write" in the manifest forces policy.decide().requires_approval=True unconditionally; self_review has no code path that bypasses tool_service.invoke |
| T-1005-08 | Tampering | golden_review_write args.path | mitigate | policy.check_workspace_boundary re-validated inside the adapter at execute-time (approve()), not trusted from the stored pending row alone |
