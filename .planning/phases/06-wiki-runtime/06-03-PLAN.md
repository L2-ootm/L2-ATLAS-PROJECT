FTS5 critical constraint — ALWAYS use this JOIN pattern:
```sql
SELECT wp.slug, wp.title, wp.source_id, bm25(wiki_fts) AS rank
FROM wiki_fts
JOIN wiki_pages wp ON wiki_fts.rowid = wp.rowid
WHERE wiki_fts MATCH ?
ORDER BY rank  -- bm25() returns negative; ASC = most relevant first
LIMIT ?
```
NEVER SELECT from wiki_fts alone — it is a content table with no row data.

    ingest_source(conn, lock, *, path, run_id, untrusted=False, wiki_dir):
    - Reads file bytes, computes SHA-256 with hashlib.sha256(content).hexdigest()
    - Constructs Source(path=path, sha256=sha256, size_bytes=len(content), untrusted=untrusted, ingested_by_run_id=run_id) BEFORE any SQL
    - Copies file to wiki_dir / "raw" / f"{source.id}_{pathlib.Path(path).name}" using shutil.copy2
    - Inside with lock: with conn: — checks SELECT id FROM sources WHERE sha256=? AND path=?; if found, UPDATE and replace source.id with existing id; if not found, INSERT
    - AFTER lock exits, calls emit(conn, lock, run_id=run_id, event_type="wiki_update", data={"slug": None, "source_id": source.id, "op": "ingest"})
    - Returns Source

    update_wiki_page(conn, lock, *, slug, title, body, source_id=None, run_id, wiki_dir):
    - Constructs WikiPage(slug=slug, title=title, body=body, source_id=source_id) BEFORE SQL
    - Inside with lock: with conn: — checks SELECT id, version FROM wiki_pages WHERE slug=?; if found, UPDATE with version+1 and new id; if not found, INSERT
    - AFTER lock exits, calls emit(conn, lock, run_id=run_id, event_type="wiki_update", data={"slug": slug, "source_id": source_id, "op": "update"})
    - AFTER emit, calls provenance_service.write_provenance(conn, lock, layer="WIKI", item_id=slug, run_id=run_id, source_id=source_id, audit_event_id=event.id)
    - Calls _update_index(wiki_dir, conn) and _append_log(wiki_dir, page, event)
    - Returns WikiPage

    search_wiki(conn, query, limit=10):
    - Executes: SELECT wp.slug, wp.title, wp.source_id, bm25(wiki_fts) AS rank FROM wiki_fts JOIN wiki_pages wp ON wiki_fts.rowid = wp.rowid WHERE wiki_fts MATCH ? ORDER BY rank LIMIT ?
    - Returns list of dicts with keys: slug, title, source_id, rank

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied path → ingest_source | Untrusted filesystem path; must be validated before shutil.copy2 |
| user-supplied query → search_wiki | FTS5 query string; must be parameterized, never string-interpolated |
| wiki page body → lint | Internal content; no external trust issue but contradiction detection must handle unicode safely |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | Path traversal in ingest_source path arg | mitigate | pathlib.Path(path).resolve().is_file() check before copy; raise ValueError for non-existent paths |
| T-06-06 | Tampering | FTS5 query injection | mitigate | Parameterized query with ? placeholder — not string interpolation in search_wiki |
| T-06-07 | Information Disclosure | Full page body in AuditEvent.data | mitigate | data dict contains only slug + source_id + op — not body content |
| T-06-08 | Tampering | emit() called inside with lock: | mitigate | Emit-after-lock pattern required; comment every emit() call site |
| T-06-09 | Availability | Large file ingest DoS | accept | size_bytes stored; warn if > 10MB (print warning, do not block) — production rate-limiting is Phase 7+ |
| T-06-SC | Tampering | sqlite_vec/fastembed lazy import | mitigate | Both imports inside function body inside try/except; module-level import is explicitly forbidden |
</threat_model>
