**Mapped:** 2026-06-25
**Scope:** config/auth files, frozen contracts, audit, model registry, CLI, Rust gateway

| Planned role | Closest shipped analog | Reuse rule |
|--------------|------------------------|------------|
| Frozen control-plane schemas | `agent_contract.py`, `surface_session.py` | `ConfigDict(frozen=True, extra="forbid")`; JSON-stable fields |

## Load-Bearing Patterns

### Frozen Public Contract

- Public response/request types live in `packages/atlas-core`.
- Use Literals for status/error vocabularies.
- No public raw dict payloads; nested frozen models or JSON strings only.
- Secret fields do not exist in public models.

### Shared Secure Store

- `secure_store.locked(path)` uses OS advisory lock on a sidecar.
- Reads and writes both acquire the lock.
- Temp file: `O_WRONLY|O_CREAT|O_EXCL`, mode `0o600`.
- Flush + file fsync; replace; parent fsync on POSIX.
- Apply `0600`/`icacls`; never `shell=True`.
- Preserve malformed source as `*.corrupt`; config fails closed, auth status reports corrupt.

### Revisioned Config Mutation

```text
lock
  load current + migrate in memory
  compare expected_revision
  apply dotted changes to model_dump copy
  validate frozen AtlasConfig
  increment revision once
  durable write
unlock
emit config_change(revision, masked diff, source)
return masked snapshot
```

### Typed Error Shape

Every expected failure exposes:

```text
code
message
remediation
current_revision?  # conflicts
field?             # validation/unknown key
```

CLI renders concise text; JSON/gateway preserves machine fields.

### Model Plane

- `model_registry.refresh()` retains current CLI behavior and additionally upserts
  provider/model v2 rows.
- The status projection queries v2 first and may read legacy rows for compatibility.
- An unavailable configured model is reported honestly; no silent fallback occurs in this phase.

## Pitfall Guards

| Pitfall | Guard |
|---------|-------|
| Deadlock from nested process/file locks | One file lock at a time; audit emitted after release |
| Lost update | expected revision checked under lock |
| Secret in audit | masked before/after + `SECRET_PATTERNS`; auth writes metadata-only |
| External credential mutation | hash/mtime/bytes invariant tests around status commands |
| Active-session drift | tests assert stored `SurfaceSession.model` remains unchanged after patch |
| CLI/gateway contract drift | same JSON request/response fixtures exercise both adapters |
| Windows command injection | `subprocess.run([...], shell=False)` asserted in tests |

## Assumptions

- Config/auth files remain small enough for whole-file locked replacement.
- Single-user local operation remains the v1.1 trust model.
- Gateway CLI dispatch overhead is accepted; Python service latency target excludes process spawn.
- Audit/file writes cannot be one physical transaction; success-path audit invariants are tested and
  failures are surfaced honestly.
