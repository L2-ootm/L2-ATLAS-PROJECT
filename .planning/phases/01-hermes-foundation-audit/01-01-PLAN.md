## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream GitHub → local filesystem | Fresh clone from NousResearch/hermes-agent; untrusted code arrives but is never executed |
| _EXTERNAL_REPOS/ (working copy) → git index | Hard constraint: gitignore rule prevents any clone file from entering ATLAS git history |
| ATLAS working tree → git commit | Secret-scan gate is a blocking checkpoint before any `git add` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | _EXTERNAL_REPOS/hermes-agent entering git history | mitigate | Add `_EXTERNAL_REPOS/` to .gitignore BEFORE clone; verify git status is clean after clone; acceptance criterion asserts no tracked files under _EXTERNAL_REPOS/ |
| T-1-02 | Information Disclosure | Runtime install at AppData vendored instead of upstream clone | mitigate | Task 1 action explicitly forbids copying AppData path; clone URL is hard-coded to upstream GitHub; HERMES_FOUNDATION_PIN.md warning cited |
| T-1-03 | Information Disclosure | .env / auth.json / *.db entering ATLAS git history | mitigate | Secret-scan gate (Task 2) runs git ls-files assertions and detect-secrets scan before CLONE_VERIFICATION.md is created; gate is a hard blocker |
| T-1-04 | Tampering | Windows CRLF corruption of cloned Python source | mitigate | Clone with --config core.autocrlf=false (Task 1 action step 2) |
| T-1-05 | Tampering | Malicious upstream code executed during audit | accept | Audit is read-only; no code from the clone is executed in this phase; clone is never pip-installed or run |
| T-1-SC | Tampering | npm/pip/cargo installs (detect-secrets) | mitigate | detect-secrets is a well-known Yelp OSS tool; pip install is bounded; no other package installs in this plan; PowerShell fallback avoids pip entirely if needed |
</threat_model>
