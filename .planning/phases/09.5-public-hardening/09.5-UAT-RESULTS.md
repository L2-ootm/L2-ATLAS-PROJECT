## Findings

### F1 — Cockpit fetched remote Google Fonts — RESOLVED 2026-06-15

**Resolution:** Inter / JetBrains Mono / Orbitron are now self-hosted via the
`@fontsource/*` packages; `tokens.css` imports local weight CSS (woff2 bundled by
Vite, content-hashed) and the remote `@import` is gone. Rebuild confirmed **zero**
`fonts.googleapis.com`/`fonts.gstatic.com` references in `build/`; cockpit re-verified
in-browser with typography intact and 0 console errors. Original finding below.

`services/web-ui/src/lib/tokens.css:6` did
`@import url('https://fonts.googleapis.com/css2?family=Inter…&family=JetBrains+Mono…&family=Orbitron…')`.
The built bundle carries the same `@import`
(`build/_app/immutable/assets/0.CAd_4-0e.css`). On every cockpit load this calls out
to Google's CDN, which:

- contradicts the stated non-negotiable "all components are loopback-only; nothing
  calls the network except the cockpit talking to `127.0.0.1:8484`" (RUNNING.md, D-005);
- leaks the operator's IP to Google;
- breaks the offline/local-first claim (fonts degrade to fallbacks offline).

**Fix before public publish:** self-host Inter / JetBrains Mono / Orbitron as local
`@font-face` (woff2 in `static/fonts/`) and drop the remote `@import`, OR explicitly
soften the loopback-only/offline claim. Functionally non-blocking (graceful fallback).

### F2 — Dynamic routes 404 under static preview (Low; deployment note)

`/runs/{id}` and `/missions/{id}` return HTTP **404** on direct load under `vite
preview`; the SPA fallback still boots and renders, and invalid IDs show a controlled
"mission not found" error. Production static hosting must be configured with an SPA
fallback (serve `200.html`/index with 200) so deep links don't surface a 404 status.
Already noted as "expected" in the 09.5 edge-case matrix.

### F3 — Prebuilt gateway binary was stale (Low; dev-hygiene / runbook)

The on-disk `target/release/atlas-gateway.exe` was built `2026-06-11`, predating the
`2026-06-12` CORS middleware in `lib.rs`. Running the cockpit against it showed
"GATEWAY OFFLINE" (CORS-blocked). `cargo build --release -p atlas-gateway` (18s)
fixed it. Local-only — fresh public checkouts build from source. RUNNING.md §2 should
tell operators to rebuild after source changes rather than trust an existing binary.
