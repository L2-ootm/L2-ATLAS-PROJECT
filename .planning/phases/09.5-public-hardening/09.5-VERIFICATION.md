## Idempotency / antifragility pass (Task 9)

| Surface | Canonical key | Behaviour on replay/retry | State | Required fix |
|---|---|---|---|---|
| Mission create | none (server-generated UUID) | retry creates a **duplicate** mission | operator-visible, low risk | future: optional client idempotency key |
| Run start (twice) | mission_id + pending state | 2nd call raises `ValueError` (mission no longer pending) | **safe** | none |
| Run cancel (twice) | run_id + running state | 2nd call raises `ValueError` (run terminal); audit preserved | **safe** | none |
| Run complete (twice) | run_id + running state | 2nd call raises `ValueError` (run not running) | **safe** | none |
| Audit event append | append-only, single transaction | no partial writes; no orphans (smoke-verified) | **safe** | none |
| Gateway `mission cancel` | dispatches `atlas mission cancel` | cancels **every** active run for the mission (documented in `api.ts`) | intentional, documented | none for v1.0 |
| SSE reconnect | rowid cursor | client resumes from last rowid; full reload re-reads history idempotently | **safe** | none |
| Fresh-DB bootstrap | migrations `IF NOT EXISTS` | re-running is a no-op | **safe / antifragile** | none |
| Browser reload during active run | read-only re-fetch | event history reloads; stream resumes | **safe** | confirm manual §9 |

State transitions are guarded inside a `threading.Lock` with the pre-condition check
in the same critical section (TOCTOU-protected), and audit rows are never deleted on
cancel/fail. The only non-idempotent surface is mission **create** (no natural key) —
low risk because it is operator-initiated and visible; flagged for a future idempotency key.
