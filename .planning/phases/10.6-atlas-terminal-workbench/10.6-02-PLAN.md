<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| control_plane_service snapshot -> header.py | Masked-at-source config data crossing into render layer; header.py must not re-resolve secrets |
| stdin TTY -> session_select.py | Untrusted interactive input (Project id choice) crosses into resolve_workspace |
| brand/ palette -> theme.py | Static asset read at import/render time, not user-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.6-01 | Information Disclosure | tui/header.py | mitigate | render_status_header only reads fields from the already-masked ControlPlaneSnapshot/SurfaceSession objects; never calls auth_service or any raw-secret resolver; test_header_never_contains_resolved_api_key asserts this explicitly |
| T-10.6-02 | Tampering | tui/session_select.py project_flag | mitigate | project_flag value passed verbatim to workspace_service.resolve_workspace, which raises WorkspaceError("unregistered"/"cross_project") on invalid/foreign ids rather than session_select inventing its own validation |
| T-10.6-03 | Denial of Service | tui/session_select.py interactive picker | mitigate | non-TTY + no-flags path raises ValueError immediately instead of blocking on prompt_toolkit input indefinitely |
| T-10.6-04 | Information Disclosure | tui/theme.py brand palette read | accept | brand/ is a static repo asset, not user input; no secret material possible |
</threat_model>
