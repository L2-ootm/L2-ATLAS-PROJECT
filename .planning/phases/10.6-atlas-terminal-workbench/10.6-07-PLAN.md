  str`, `deferred: bool = False`) and a `COMMAND_REGISTRY: dict[str, Callable[..., CommandResult]]`
  built via a `_register(name: str, handler: Callable, *, deferred: bool = False)` helper so every

  guarantee that satisfies "clean extension seams, not opaque error stubs": adding a real

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator composer text -> dispatch_command -> existing services | Untrusted free-text command/args cross into project/focus/config/permission service calls |
| dispatch_command outer try/except -> asyncio main loop | Last line of defense preventing any handler bug from crashing the TUI process |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.6-22 | Denial of Service | dispatch_command handler exceptions | mitigate | Outer try/except Exception around every handler call converts any failure into a CommandResult(ok=False, ...) instead of propagating into the asyncio main loop — the one deliberate broad-except in this phase, justified because it is the TUI's last line of defense against a single malformed command killing the whole session |
| T-10.6-23 | Elevation of Privilege | "permission" command group decide subcommand | mitigate | decide delegates to the SAME tui.permission_ui.resolve_approval_choice function the interactive dialog uses — no parallel/looser decision path exists via the composer; broker authority (claim/record_allow_rule) is identical regardless of entry point |
| T-10.6-24 | Tampering | Deferred group commands silently appearing to succeed | mitigate | Deferred handlers always return ok=False, deferred=True with an explicit named message; no deferred command can be mistaken for a successful no-op |
| T-10.6-25 | Information Disclosure | config get/set delegation | accept | Delegates to the same underlying function cli/config.py's Typer commands call, which already enforces D-013 masking; command_dispatch adds no new secret-exposure surface |
</threat_model>
