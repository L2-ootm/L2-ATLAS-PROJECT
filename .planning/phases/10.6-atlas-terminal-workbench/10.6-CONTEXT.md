# Phase 10.6: ATLAS Terminal Workbench - Context

**Gathered:** 2026-06-27
**Status:** Ready for planning
**Mode:** Smart discuss (autonomous)

<domain>
## Phase Boundary

Ship a **native** ATLAS terminal client launched by `atlas` and `atlas tui` for the existing
agent, Projects, Focus, missions/runs, configuration, tools, subagents/tasks, context, and
permissions. The workbench is a thin terminal surface over the contracts already built in phases
10.2-10.5 — it adds presentation, input, and a command dispatch layer, NOT new business logic,
new event buses, or new persistence. The current `atlas tui` (a wrapper over the vendored Hermes
Ink TUI) is replaced; de-brand criterion 5 forbids shipping imported product identity as the
default surface.

In scope: session startup/selection, status/identity header, streaming transcript + multiline
composer, normalized event rendering (agent/tool/task/subagent/retry/retrieval/context),
native permission prompts, Ctrl-C/cancel unwind, resume/replay with version pinning, cross-terminal
layout robustness, de-brand of source/bundles.

Out of scope (deferred): the full TUI-07 command palette breadth (wiki/Brain, subagents, context
command groups) — see Deferred Ideas. Tauri/native-shell packaging (roadmap-future).
</domain>

<decisions>
## Implementation Decisions

### Rendering & Input Stack
- Target the **best-in-class terminal UX**, modeled on the best of the vendored Hermes Ink
  foundation TUI (`foundation/atlas-hermes/`) and **mimocode** — port their strongest layout,
  streaming, and composer interaction patterns into ATLAS's Python surface.
- Vehicle: implement in the **leanest capable Python stack** — anchor on **Rich**
  (already a Typer transitive dependency: transcript/status/streaming, `Live` incremental render,
  built-in no-color + ASCII box modes, terminal width detection) **+ prompt_toolkit** (multiline
  composer: history, key bindings, Ctrl-C handling). Sync model — matches the existing Typer codebase.
- **Planning research MUST** (a) read the `foundation/atlas-hermes` Ink TUI source to extract its
  layout/streaming/composer UX patterns, and (b) web-research **mimocode** to identify the patterns
  the user wants emulated. The concrete library choice stays anchored to Rich + prompt_toolkit
  unless research surfaces a clearly leaner, superior option — record the rationale either way.
- Declare any new runtime dependency explicitly in `pyproject.toml` with a version constraint
  (supply-chain honesty); guard optional imports.

### Command Surface Disposition
- Both bare `atlas` (no subcommand) and `atlas tui` launch the new native workbench (TUI-01).
- Retire the Hermes Ink wrapper from the default path; if foundation launch must remain reachable,
  hide it behind an explicit dev/hidden flag — never as the default `tui` behavior.
- No Hermes/imported product identity in help text, console output, package names, paths, runtime
  output, or binaries (TUI-11 / criterion 5).

### Streaming / Refresh Model
- Poll the per-session event stream in-process (no SSE): fetch via
  `audit_service.get_events_for_session`, normalize with `surface_events.normalize_surface_events`,
  and use `replay_since(last_seq)` for gap-free, **append-only** incremental rendering.
- Use Rich `Live` for the status header + active streaming region; the transcript is append-only
  to avoid flicker and support narrow/wide reflow.
- Reuse the 10.3 contracts exactly — no new event bus, no new normalizer.

### Command Palette Scope (Core-first)
- **Must-have this phase:** TUI-01..06 and TUI-08..11 in full — session select (global workspace
  or registered Project), compact identity/model/auth/permission-mode/context-budget/Focus/session
  header, streaming transcript + multiline composer, normalized rendering of tool calls/results +
  diffs + tasks/subagents + retries + retrieval provenance + cancellation, native permission
  prompts (approve once / scoped allow / reject / cancel via `permission_broker`), Ctrl-C + cancel
  unwind without corrupting session state, resume/replay preserving workspace + prompt/context/
  tool-catalog versions (fail closed on drift via `agent_contract_service`), cross-terminal layout
  matrix, and de-brand.
- **Command groups delivered now (thin dispatch reusing existing services):** project/workspace
  selection, mission/focus, config/model, permissions.
- **Deferred to a follow-up:** wiki/Brain, subagents, and context command groups in the palette
  (the remainder of TUI-07). The dispatch layer should leave clean extension seams for these.

### Claude's Discretion
- Exact widget layout, key bindings, color tokens (draw from `brand/`), polling cadence, transcript
  virtualization, and the precise cross-terminal test matrix mechanics are at Claude's discretion,
  guided by the research findings and codebase conventions.
</decisions>

## Existing Code Insights

### Reusable Assets (consume, do not reinvent)
- CLI entry: `services/agent-runtime/atlas_runtime/cli/main.py` (`app`, Typer); entry point
  `atlas = "atlas_runtime.cli.main:app"`. Current wrapper: `cli/tui.py`
  (`_resolve_launcher`, `_foundation_dir`) — to be retired/repointed.
- Sessions: `atlas_runtime/surface_session_service.py` — `create_session`, `get_session`,
  `list_sessions`, `transition_session`, `heartbeat`, `reconcile_orphans`, `resume_session`.
- Session/event schema: `packages/atlas-core/atlas_core/schemas/surface_session.py` —
  `SurfaceSession`, `SessionState`, `SurfaceEvent`, `EventKind`.
- Event normalizer: `atlas_runtime/surface_events.py` — `normalize_surface_events`, `replay_since`.
- Audit source: `atlas_runtime/audit_service.py` — `emit`, `get_events_for_session`.
- Agent contract / resume: `atlas_runtime/agent_contract_service.py` — `RunContractSnapshot`,
  `ResumeSnapshot`, `prepare_run_contract`, `load_contract`, `replay_contract`
  (raises `ContractCompatibilityError` on version drift — TUI-09 fail-closed).
- Config/auth/model: `atlas_runtime/config_service.py` (`load_config`, `get_value`, `atlas_home`),
  `auth_service.py`, schema `packages/atlas-core/atlas_core/schemas/control_plane.py` (`AtlasConfig`,
  `ProviderConfig`, `PermissionConfig`); masked snapshot via `control_plane_service.get_config_snapshot`.
  CLI surface in `atlas_runtime/cli/config.py`.
- Permissions: `atlas_runtime/permission_broker.py` — `list_actionable` (session-scoped queue),
  `claim` (at-most-once, typed errors: `AlreadyDecided`, `WrongSessionError`,
  `NotActiveSessionError`, `ApprovalChannelMissingError`, `StaleApprovalError`),
  `record_allow_rule` / `match_allow_rule` (session-scoped allow-rules), `reconcile_executing_orphans`.
  Schema: `packages/atlas-core/atlas_core/schemas/tool.py` (`ToolApproval`, `SessionAllowRule`,
  `ApprovalChannel`).
- Brand assets: `brand/` (logos, palette, tokens).

### Established Patterns
- Typer CLI, sync; `typer.testing.CliRunner` for command tests.
- Frozen Pydantic schemas; redaction via `SECRET_PATTERNS` at the audit/persistence boundary (D-013).
- Surface events carry monotonic per-session `seq`; `replay_since` is the reconnect/gap primitive.
- Vendored foundation under `foundation/atlas-hermes` is no-edit (D-001).

### Integration Points
- New TUI module(s) under `atlas_runtime/cli/` (or a dedicated `atlas_runtime/tui/` package),
  wired into `cli/main.py` so bare `atlas` and `atlas tui` both launch it.
- Permission prompts call `permission_broker.list_actionable` + `claim`; allow-once/scoped map to
  `record_allow_rule` rule kinds.
- Resume path calls `surface_session_service.resume_session` + `agent_contract_service.replay_contract`.

### Testing Caveat (CRITICAL)
- CLI commands open `~/.atlas/atlas.db` (or `$ATLAS_HOME`) in production. Unit tests MUST inject the
  in-memory/temp DB via the `conftest.py` `db` fixture — NEVER drive the live CLI against the real DB.
- Cross-terminal (TUI-10) verification of real terminals (Windows Terminal, cmd, PowerShell, VS Code,
  WSL) is human-verify; automated tests cover width/no-color/ASCII/Unicode rendering logic.
</code_context>
