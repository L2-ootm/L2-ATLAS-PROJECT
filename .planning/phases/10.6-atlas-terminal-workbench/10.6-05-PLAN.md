<objective>
Implement the TUI-facing resume/replay orchestration wrapper (TUI-09) that turns Wave 0's RED
`test_resume_replay.py` stub GREEN. This module does NOT reimplement version-drift checking — it
is a thin error-rendering wrapper around `surface_session_service.resume_session`, which already
calls `agent_contract_service.replay_contract` BEFORE any state transition and fails closed via
`ContractCompatibilityError`.

Purpose: TUI-09 requires resume to "preserve workspace plus prompt/context/tool-catalog versions"
and fail closed on drift. The actual fail-closed logic already exists in
`surface_session_service.resume_session` (verified by direct code read during research, lines
377-414) — this plan's job is solely to give the TUI a clean call site that surfaces the
resulting error as a readable error card rather than letting an unhandled exception crash the
terminal, while guaranteeing it never re-orders or bypasses the existing check sequence.
Output: `atlas_runtime/tui/resume.py` with `resume_or_fail_closed`.
</objective>
