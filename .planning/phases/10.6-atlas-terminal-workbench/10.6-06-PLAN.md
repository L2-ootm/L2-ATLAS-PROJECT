RESEARCH.md Pattern 1 (single asyncio loop, code example structure):
```python
from prompt_toolkit.patch_stdout import patch_stdout
from prompt_toolkit import PromptSession

async def _main_loop(...):
    async with patch_stdout():
        ptk_session = PromptSession(multiline=True)
        poll_task = asyncio.create_task(_poll_loop(...))
        try:
            while True:
                line = await ptk_session.prompt_async("> ")
                ...
        except (EOFError, KeyboardInterrupt):
            stop_event.set()
        finally:
            poll_task.cancel()
            await asyncio.gather(poll_task, return_exceptions=True)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator composer input -> command_dispatch / agent submission | Untrusted free-text crosses into command parsing and (in a later phase) agent invocation |
| SIGINT/EOF -> handle_cancel -> transition_session | OS-level interrupt crosses into session-state authority |
| Background poll task -> SQLite WAL connection | Concurrent read access while the main loop may also write (heartbeat/transition) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10.6-18 | Denial of Service | tui/app.py composer line dispatch | mitigate | command_dispatch.dispatch_command (plan 10.6-07) is required to catch and render error cards for malformed input rather than propagate exceptions into the asyncio loop and kill the TUI process, per RESEARCH security domain "malformed composer command crashing the render loop" |
| T-10.6-19 | Denial of Service | Background poll task left running after Ctrl-C | mitigate | poll_task.cancel() + await asyncio.gather(poll_task, return_exceptions=True) in a finally block guarantees the task is reaped before patch_stdout's context exits, preventing an orphaned task racing a half-rendered event (RESEARCH Pitfall 4) |
| T-10.6-20 | Tampering | Rich Live racing prompt_toolkit for terminal control | mitigate | Live is constructed once, scoped exclusively to header updates; all transcript/event content flows through plain console.print under patch_stdout, never through a second Live region (RESEARCH Pattern 2 / Pitfall 1) |
| T-10.6-21 | Information Disclosure | _submit_to_agent extension stub | accept | Stub is a no-op placeholder explicitly logged/marked as not-yet-wired, not a silent swallow; no agent invocation occurs in this plan's scope, so no secret/credential path exists yet to leak |
</threat_model>
