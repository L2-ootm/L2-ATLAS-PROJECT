<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WikiPageForm input → gateway | User-authored wiki content sent as JSON body to gateway; gateway dispatches to atlas wiki update |
| Wiki content display | Page body rendered as HTML via minimal markdown renderer — must not use innerHTML unsafely |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-16 | Tampering | Wiki body markdown renderer | mitigate | Implement renderer using Svelte {@html} only after sanitizing: strip script tags, on* attributes, and iframe elements before rendering. Alternatively: render as pre-formatted text (white-space pre-wrap) if sanitization is complex — operator tool, single user, acceptable tradeoff |
| T-08-17 | Tampering | WikiPageForm slug input (create mode) | accept | Slug is passed to atlas wiki update CLI arg (--slug); atlas validates slug format; loopback-only, single operator |
| T-08-18 | Information Disclosure | Provenance SHA-256 display | accept | SHA-256 is a content hash, not a secret; loopback, single operator |
| T-08-19 | Denial of Service | Unbounded wiki search results | accept | API returns max 100 results (gateway clamp_limit); no risk for MVP |
| T-08-SC | Tampering | No new npm packages in this plan | accept | No new npm packages; markdown rendering is custom minimal implementation |
</threat_model>
