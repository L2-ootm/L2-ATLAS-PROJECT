## Layout Architecture

### Shell Structure

```
+--sidebar (56px collapsed / 200px expanded)--+--main content area (flex-1)--+
| [wordmark: L2 // SYSTEMS]                   |  [section header]            |
| [nav: Missions]                             |  [primary surface panel]     |
| [nav: Runs]                                 |  [data content]              |
| [nav: Wiki]                                 |                              |
| [nav: Models]                               |                              |
+---------------------------------------------+------------------------------+
```

- Sidebar: fixed left, `width: 56px` collapsed / `200px` expanded, toggle on icon click.
- No top navbar. Four nav items only: Missions, Runs, Wiki, Models.
- Main content: `flex-1`, `overflow-y: auto`, `padding: 24px` (--l2-space-6).
- App root background: `#030305` (--l2-void-page). Sidebar background: `#050505` (--l2-void-base).

### Sidebar Behavior

- Collapsed state: icon only (20px Lucide, centered). No text labels.
- Expanded state: icon (16px) + label (Inter 600, 14px, uppercase, letter-spacing 0.1em).
- Active nav item: `border-left: 2px solid #00F0FF` + `background: rgba(0,240,255,0.06)` + color `#00F0FF`.
- Inactive nav item: `border-left: 2px solid transparent` + color `#505050`.
- Hover (inactive): `background: rgba(255,255,255,0.02)` + color `#A0A0A0`. Duration: 80ms.
- Sidebar bottom: wordmark `L2 // SYSTEMS` (Orbitron, 12px, uppercase, collapsed: hidden).

## Typography

Source: `design-tokens.json` typography block + DESIGN.md §4.

### Font Roles (enforced, no exceptions)

| Role | Font | Token | Usage |
|------|------|-------|-------|
| UI body / labels / nav | Inter | --l2-font-sans | All prose, descriptions, nav labels |
| Display / wordmark only | Orbitron | --l2-font-display | Section headers at h2 size, wordmark, no body use |
| ALL data / IDs / timestamps / metrics / code | JetBrains Mono | --l2-font-mono | Every datum, every ID, every timestamp — mandatory |

### Type Scale

| Level | Size Token | Px | Font | Weight | Line Height | Usage |
|-------|------------|-----|------|--------|-------------|-------|
| H2 / Section | --l2-text-h2 | 32px | Orbitron | 700 | 1.2 | Surface section titles (e.g. "MISSIONS") |
| Body | --l2-text-body | 16px | Inter | 400 | 1.5 | Descriptions, mission intent, wiki content, panel sub-headings |
| Small / Nav label | --l2-text-sm | 14px | Inter | 600 | 1.4 | Nav labels, form labels, secondary copy |
| Data / HUD | --l2-text-xs | 12px | JetBrains Mono | 400 | 1.2 | ALL mono data: mission IDs, run IDs, latency, token count, status badges, HUD labels, SSE event types, timestamps |

Rules:
- `letter-spacing: 0.2em` on all HUD labels (uppercase mono system labels).
- `letter-spacing: 0.1em` on nav labels and button text.
- `font-variant-numeric: tabular-nums` on all JetBrains Mono data.
- Hero/H1 (96px/64px Orbitron) are NOT used in the cockpit — cockpit is data-dense, not landing page.
- Orbitron is restricted to section titles (32px) and the wordmark. Never body text, never data.
- Inter weights: exactly 2 — 400 (body, descriptions, metadata) and 600 (nav labels, sub-labels, headings). No other Inter weights.
- JetBrains Mono weight: 400 throughout. All mono data at 12px. No separate 13px data tier.

## SSE Stream Contract

Source: DESIGN.md §7 SSE Event Stream Entry + CONTEXT.md COCKPIT-02/03.

### Live Stream Behavior

- Connection: `EventSource` to `GET /v1/runs/{id}/stream` on `127.0.0.1:8484`.
- On open: container border-left activates Cyber Blue (`#00F0FF`), `LIVE` badge appears.
- `LIVE` badge: JetBrains Mono 12px uppercase, `#00F0FF`, `background: rgba(0,240,255,0.12)`, `border: 1px solid rgba(0,240,255,0.35)`, `border-radius: 4px`, `padding: 4px 8px`.
- Live pulse: `LIVE` badge has a single `opacity: 1 → 0.5 → 1` CSS animation, `1.5s infinite`, only when connected. No other idle animations.
- On `end` event received or run reaches terminal state: SSE closes, `LIVE` badge replaced by run status badge, Cyber Blue container border removed.
- Reconnect on disconnect: single retry after 2000ms. If retry fails, show `STREAM INTERRUPTED — reconnecting` in error crimson at top of stream container.

### Row Rendering

- Each SSE event maps to one `sse-event-row`.
- Timestamp column: ISO-8601 formatted as `HH:MM:SS.mmm` (local time). Color: `rgba(0,240,255,0.5)`.
- Event type column: value from `event_type` field, uppercase. Color:
  - `TOOL_CALL`, `TOOL_RESULT` → Signal Teal `#00E5C8`
  - `LLM_CALL`, `LLM_RESPONSE` → Electric Violet `#7F00FF`
  - `ERROR`, `EXCEPTION` → Error Crimson `#FF0055`
  - `RUN_START`, `RUN_END`, `CHECKPOINT` → Cyber Blue `#00F0FF`
  - All others → Mercury Grey `#A0A0A0`
- Payload column: raw JSON payload truncated at 120 chars, `…` suffix. Click to expand inline (full payload, no modal, `white-space: pre-wrap`, JetBrains Mono 12px).
- Maximum rows rendered in DOM: 500. Older rows virtualized/removed when limit exceeded to maintain 60fps.

## Topographic System

Source: DESIGN.md §2 + tokens.css `--l2-topo-*` tokens.

### Implementation

- Engine: `wireTopo()` from `~/.claude/skills/L2-Systems-Design-System/topo_engine.js`.
- Import in SvelteKit: `onMount(() => { import('topo_engine.js').then(m => m.wireTopo()) })`.
- Attaches to document root via `::before` pseudo-element.
- Pattern: SVG contour-line terrain. Line color: `rgba(255,255,255,0.06)` (--l2-topo-line).
- At rest: `opacity: 0.5`. Never louder than 0.5 at idle.

### Activation Rules (prescriptive — per surface)

| Surface / Element | Topo Glow Color | Token | Trigger |
|-------------------|-----------------|-------|---------|
| Missions section | Violet | --l2-topo-glow-brand | Cursor over mission list panel |
| Mission create form | Violet | --l2-topo-glow-brand | Form open |
| SSE stream container (active run) | Info/Blue | --l2-topo-glow-info | SSE connection open |
| SSE stream container (succeeded) | Success | --l2-topo-glow-good | Run terminal: SUCCEEDED |
| SSE stream container (failed) | Error | --l2-topo-glow-bad | Run terminal: FAILED |
| Wiki browser | Violet | --l2-topo-glow-brand | Cursor over wiki panel |
| Model registry | Info/Blue | --l2-topo-glow-info | Cursor over model table |
| Primary CTA buttons | Violet | --l2-topo-glow-brand | Button hover |
| Danger button | Error | --l2-topo-glow-bad | Button hover |
| Input fields (focus) | Violet | --l2-topo-glow-brand | Input focus |
| Sidebar (global) | None | — | No topo on sidebar — too narrow |

### Constraints

- Topo is background only. `z-index: 0` or lower. Content always above (`z-index: 1+`).
- No topo parallax on `prefers-reduced-motion: reduce`.
- Parallax factor: `0.4` (--l2-topo-parallax). Scroll-linked, not idle.
- Idle decorative loops: **banned**. Only the LIVE badge pulse is permitted.

## WebView2 Portability Constraints

Source: CONTEXT.md D-021 (native-portable, Phase 10 zero-rework).

- No `window.showOpenFilePicker` or other privileged File System Access APIs.
- No `navigator.bluetooth`, `navigator.serial`, or hardware APIs.
- No `SharedArrayBuffer` (requires COOP/COEP headers not available in WebView2 default config).
- SSE via `EventSource` is supported in WebView2 — use it.
- `fetch` to `127.0.0.1` is supported — use it for all API calls.
- `localStorage` is supported — use for sidebar collapse state only.
- No `crypto.subtle` in insecure origins — not needed (no auth in v1.0).
- CSS `backdrop-filter` is supported in Chromium-based WebView2 — use it.
- `prefers-color-scheme` — always dark; no light mode toggle required.

## Accessibility Floor

Minimum requirements (operator tool, single-user local — full WCAG 2.1 AA is Phase 10 goal):

- All interactive elements keyboard-focusable.
- Focus ring: `outline: 2px solid rgba(127,0,255,0.6)`, `outline-offset: 2px`.
- `aria-label` on all icon-only buttons (collapsed sidebar nav items, ghost icon actions).
- `role="log"` and `aria-live="polite"` on SSE stream container.
- `role="status"` on `LIVE` badge.
- Color is never the sole differentiator for status — status badges include text label.
- `prefers-reduced-motion: reduce` — disable topo parallax and all transitions > 80ms.
