# Phase 10.8 — Go TUI Corrective Slice

**Date:** 2026-07-01
**Status:** Implementation and automated verification complete; interactive operator UAT pending

## UAT failure addressed

The first operator test launched a stale 2026-06-28 source-checkout executable, displayed a
dashboard instead of an agent conversation, exposed old global approval history, and offered no
usable path from an unavailable API-key profile to a real agent.

The corrective implementation preserves the Go/BubbleTea sidecar architecture and shared
Rust/Python authority while replacing the interaction model.

## Delivered behavior

- Composer is focused at launch; `Enter` submits and `Alt+Enter` inserts a line.
- Idle state is chat-first and contains no permanent provider/mission/permission dashboard.
- Active state is transcript-first with a persistent composer and optional contextual sidebar.
- One surface session is the visible conversation; internal mission/run records implement turns.
- A second turn preserves the visible transcript and surface identity.
- Implicit mock execution is blocked. Draft text is preserved and provider setup opens.
- Mock execution requires the explicit `ATLAS_TUI_ALLOW_MOCK=1` development override.
- Unconfigured launch lists API key, Codex OAuth, Claude Code, and other reported provider modes.
- Claude Code auth mode dispatches the `claude_code` agent runtime instead of hard-coded native.
- Slash commands expose settings, history, approvals, context, new session view, help, and quit.
- Owned approvals keep recent transcript context visible and retain four-choice scope behavior.
- `Ctrl+C` requests surface cancellation while work is active; idle `Ctrl+C` exits.
- ASCII/Unicode and narrow/wide layouts retain the same information hierarchy.

## Resolved executable integrity

The Python launcher now compares the source-checkout executable with `.go`, `go.mod`, and
`go.sum`. A missing or stale executable is rebuilt before launch. Builds embed a version/commit
identity and `atlas-tui --version` exposes it.

Fresh resolved-artifact evidence:

- artifact: `services/atlas-tui/atlas-tui.exe`
- size: 8,340,480 bytes (7.95 MiB; below the 15 MiB ceiling)
- retired `/v1/tools/approvals` string hits: 0
- owned `/v1/surface-sessions/` route string hits: 1
- version probe: `atlas-tui dev (working-tree)` before final commit

The ignored checkout executable must be rebuilt with the final commit identity after commit.
