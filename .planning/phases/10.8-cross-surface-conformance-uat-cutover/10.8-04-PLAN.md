<objective>
Success criteria 4 and 5: operator UAT with runbooks, then the cutover/retirement decision
with a tested rollback, then v1.1 archive. This plan is NOT autonomous — it prepares and
sequences the human gates.
</objective>

<tasks>

1. **UAT runbook.** One script per surface covering: session start (global + project
   workspace), a real provider run, Brain/wiki retrieval visible with provenance, a
   permission ask decided on the owning surface, forced-failure recovery (kill gateway
   mid-run), audit attribution check (`atlas` CLI query of the run's audit rows), and
   rollback drill. Include the F12 protocol: if the session toast appears, capture
   `%TEMP%\atlas-terminal-diagnostics.log` before anything else. Fold in the eight
   interactive Go TUI checks from 10.8-TUI-CORRECTION.md.

2. **Blocker gate.** UAT sign-off requires: F12 root-caused and fixed (not just
   instrumented), the Go TUI interactive checklist passing, and the 10.8-02/-03 suites
   green on the UAT day (fresh run, not a cached result).

3. **Retirement/rollback runbook.** Document how the default `atlas` entry switches to
   atlas-terminal, the config/env knob that restores the Go TUI, and the retention window
   for the legacy path (no indefinite duplicate stack — set a dated removal milestone).
   Rehearse the rollback once and record it.

4. **UAT record + decision.** `10.8-UAT-RECORD.md`: operator acceptance per runbook item,
   the dated retirement decision (go / no-go / deferred with reason), and the evidence
   links. DOC-02 flips only when this file carries the operator's decision.

5. **Milestone close-out.** After the decision: flip remaining REQUIREMENTS checkboxes with
   citations, push the accumulated unpushed commits, archive v1.1 via the milestone
   workflow, and activate the v1.2 provider-mesh draft roadmap.

</tasks>
