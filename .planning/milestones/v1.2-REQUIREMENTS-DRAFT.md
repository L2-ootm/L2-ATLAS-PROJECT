# v1.2 Requirements Draft — Provider Mesh & Runtime Interoperability

**Status:** Draft only; do not merge into active `.planning/REQUIREMENTS.md` until v1.2 activation
**Phase mapping:** milestone-local `PM-*`

## Foundation boundary

- [ ] **PM-BND-01:** Every foundation module is classified as `tracked`, `owned`, or `diverged-frozen`.
- [ ] **PM-BND-02:** Upstream deltas are partitioned deterministically with machine-readable evidence.
- [ ] **PM-BND-03:** Known identity transforms are explicit and reproducible.
- [ ] **PM-BND-04:** v1.2 boundary tooling is report-only and cannot apply or commit changes automatically.

## Runtime and routing contract

- [ ] **PM-RUN-01:** The existing `AgentRuntime` contract exposes versioned capabilities.
- [ ] **PM-RUN-02:** Each backend declares one capability tier: model gateway, controlled runtime, or opaque runtime.
- [ ] **PM-RUN-03:** Controlled runtimes prove normalized events, cancellation, workspace binding, and permission bridging before write access.
- [ ] **PM-RUN-04:** Opaque runtimes cannot be presented as ATLAS-enforced or used for write-capable sessions.
- [ ] **PM-RUN-05:** Global defaults resolve into immutable project/session/run bindings.
- [ ] **PM-RUN-06:** Active sessions cannot be hot-switched to a different runtime, mode, or profile.
- [ ] **PM-RUN-07:** ATLAS-to-ATLAS routes carry trace IDs, bounded hop counts, and cycle detection.

## Profiles and dispatch

- [ ] **PM-PRO-01:** Role-keyed provider/model/effort profiles use one versioned config schema.
- [ ] **PM-PRO-02:** Profile edits are atomic, conflict-aware, masked, and shared by CLI and WebUI.
- [ ] **PM-PRO-03:** Each run records the resolved profile version and exact role bindings.
- [ ] **PM-PRO-04:** Every function declares `routed`, `inherited`, or `unavailable`; surfaces cannot offer an override the runtime cannot honor.
- [ ] **PM-PRO-05:** The initial registry maps the real foundation seams: main, curator, and named auxiliary tasks; background review remains inherited until a supported seam exists.
- [ ] **PM-DSP-01:** Dynamic scoring is deterministic and explainable from capability, cost, health, latency, and failure signals.
- [ ] **PM-DSP-02:** Explicit rulebook assignments override dynamic scoring.
- [ ] **PM-DSP-03:** Health discovery is local/read-only by default; network or paid probes are opt-in, cached, timed out, and budgeted.
- [ ] **PM-DSP-04:** Fallback behavior follows the Phase 10.0 error-classification contract.
- [ ] **PM-TST-01:** One shared connectivity probe reports live/mock/failed, latency, and bounded sample output across CLI, TUI, and WebUI.
- [ ] **PM-TST-02:** Routed functions have canonical smoke tests; inherited/unavailable functions are reported honestly rather than simulated.
- [ ] **PM-TST-03:** Candidate models can opt into the Phase 10.2 evaluation gate under an explicit network/token budget.
- [ ] **PM-TST-04:** Probe missions/runs are auditable and cleaned up or archived deterministically.

## Backends and identity

- [ ] **PM-BCK-01:** Every shipped backend passes conformance for its declared capability tier.
- [ ] **PM-BCK-02:** Backends have bounded startup, timeout, cancellation, and recovery behavior.
- [ ] **PM-BCK-03:** External product integrations remain adapters; donor runtime/config/auth/telemetry is not imported.
- [ ] **PM-BCK-04:** Operator-facing native/foundation labels are ATLAS-native; upstream names appear only where integration identity or provenance requires them.
- [ ] **PM-BCK-05:** Attach, detach, route, selection, fallback, failure, and capability decisions are audited.

## UX, security, and budgets

- [ ] **PM-UX-01:** Discovery reports installed local capabilities before requesting configuration.
- [ ] **PM-UX-02:** The least invasive supported path is the default; migration is explicit and reversible.
- [ ] **PM-UX-03:** Empty, unavailable, unhealthy, denied, and misconfigured states explain the next safe action.
- [ ] **PM-UX-04:** `atlas setup`, `atlas models`, and `atlas profiles` expose the same function/profile contract as the graphical surfaces.
- [ ] **PM-UX-05:** The Go TUI exposes current-provider wiring and direct probes without owning auth/config/run business logic.
- [ ] **PM-UX-06:** The WebUI Models suite includes provider modes, model catalog, function matrix, test results, and effective resolution.
- [ ] **PM-UX-07:** API-key entry is hidden and transported without argv exposure; Codex import, Claude Code, and FreeLLMAPI show mode-specific remediation/privacy states.
- [ ] **PM-SEC-01:** Secrets remain ATLAS-owned references and never cross masked APIs or audit output.
- [ ] **PM-SEC-02:** Capability mismatches and unsupported permission bridges fail closed.
- [ ] **PM-BUD-01:** Added dependencies, startup time, idle memory, probe traffic, and binary/bundle growth have explicit measured budgets.
