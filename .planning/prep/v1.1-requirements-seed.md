# v1.1 Requirements Seed — Agentic CLI + Native Operator Shell

**Status:** seed document for `/gsd-new-milestone`; not final requirements.  
**Created:** 2026-06-15  
**Reason:** v1.0 visual CLI inspection exposed that ATLAS has a cockpit MVP but not a complete agentic CLI/harness surface.

## Milestone candidate

**v1.1 — Agentic CLI + Native Operator Shell**

## Candidate goal

Turn ATLAS from a browser cockpit backed by an operational CLI into a credible local operator harness:

1. ATLAS-branded agentic CLI;
2. local auth/provider discovery;
3. real model inventory;
4. native shell that visually hosts cockpit + terminal/CLI pane.

## Draft requirements

### CLI — Agentic command surface

| ID | Requirement | Acceptance signal |
|---|---|---|
| CLI-01 | User can run `atlas --help` and see an ATLAS-branded command tree. | `atlas --help` works without `python -m`; command tree includes chat/auth/models/doctor. |
| CLI-02 | User can run `atlas chat -q "..."` for one-shot agentic interaction. | Returns a model response or precise auth remediation. |
| CLI-03 | User can open an interactive agentic CLI session. | `atlas chat` or equivalent starts an interactive session and exits cleanly. |
| CLI-04 | User can run `atlas doctor` for environment readiness. | Reports Python/Rust/Node, DB, gateway, cockpit, auth, models, and foundation status. |
| CLI-05 | CLI output never leaks secrets. | Tests scan output for token/API-key patterns and known credential file contents are never printed. |

### AUTH — Local provider/auth detection

| ID | Requirement | Acceptance signal |
|---|---|---|
| AUTH-01 | ATLAS detects local Codex installation. | `atlas auth status` reports Codex installed/version when `codex` is on PATH. |
| AUTH-02 | ATLAS detects Codex auth/config presence safely. | Reports `auth present` / `missing` without printing file contents. |
| AUTH-03 | ATLAS detects Hermes/ATLAS provider config. | Reports active profile and configured providers. |
| AUTH-04 | Auth status is actionable. | Missing auth includes exact next command or file path to configure, without guessing. |

### MODELS — Real model discovery

| ID | Requirement | Acceptance signal |
|---|---|---|
| MODELS-01 | ATLAS merges model sources. | Seeded + gateway + FreeLLMAPI + Hermes config + Codex/OpenRouter/custom providers are represented when available. |
| MODELS-02 | Model list shows source/status/auth. | `atlas models list --all` includes model_id, provider, source, status, auth, last_seen. |
| MODELS-03 | Model discovery is idempotent. | Repeated discovery does not duplicate rows or erase audit-relevant history. |
| MODELS-04 | Cockpit Models page reflects real status. | Browser model panel shows sources/status, not only three seeded rows. |

### NATIVE — Shell and PTY

| ID | Requirement | Acceptance signal |
|---|---|---|
| NATIVE-01 | Tauri 2 shell wraps the existing SvelteKit cockpit. | Native app opens cockpit locally without Electron. |
| NATIVE-02 | Shell includes PTY/terminal pane. | User can run `atlas --help` in the embedded terminal. |
| NATIVE-03 | IPC is capability-scoped. | Threat model exists and IPC commands are enumerated/limited. |
| NATIVE-04 | Shell surfaces auth/model readiness. | Native UI can show readiness and route to CLI remediation. |
| NATIVE-05 | Shell remains local-first. | No external network calls except explicit model/provider integrations. |

## Non-goals for v1.1 unless explicitly approved

- CRM/Twenty production integration.
- WhatsApp production channel.
- Full public installer/distribution polish.
- Multi-user SaaS/auth.
- Self-modifying autonomous agent behavior.
- Rebuilding Hermes internals instead of wrapping/exposing the vendored foundation.

## Risk register

| Risk | Why it matters | Mitigation |
|---|---|---|
| Duplicating Hermes chat runtime | Creates permanent fork burden. | Thin ATLAS wrapper first; reuse foundation. |
| Leaking auth material | Auth/status code touches credential files. | Presence/status only; redaction tests. |
| Model registry bloat | Multiple sources can produce confusing duplicates. | Source/status schema + deterministic merge. |
| Native shell before CLI | Shell becomes empty visual wrapper. | Gate shell completion on `atlas chat -q` and auth/model readiness. |
| Scope creep into CRM/Pulse | v1.1 loses focus. | Keep CRM/Pulse for v2.0 unless explicitly pulled forward. |

## Recommended execution order

1. Auth/provider discovery spike.
2. `atlas auth status` + redaction tests.
3. `atlas chat -q` wrapper over foundation.
4. `atlas models discover/list --all` merged registry.
5. Cockpit Models page alignment.
6. Tauri shell + PTY.
7. End-to-end UAT: CLI + cockpit + shell.
