# ATLAS v1.1 — Exhaustive Backlog Seed

**Status:** comprehensive candidate backlog; prune during `/gsd-new-milestone`.  
**Created:** 2026-06-15.  
**Purpose:** capture everything that may belong to v1.1 so it is not forgotten.

## P0 — Must address for v1.1 credibility

### TUI / agent UX

- [ ] ATLAS-branded TUI exists.
- [ ] TUI is based on Hermes TUI architecture or explicitly rejects it with evidence.
- [ ] TUI has transcript, composer, streaming, activity, status bar.
- [ ] TUI shows provider/model/auth status.
- [ ] TUI supports ATLAS mission context.
- [ ] TUI supports slash/command help.
- [ ] TUI can run one prompt end-to-end.
- [ ] TUI handles no-auth state with remediation.
- [ ] TUI exits cleanly.
- [ ] TUI has Windows terminal UAT.

### CLI

- [ ] `atlas` direct command works.
- [ ] `atlas --help` has ATLAS-branded command tree.
- [ ] `atlas chat -q` exists.
- [ ] `atlas chat` interactive exists or routes to TUI.
- [ ] `atlas doctor` exists.
- [ ] `atlas auth status` exists.
- [ ] `atlas models discover` exists.
- [ ] `atlas models list --all` exists.
- [ ] Commands use clear exit codes.
- [ ] Commands never print secrets.

### Auth

- [ ] ATLAS-owned auth store exists.
- [ ] Auth store lives under `~/.atlas` or ATLAS profile path.
- [ ] Atomic write implemented.
- [ ] Cross-process lock implemented.
- [ ] Redacted status output implemented.
- [ ] `atlas auth add` supports at least one provider.
- [ ] `atlas auth remove` safely removes credential.
- [ ] `atlas auth doctor` checks permissions/parse/token state.
- [ ] Codex installation detected read-only.
- [ ] No Codex mutation: `~/.codex` is never mutated by ATLAS.
- [ ] Tests prove no Codex mutation.
- [ ] OAuth feasibility documented.
- [ ] Manual paste/browser fallback documented.

### Provider/model registry

- [ ] Providers modeled separately from models.
- [ ] Credentials modeled separately from providers.
- [ ] API mode/runtime adapter modeled.
- [ ] Model rows include source/status/auth.
- [ ] Discovery merges seeded + config + auth + sidecar + external read-only sources.
- [ ] Source-scoped deactivation preserves audit history.
- [ ] `atlas models doctor` explains missing/offline providers.
- [ ] Cockpit Models page no longer implies only three seeded models exist.

### Agent runtime

- [ ] ATLAS runtime adapter over Hermes AIAgent selected.
- [ ] One-shot chat uses real provider or gives auth remediation.
- [ ] Interactive chat works.
- [ ] Model calls emit audit metadata where feasible.
- [ ] Tool calls visible in TUI/activity.
- [ ] Dangerous tool approvals preserved.
- [ ] Session state is persisted/resumable.

### Native shell

- [ ] Tauri 2 selected/scaffolded.
- [ ] Existing SvelteKit cockpit embedded.
- [ ] PTY terminal pane runs `atlas --help`.
- [ ] PTY pane can run `atlas tui` or `atlas chat`.
- [ ] IPC allowlist documented.
- [ ] Threat model documented.
- [ ] No Electron.
